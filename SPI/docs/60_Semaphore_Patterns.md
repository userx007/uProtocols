## Introduction

The Serial Peripheral Interface (SPI) bus is one of the most widely used communication protocols in embedded systems — connecting microcontrollers to sensors, displays, ADCs, DACs, flash memory, and more. SPI operates in a **single-master / multi-slave** topology where:

- One master controls the clock (`SCLK`) and data lines (`MOSI`, `MISO`)
- Each slave is selected by a dedicated **Chip Select** (`CS`) line
- The bus is **inherently not reentrant**: only one transaction may be in flight at a time

As soon as multiple software components — tasks, threads, interrupt handlers, or async coroutines — share the same SPI bus, a **bus access control problem** arises. Without synchronization, two tasks can interleave SPI transactions, corrupting the protocol framing and producing garbage data.

**Semaphores** provide the classical solution. This document covers:

- The theory of binary and counting semaphores
- How they map onto SPI bus access patterns
- Practical, production-grade code in **C/C++** and **Rust** for both RTOS (FreeRTOS) and POSIX/`tokio` environments

## SPI Bus Fundamentals & the Concurrency Problem

### Typical Multi-Task SPI Scenario

```
Task A (Sensor Read)     Task B (Display Write)
        |                        |
   CS_SENSOR low            CS_DISPLAY low
   MOSI: [CMD][DATA]        MOSI: [PIXEL_DATA...]
        ...                      ...
   CS_SENSOR high           CS_DISPLAY high
```

If both tasks start a transaction simultaneously on the same physical bus:

- `SCLK` edges are generated by both tasks' SPI peripheral register writes
- `MOSI` bytes collide
- Neither slave receives a valid frame

This is a **classic critical section** problem. The SPI transaction (CS low → clock data → CS high) must be treated as an **atomic unit** from the bus's perspective.

### Resource Contention Categories

| Scenario | Semaphore Type Needed |
|---|---|
| Single SPI bus, N tasks | Binary semaphore (mutex-like) |
| N identical SPI peripherals (pool) | Counting semaphore |
| One bus, N slaves, priority tasks | Priority-inheriting mutex or binary semaphore |
| Async I/O with back-pressure | Counting semaphore (permit-based) |

## Semaphore Theory

### Binary Semaphore

A **binary semaphore** has a value of either `0` (unavailable) or `1` (available).

| Operation | Also Called | Effect |
|---|---|---|
| `wait()` / `P()` | `acquire`, `take`, `down` | Decrements value; blocks if `0` |
| `signal()` / `V()` | `release`, `give`, `up` | Increments value; unblocks a waiter |

For SPI bus protection:

- Initialized to `1` (bus free)
- Task calls `wait()` before asserting CS
- Task calls `signal()` after de-asserting CS
- A second task trying to `wait()` while the bus is in use **blocks** until the first signals

**Key property:** A binary semaphore (unlike a mutex) can be **given by a different task than the one that took it**. This is important for ISR-to-task signalling patterns but requires discipline for bus protection.

### Counting Semaphore

A **counting semaphore** holds an integer value `0 ≤ n ≤ MAX`.

- Initialized to `N` (number of available resources, e.g., SPI bus instances)
- Each `wait()` decrements the count
- Each `signal()` increments the count
- Blocks when count reaches `0`

For SPI, counting semaphores are used when you have a **pool of identical SPI buses or peripherals** — allow up to N concurrent transactions across the pool.

### Semaphores vs. Mutexes

| Property | Binary Semaphore | Mutex |
|---|---|---|
| Ownership | None — any task can signal | Only the acquiring task can release |
| Priority Inheritance | Usually no | Often yes (RTOS mutexes) |
| ISR signalling | Yes | No (ISRs cannot own a mutex) |
| Recursive locking | No | Optional (`PTHREAD_MUTEX_RECURSIVE`) |
| Best for SPI | Task-level bus lock | Task-level bus lock (preferred) |
| Best for ISR → Task | Signal from ISR | Not suitable |

> **Recommendation:** For SPI **bus access control** between tasks, prefer a **mutex** (which is a binary semaphore with ownership semantics). Use a raw binary semaphore when an **ISR** needs to signal that the DMA transfer completed and a task should process the result.

## Semaphore Patterns in C/C++

### Pattern 1 — POSIX Binary Semaphore for SPI Bus Lock

This pattern suits Linux-based embedded systems (e.g., Raspberry Pi, BeagleBone) where multiple POSIX threads share a `/dev/spidev` handle.

```c
/* spi_bus.h */
#ifndef SPI_BUS_H
#define SPI_BUS_H

#include <semaphore.h>
#include <stdint.h>
#include <stddef.h>

typedef struct {
    int       fd;          /* /dev/spidevX.Y file descriptor */
    sem_t     sem;         /* binary semaphore (value 0 or 1) */
} SpiBus;

int  spi_bus_init(SpiBus *bus, const char *device, uint32_t speed_hz);
int  spi_bus_transfer(SpiBus *bus, const uint8_t *tx, uint8_t *rx, size_t len);
void spi_bus_destroy(SpiBus *bus);

#endif /* SPI_BUS_H */
```

```c
/* spi_bus.c */
#include "spi_bus.h"
#include <fcntl.h>
#include <unistd.h>
#include <string.h>
#include <errno.h>
#include <linux/spi/spidev.h>
#include <sys/ioctl.h>
#include <stdio.h>

int spi_bus_init(SpiBus *bus, const char *device, uint32_t speed_hz) {
    bus->fd = open(device, O_RDWR);
    if (bus->fd < 0) {
        perror("open spidev");
        return -1;
    }

    uint8_t mode = SPI_MODE_0;
    ioctl(bus->fd, SPI_IOC_WR_MODE, &mode);
    ioctl(bus->fd, SPI_IOC_WR_MAX_SPEED_HZ, &speed_hz);

    /*
     * sem_init: pshared=0 (thread-local), value=1 (bus free).
     * A value of 1 means the resource is immediately available.
     */
    if (sem_init(&bus->sem, 0, 1) != 0) {
        perror("sem_init");
        close(bus->fd);
        return -1;
    }
    return 0;
}

int spi_bus_transfer(SpiBus *bus, const uint8_t *tx, uint8_t *rx, size_t len) {
    /* ---- ACQUIRE ---- Block until bus is free */
    while (sem_wait(&bus->sem) != 0) {
        if (errno == EINTR) continue;   /* retry on signal interrupt */
        perror("sem_wait");
        return -1;
    }

    /* ---- CRITICAL SECTION: SPI transaction ---- */
    struct spi_ioc_transfer xfer = {
        .tx_buf        = (unsigned long)tx,
        .rx_buf        = (unsigned long)rx,
        .len           = (uint32_t)len,
        .speed_hz      = 0,     /* use default */
        .bits_per_word = 8,
        .cs_change     = 0,
    };

    int ret = ioctl(bus->fd, SPI_IOC_MESSAGE(1), &xfer);
    if (ret < 0) {
        perror("SPI_IOC_MESSAGE");
    }

    /* ---- RELEASE ---- Always release, even on error */
    sem_post(&bus->sem);

    return (ret < 0) ? -1 : 0;
}

void spi_bus_destroy(SpiBus *bus) {
    sem_destroy(&bus->sem);
    close(bus->fd);
}
```

```c
/* Usage example: two threads sharing one SPI bus */
#include "spi_bus.h"
#include <pthread.h>
#include <stdio.h>

static SpiBus g_spi;

void *sensor_task(void *arg) {
    uint8_t tx[] = { 0x0F };   /* WHO_AM_I register */
    uint8_t rx[1];
    for (int i = 0; i < 10; i++) {
        spi_bus_transfer(&g_spi, tx, rx, 1);
        printf("[Sensor] id=0x%02X\n", rx[0]);
    }
    return NULL;
}

void *display_task(void *arg) {
    uint8_t frame[4] = { 0x2C, 0xFF, 0x00, 0xFF }; /* display command + color */
    for (int i = 0; i < 10; i++) {
        spi_bus_transfer(&g_spi, frame, NULL, sizeof(frame));
        printf("[Display] frame sent\n");
    }
    return NULL;
}

int main(void) {
    spi_bus_init(&g_spi, "/dev/spidev0.0", 1000000);

    pthread_t t1, t2;
    pthread_create(&t1, NULL, sensor_task, NULL);
    pthread_create(&t2, NULL, display_task, NULL);
    pthread_join(t1, NULL);
    pthread_join(t2, NULL);

    spi_bus_destroy(&g_spi);
    return 0;
}
```

### Pattern 2 — Counting Semaphore for Pooled SPI Buses

When a microcontroller has multiple independent SPI peripherals (SPI1, SPI2, SPI3) and tasks should grab any free one:

```c
#include <semaphore.h>
#include <stdbool.h>
#include <pthread.h>
#include <stdio.h>

#define SPI_POOL_SIZE 3

typedef struct {
    int      fd;
    bool     in_use;
} SpiHandle;

static SpiHandle   spi_pool[SPI_POOL_SIZE];
static sem_t       spi_pool_sem;   /* counting semaphore, max = SPI_POOL_SIZE */
static pthread_mutex_t pool_lock;  /* protects the in_use flags */

void spi_pool_init(void) {
    /*
     * Initialize counting semaphore to SPI_POOL_SIZE.
     * Each wait() consumes one "permit"; signal() returns a permit.
     */
    sem_init(&spi_pool_sem, 0, SPI_POOL_SIZE);
    pthread_mutex_init(&pool_lock, NULL);

    for (int i = 0; i < SPI_POOL_SIZE; i++) {
        spi_pool[i].fd     = i + 10;  /* placeholder FDs */
        spi_pool[i].in_use = false;
    }
}

/* Acquire one SPI bus from the pool; returns handle index */
int spi_pool_acquire(void) {
    sem_wait(&spi_pool_sem);        /* blocks when all buses are busy */

    pthread_mutex_lock(&pool_lock);
    int idx = -1;
    for (int i = 0; i < SPI_POOL_SIZE; i++) {
        if (!spi_pool[i].in_use) {
            spi_pool[i].in_use = true;
            idx = i;
            break;
        }
    }
    pthread_mutex_unlock(&pool_lock);
    return idx; /* guaranteed to find one after sem_wait succeeded */
}

/* Release a SPI bus back to the pool */
void spi_pool_release(int idx) {
    pthread_mutex_lock(&pool_lock);
    spi_pool[idx].in_use = false;
    pthread_mutex_unlock(&pool_lock);

    sem_post(&spi_pool_sem);        /* returns a permit to the semaphore */
}

/* Worker task */
void *worker(void *arg) {
    long id = (long)arg;
    int bus = spi_pool_acquire();
    printf("Task %ld using SPI bus %d\n", id, bus);
    /* ... perform SPI transfer on spi_pool[bus].fd ... */
    spi_pool_release(bus);
    return NULL;
}

int main(void) {
    spi_pool_init();

    /* Launch 6 tasks competing for 3 buses */
    pthread_t threads[6];
    for (long i = 0; i < 6; i++)
        pthread_create(&threads[i], NULL, worker, (void *)i);
    for (int i = 0; i < 6; i++)
        pthread_join(threads[i], NULL);
    return 0;
}
```

### Pattern 3 — FreeRTOS SPI Access Control

FreeRTOS is the most widely used RTOS in embedded systems. It provides both binary and counting semaphores natively.

```c
/* freertos_spi.c — FreeRTOS SPI bus semaphore pattern */
#include "FreeRTOS.h"
#include "task.h"
#include "semphr.h"

/* ------------------------------------------------------------------ */
/* SPI HAL stubs (replace with your MCU's HAL, e.g., STM32 HAL_SPI)  */
/* ------------------------------------------------------------------ */
typedef struct { uint32_t instance; } SPI_HandleTypeDef;

static HAL_StatusTypeDef HAL_SPI_TransmitReceive(
    SPI_HandleTypeDef *hspi, uint8_t *tx, uint8_t *rx,
    uint16_t len, uint32_t timeout)
{
    (void)hspi; (void)tx; (void)rx; (void)len; (void)timeout;
    return 0; /* HAL_OK */
}

/* ------------------------------------------------------------------ */
/* SPI bus manager                                                     */
/* ------------------------------------------------------------------ */
#define SPI_TIMEOUT_MS   100

static SPI_HandleTypeDef hspi1;
static SemaphoreHandle_t xSpiMutex;   /* Binary semaphore used as mutex */

void SPI_BusInit(void) {
    /*
     * xSemaphoreCreateMutex creates a binary semaphore with priority
     * inheritance — the preferred choice for resource protection.
     * Use xSemaphoreCreateBinary() when an ISR must give the semaphore.
     */
    xSpiMutex = xSemaphoreCreateMutex();
    configASSERT(xSpiMutex != NULL);
}

BaseType_t SPI_Transfer(uint8_t *tx, uint8_t *rx, uint16_t len) {
    /* ACQUIRE — block up to SPI_TIMEOUT_MS ticks */
    if (xSemaphoreTake(xSpiMutex, pdMS_TO_TICKS(SPI_TIMEOUT_MS)) != pdTRUE) {
        /* Timeout: another task is holding the bus too long */
        return pdFAIL;
    }

    /* CRITICAL SECTION */
    BaseType_t status = (HAL_SPI_TransmitReceive(&hspi1, tx, rx, len, SPI_TIMEOUT_MS) == 0)
                        ? pdPASS : pdFAIL;

    /* RELEASE */
    xSemaphoreGive(xSpiMutex);
    return status;
}

/* ------------------------------------------------------------------ */
/* ISR-driven DMA completion: use a BINARY semaphore (not mutex)      */
/* ------------------------------------------------------------------ */
static SemaphoreHandle_t xDmaCompleteSem;

void SPI_DMA_Init(void) {
    /* Binary semaphore: given by ISR, taken by task */
    xDmaCompleteSem = xSemaphoreCreateBinary();
    configASSERT(xDmaCompleteSem != NULL);
}

/* Called from DMA transfer-complete ISR */
void HAL_SPI_TxRxCpltCallback(SPI_HandleTypeDef *hspi) {
    BaseType_t xHigherPriorityTaskWoken = pdFALSE;
    /* Give semaphore FROM ISR — unblocks the waiting task */
    xSemaphoreGiveFromISR(xDmaCompleteSem, &xHigherPriorityTaskWoken);
    portYIELD_FROM_ISR(xHigherPriorityTaskWoken);
}

/* Task waits for DMA completion */
void vSpiDmaTask(void *pvParams) {
    for (;;) {
        /* Start async DMA transfer here ... */

        /* Block until ISR signals completion */
        if (xSemaphoreTake(xDmaCompleteSem, pdMS_TO_TICKS(500)) == pdTRUE) {
            /* Process received data */
        } else {
            /* DMA timeout — handle error */
        }
    }
}

/* ------------------------------------------------------------------ */
/* Counting semaphore: track N outstanding SPI requests               */
/* ------------------------------------------------------------------ */
#define MAX_OUTSTANDING_SPI_OPS  4

static SemaphoreHandle_t xSpiCountSem;

void SPI_CountingSemInit(void) {
    /*
     * xSemaphoreCreateCounting(maxCount, initialCount)
     * initialCount = MAX means all slots available immediately.
     */
    xSpiCountSem = xSemaphoreCreateCounting(MAX_OUTSTANDING_SPI_OPS,
                                             MAX_OUTSTANDING_SPI_OPS);
    configASSERT(xSpiCountSem != NULL);
}

BaseType_t SPI_AcquireSlot(void) {
    return xSemaphoreTake(xSpiCountSem, pdMS_TO_TICKS(SPI_TIMEOUT_MS));
}

void SPI_ReleaseSlot(void) {
    xSemaphoreGive(xSpiCountSem);
}
```

### Pattern 4 — C++ RAII Wrapper

A RAII (Resource Acquisition Is Initialization) guard ensures the semaphore is always released, even if an exception is thrown.

```cpp
// SpiBusGuard.hpp — C++17 RAII semaphore guard for SPI
#pragma once
#include <semaphore.h>
#include <stdexcept>
#include <cstring>
#include <cerrno>

class SpiBusSemaphore {
public:
    explicit SpiBusSemaphore(unsigned int initial_value = 1) {
        if (sem_init(&sem_, 0, initial_value) != 0) {
            throw std::runtime_error(std::strerror(errno));
        }
    }

    ~SpiBusSemaphore() { sem_destroy(&sem_); }

    void acquire() {
        while (sem_wait(&sem_) != 0) {
            if (errno != EINTR) throw std::runtime_error(std::strerror(errno));
        }
    }

    void release() { sem_post(&sem_); }

    // Non-copyable, non-movable
    SpiBusSemaphore(const SpiBusSemaphore&)            = delete;
    SpiBusSemaphore& operator=(const SpiBusSemaphore&) = delete;

private:
    sem_t sem_;
};

// RAII guard: acquires on construction, releases on destruction
class SpiBusGuard {
public:
    explicit SpiBusGuard(SpiBusSemaphore &sem) : sem_(sem) {
        sem_.acquire();
    }
    ~SpiBusGuard() { sem_.release(); }

    SpiBusGuard(const SpiBusGuard&)            = delete;
    SpiBusGuard& operator=(const SpiBusGuard&) = delete;

private:
    SpiBusSemaphore &sem_;
};
```

```cpp
// Usage
#include "SpiBusGuard.hpp"
#include <cstdio>
#include <thread>
#include <vector>

class SpiDevice {
public:
    explicit SpiDevice(SpiBusSemaphore &bus_sem, int cs_pin)
        : bus_sem_(bus_sem), cs_pin_(cs_pin) {}

    void readRegister(uint8_t reg, uint8_t *out) {
        SpiBusGuard lock(bus_sem_);  // automatically acquires & releases
        // CS low, transfer, CS high
        (void)reg; (void)out;
        std::printf("  [CS%d] SPI transfer reg=0x%02X\n", cs_pin_, reg);
        // If an exception is thrown here, lock's destructor still releases
    }

private:
    SpiBusSemaphore &bus_sem_;
    int              cs_pin_;
};

int main() {
    SpiBusSemaphore bus_sem(1); // binary semaphore

    SpiDevice sensor(bus_sem, 0);
    SpiDevice display(bus_sem, 1);

    auto sensor_thread  = std::thread([&]{ for (int i=0;i<5;i++) sensor.readRegister(0x0F, nullptr); });
    auto display_thread = std::thread([&]{ for (int i=0;i<5;i++) display.readRegister(0x28, nullptr); });

    sensor_thread.join();
    display_thread.join();
    return 0;
}
```

## Semaphore Patterns in Rust

Rust's ownership model prevents many of the pitfalls of C semaphore misuse (double-release, forgetting to release). The type system enforces correct usage at compile time.

### Pattern 1 — `std::sync` Semaphore Emulation (no-std compatible approach)

Rust's standard library does not ship a `Semaphore` type (it was stabilized then removed). The idiomatic approach uses a `Mutex<u32>` + `Condvar`:

```rust
// spi_semaphore.rs — Binary and counting semaphores using Mutex + Condvar
use std::sync::{Arc, Condvar, Mutex};

/// A general counting semaphore (works as binary when max_count = 1).
pub struct Semaphore {
    state:   Mutex<u32>,
    condvar: Condvar,
    max:     u32,
}

impl Semaphore {
    /// Create a new semaphore with `initial` permits available.
    pub fn new(initial: u32, max: u32) -> Arc<Self> {
        Arc::new(Self {
            state:   Mutex::new(initial),
            condvar: Condvar::new(),
            max,
        })
    }

    /// Acquire one permit (blocks if count == 0).
    pub fn acquire(&self) {
        let mut count = self.state.lock().unwrap();
        while *count == 0 {
            count = self.condvar.wait(count).unwrap();
        }
        *count -= 1;
    }

    /// Release one permit (panics if count == max — over-release guard).
    pub fn release(&self) {
        let mut count = self.state.lock().unwrap();
        assert!(*count < self.max, "Semaphore over-released");
        *count += 1;
        self.condvar.notify_one();
    }
}

/// RAII guard — releases the semaphore when dropped.
pub struct SemaphoreGuard<'a>(&'a Semaphore);

impl<'a> SemaphoreGuard<'a> {
    pub fn acquire(sem: &'a Semaphore) -> Self {
        sem.acquire();
        SemaphoreGuard(sem)
    }
}

impl Drop for SemaphoreGuard<'_> {
    fn drop(&mut self) {
        self.0.release();
    }
}
```

```rust
// spi_bus.rs — SPI bus protected by binary semaphore
use crate::spi_semaphore::{Semaphore, SemaphoreGuard};
use std::sync::Arc;

pub struct SpiBus {
    device:    String,
    semaphore: Arc<Semaphore>,
}

impl SpiBus {
    pub fn new(device: &str) -> Self {
        SpiBus {
            device:    device.to_owned(),
            semaphore: Semaphore::new(1, 1), // binary
        }
    }

    /// Transfer bytes over SPI.  Returns received bytes.
    pub fn transfer(&self, tx: &[u8]) -> Vec<u8> {
        // Acquiring the guard blocks until the bus is free.
        // The guard is dropped (and bus released) at the end of this scope.
        let _guard = SemaphoreGuard::acquire(&self.semaphore);

        // ---- CRITICAL SECTION ----
        println!("[{}] Transferring {} bytes: {:02X?}", self.device, tx.len(), tx);
        // In a real driver: write to /dev/spidev, call ioctl, etc.
        vec![0u8; tx.len()] // placeholder rx
    }
}

// Usage
fn main() {
    use std::{sync::Arc, thread};

    let bus = Arc::new(SpiBus::new("/dev/spidev0.0"));

    let handles: Vec<_> = (0..4).map(|i| {
        let b = Arc::clone(&bus);
        thread::spawn(move || {
            let tx = vec![0x0F_u8 + i as u8];
            let rx = b.transfer(&tx);
            println!("Thread {} rx={:02X?}", i, rx);
        })
    }).collect();

    for h in handles { h.join().unwrap(); }
}
```

### Pattern 2 — Tokio Async Semaphore for SPI

For async Rust applications (e.g., running on Linux with `tokio`), the `tokio::sync::Semaphore` provides non-blocking permit acquisition.

```rust
// async_spi.rs — Tokio async semaphore for SPI bus control
use std::sync::Arc;
use tokio::sync::{Semaphore, SemaphorePermit};

/// Async SPI bus — safe to clone and share across async tasks.
#[derive(Clone)]
pub struct AsyncSpiBus {
    semaphore: Arc<Semaphore>,
    device:    String,
}

impl AsyncSpiBus {
    /// Creates a new bus handle.
    /// `max_concurrent` = 1 for a single bus (binary semaphore behavior).
    pub fn new(device: &str, max_concurrent: usize) -> Self {
        AsyncSpiBus {
            semaphore: Arc::new(Semaphore::new(max_concurrent)),
            device:    device.to_owned(),
        }
    }

    /// Acquire bus access asynchronously (yields to executor while waiting).
    pub async fn acquire(&self) -> SemaphorePermit<'_> {
        self.semaphore
            .acquire()
            .await
            .expect("semaphore closed")
        // The returned SemaphorePermit is the RAII guard —
        // dropping it automatically releases the permit.
    }

    /// Perform a full SPI transaction: acquire → transfer → release.
    pub async fn transfer(&self, tx: Vec<u8>) -> Vec<u8> {
        let _permit = self.acquire().await; // blocks async until bus free

        // ---- CRITICAL SECTION (async) ----
        println!("[{}] async transfer {:02X?}", self.device, tx);

        // Simulate I/O delay (replace with actual async spidev write)
        tokio::time::sleep(tokio::time::Duration::from_millis(10)).await;

        vec![0u8; tx.len()] // placeholder
        // _permit dropped here → bus released
    }
}

/// Counting semaphore: pool of 3 SPI buses
pub struct SpiPool {
    semaphore: Arc<Semaphore>,
    pool_size: usize,
}

impl SpiPool {
    pub fn new(pool_size: usize) -> Self {
        SpiPool {
            semaphore: Arc::new(Semaphore::new(pool_size)),
            pool_size,
        }
    }

    pub async fn acquire_slot(&self) -> SemaphorePermit<'_> {
        self.semaphore.acquire().await.expect("semaphore closed")
    }

    pub fn available_slots(&self) -> usize {
        self.semaphore.available_permits()
    }
}

#[tokio::main]
async fn main() {
    let bus = AsyncSpiBus::new("/dev/spidev0.0", 1);

    // 8 concurrent tasks compete for 1 bus
    let handles: Vec<_> = (0u8..8).map(|i| {
        let b = bus.clone();
        tokio::spawn(async move {
            let rx = b.transfer(vec![0x80 | i]).await;
            println!("Task {} -> {:02X?}", i, rx);
        })
    }).collect();

    for h in handles { h.await.unwrap(); }
}
```

### Pattern 3 — Embassy Async Embedded SPI (no-std)

For bare-metal Rust on microcontrollers, the `embassy` framework provides async SPI with built-in mutex protection via `embassy_sync::mutex::Mutex`.

```rust
// embassy_spi.rs — embassy async SPI with mutex (no-std)
// Cargo.toml dependencies:
//   embassy-sync = { version = "0.6", features = ["defmt"] }
//   embassy-stm32 = { version = "0.1", features = ["stm32f4", "time-driver-any"] }
//   embassy-executor = { version = "0.5", features = ["arch-cortex-m"] }

#![no_std]
#![no_main]

use embassy_executor::Spawner;
use embassy_stm32::spi::{Config, Spi};
use embassy_stm32::time::Hertz;
use embassy_sync::blocking_mutex::raw::CriticalSectionRawMutex;
use embassy_sync::mutex::Mutex;

// Shared SPI bus protected by a critical-section mutex.
// Multiple tasks hold a reference to this static mutex.
static SPI_BUS: Mutex<CriticalSectionRawMutex, Option<Spi<'static, embassy_stm32::peripherals::SPI1, embassy_stm32::dma::NoDma, embassy_stm32::dma::NoDma>>> =
    Mutex::new(None);

#[embassy_executor::task]
async fn sensor_task() {
    loop {
        let mut bus = SPI_BUS.lock().await; // async acquire
        if let Some(spi) = bus.as_mut() {
            let mut buf = [0x8F_u8];  // read WHO_AM_I
            spi.transfer_in_place(&mut buf).await.ok();
            defmt::info!("Sensor WHO_AM_I = {:#04x}", buf[0]);
        }
        // Mutex guard dropped here — bus released
        embassy_time::Timer::after_millis(100).await;
    }
}

#[embassy_executor::task]
async fn display_task() {
    loop {
        let mut bus = SPI_BUS.lock().await; // waits if sensor_task has it
        if let Some(spi) = bus.as_mut() {
            let pixels = [0xFF_u8, 0x00, 0xFF]; // R,G,B pixel
            spi.write(&pixels).await.ok();
            defmt::info!("Display frame written");
        }
        embassy_time::Timer::after_millis(33).await; // ~30fps
    }
}

#[embassy_executor::main]
async fn main(spawner: Spawner) {
    let p = embassy_stm32::init(Default::default());

    // Initialize SPI peripheral
    let spi = Spi::new(
        p.SPI1,
        p.PB3,  // SCK
        p.PB5,  // MOSI
        p.PB4,  // MISO
        embassy_stm32::dma::NoDma,
        embassy_stm32::dma::NoDma,
        Config::default(),
    );

    // Seed the shared bus
    *SPI_BUS.lock().await = Some(spi);

    spawner.spawn(sensor_task()).unwrap();
    spawner.spawn(display_task()).unwrap();
}
```

## Advanced Patterns

### Priority Inversion Mitigation

Priority inversion occurs when a high-priority task waits for a semaphore held by a low-priority task, and a medium-priority task preempts the low-priority task, indirectly blocking the high-priority task.

**Solution:** Use a **priority-inheriting mutex** (available in POSIX with `PTHREAD_PRIO_INHERIT` and in FreeRTOS with `xSemaphoreCreateMutex`).

```c
/* POSIX priority-inheriting mutex for SPI */
pthread_mutex_t spi_mutex;

void spi_mutex_init_pi(void) {
    pthread_mutexattr_t attr;
    pthread_mutexattr_init(&attr);
    /* Enable priority inheritance — low-priority holder temporarily
     * elevated to the priority of the highest blocked waiter.    */
    pthread_mutexattr_setprotocol(&attr, PTHREAD_PRIO_INHERIT);
    pthread_mutex_init(&spi_mutex, &attr);
    pthread_mutexattr_destroy(&attr);
}

void spi_transfer_pi(uint8_t *buf, size_t len) {
    pthread_mutex_lock(&spi_mutex);
    /* ... SPI transfer ... */
    pthread_mutex_unlock(&spi_mutex);
}
```

### Timeout-Aware Semaphore Acquisition

Blocking indefinitely on a semaphore can cause a watchdog reset or system hang if the SPI bus gets stuck.

```c
/* POSIX timed wait */
#include <time.h>

int spi_transfer_timeout(SpiBus *bus, uint8_t *tx, uint8_t *rx,
                          size_t len, unsigned timeout_ms) {
    struct timespec ts;
    clock_gettime(CLOCK_REALTIME, &ts);
    ts.tv_sec  += timeout_ms / 1000;
    ts.tv_nsec += (timeout_ms % 1000) * 1000000L;
    if (ts.tv_nsec >= 1000000000L) {
        ts.tv_sec++;
        ts.tv_nsec -= 1000000000L;
    }

    if (sem_timedwait(&bus->sem, &ts) != 0) {
        if (errno == ETIMEDOUT) {
            fprintf(stderr, "SPI bus acquisition timed out after %ums\n",
                    timeout_ms);
        }
        return -1;
    }

    /* ... SPI transfer ... */

    sem_post(&bus->sem);
    return 0;
}
```

```rust
// Rust tokio: timeout on semaphore acquisition
use tokio::time::{timeout, Duration};

pub async fn transfer_with_timeout(
    bus: &AsyncSpiBus,
    tx: Vec<u8>,
    deadline: Duration,
) -> Result<Vec<u8>, &'static str> {
    match timeout(deadline, bus.acquire()).await {
        Ok(_permit) => {
            // bus acquired within deadline
            Ok(vec![0u8; tx.len()])
        }
        Err(_) => Err("SPI bus acquisition timed out"),
    }
}
```

### Multi-Bus Semaphore Hierarchies

When a transaction must lock **multiple buses** (e.g., SPI + I2C bridge), always acquire semaphores in a **fixed global order** to prevent deadlock.

```c
/*
 * Global acquisition order (must be consistent across ALL code):
 *   1. SPI bus semaphore
 *   2. I2C bus semaphore
 * Never acquire in reverse order — that causes deadlock.
 */
void bridge_transfer(SpiBus *spi, I2CBus *i2c, ...) {
    sem_wait(&spi->sem);   /* always first */
    sem_wait(&i2c->sem);   /* always second */

    /* ... cross-bus transaction ... */

    sem_post(&i2c->sem);   /* release in reverse order (good practice) */
    sem_post(&spi->sem);
}
```

## Summary

| Topic | Key Takeaway |
|---|---|
| **Binary Semaphore** | Value 0/1; protects the SPI bus as a mutually exclusive resource; any task can give it (unlike a mutex) |
| **Counting Semaphore** | Value 0..N; models a pool of N SPI buses; blocks when all are in use |
| **Mutex vs. Semaphore** | Prefer mutex (with ownership) for SPI task-level locking; use raw binary semaphore for ISR → task signalling |
| **POSIX C** | `sem_init`, `sem_wait`, `sem_post`, `sem_timedwait`; use `PTHREAD_PRIO_INHERIT` to prevent priority inversion |
| **FreeRTOS C** | `xSemaphoreCreateMutex` / `xSemaphoreCreateBinary` / `xSemaphoreCreateCounting`; `xSemaphoreGiveFromISR` for DMA callbacks |
| **C++ RAII** | `SpiBusGuard` destructor guarantees release on scope exit, even when exceptions are thrown |
| **Rust std** | `Mutex<u32>` + `Condvar` implements a counting semaphore; RAII `SemaphoreGuard` via `Drop` enforces release |
| **Rust tokio** | `tokio::sync::Semaphore` with `SemaphorePermit` RAII; non-blocking async `.acquire().await` |
| **Rust Embassy** | `embassy_sync::mutex::Mutex` guards async SPI on bare-metal; cooperative async scheduling = no priority issues |
| **Priority Inversion** | Use priority-inheriting mutexes; plain binary semaphores don't protect against inversion |
| **Timeouts** | Always use `sem_timedwait` or `xSemaphoreTake(..., timeout)` to avoid infinite blocking |
| **Deadlock prevention** | Acquire multiple bus semaphores in a fixed global order; never vary the order across code paths |

### Design Checklist

- [ ] Every SPI transaction is bracketed by `acquire` / `release` pairs
- [ ] Semaphore is **always** released in error paths (use RAII or `goto cleanup` in C)
- [ ] DMA completion uses a **binary semaphore given from ISR**, not a mutex
- [ ] Timeouts are set on all acquisitions to prevent watchdog resets
- [ ] Priority-inheriting mutex is used when task priorities differ significantly
- [ ] Multiple-bus acquisitions follow a fixed global ordering
