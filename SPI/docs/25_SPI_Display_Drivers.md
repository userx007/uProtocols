# SPI Display Drivers: Driving LCD and OLED Displays using SPI Communication

## Overview

SPI (Serial Peripheral Interface) display drivers enable microcontrollers to communicate with LCD and OLED displays using a high-speed synchronous serial protocol. SPI is favored for display applications because it requires fewer pins than parallel interfaces, supports high data transfer rates (often 10+ MHz), and provides reliable communication for updating display content.

## Understanding SPI Communication

SPI is a master-slave protocol using four main signals:

- **MOSI (Master Out Slave In)**: Data line from controller to display
- **MISO (Master In Slave Out)**: Data line from display to controller (rarely used for displays)
- **SCLK (Serial Clock)**: Clock signal generated by master
- **CS/SS (Chip Select)**: Selects which slave device is active

For displays, additional control signals are typically needed:
- **DC (Data/Command)**: Distinguishes between command and data bytes
- **RST (Reset)**: Hardware reset for the display
- **BL/LED (Backlight)**: Controls display illumination (for LCDs)

## Common SPI Display Types

### ST7735 (1.8" TFT LCD)
- Resolution: 128x160 pixels
- Color depth: 16-bit (RGB565)
- Interface: 4-wire SPI
- Typical use: Small color displays for embedded projects

### SSD1306 (OLED)
- Resolution: 128x64 or 128x32 pixels
- Monochrome (white/blue/yellow variants)
- Interface: SPI or I2C
- Low power consumption

### ILI9341 (2.4" TFT LCD)
- Resolution: 240x320 pixels
- 16-bit color
- Popular for Arduino and ESP32 projects

## Code Examples

### C/C++ Example: ST7735 Display Driver

```c
#include <stdint.h>
#include <stdbool.h>

// Hardware abstraction layer functions (platform-specific)
extern void spi_init(uint32_t frequency);
extern void spi_transfer(uint8_t data);
extern void gpio_set_output(uint8_t pin);
extern void gpio_write(uint8_t pin, bool state);
extern void delay_ms(uint32_t ms);
extern void delay_us(uint32_t us);

// Pin definitions
#define ST7735_CS_PIN    10
#define ST7735_DC_PIN    9
#define ST7735_RST_PIN   8

// ST7735 Commands
#define ST7735_NOP       0x00
#define ST7735_SWRESET   0x01
#define ST7735_SLPOUT    0x11
#define ST7735_DISPOFF   0x28
#define ST7735_DISPON    0x29
#define ST7735_CASET     0x2A
#define ST7735_RASET     0x2B
#define ST7735_RAMWR     0x2C
#define ST7735_COLMOD    0x3A
#define ST7735_MADCTL    0x36

// Display dimensions
#define ST7735_WIDTH     128
#define ST7735_HEIGHT    160

// Color definitions (RGB565)
#define COLOR_BLACK      0x0000
#define COLOR_WHITE      0xFFFF
#define COLOR_RED        0xF800
#define COLOR_GREEN      0x07E0
#define COLOR_BLUE       0x001F

// Helper functions
static inline void cs_low(void) {
    gpio_write(ST7735_CS_PIN, false);
}

static inline void cs_high(void) {
    gpio_write(ST7735_CS_PIN, true);
}

static inline void dc_command(void) {
    gpio_write(ST7735_DC_PIN, false);
}

static inline void dc_data(void) {
    gpio_write(ST7735_DC_PIN, true);
}

// Send command to display
void st7735_write_command(uint8_t cmd) {
    dc_command();
    cs_low();
    spi_transfer(cmd);
    cs_high();
}

// Send data to display
void st7735_write_data(uint8_t data) {
    dc_data();
    cs_low();
    spi_transfer(data);
    cs_high();
}

// Send multiple data bytes
void st7735_write_data_buffer(const uint8_t *buffer, uint32_t length) {
    dc_data();
    cs_low();
    for (uint32_t i = 0; i < length; i++) {
        spi_transfer(buffer[i]);
    }
    cs_high();
}

// Hardware reset
void st7735_reset(void) {
    gpio_write(ST7735_RST_PIN, true);
    delay_ms(5);
    gpio_write(ST7735_RST_PIN, false);
    delay_ms(20);
    gpio_write(ST7735_RST_PIN, true);
    delay_ms(150);
}

// Initialize display
void st7735_init(void) {
    // Configure GPIO pins
    gpio_set_output(ST7735_CS_PIN);
    gpio_set_output(ST7735_DC_PIN);
    gpio_set_output(ST7735_RST_PIN);
    
    cs_high();
    
    // Initialize SPI at 8 MHz
    spi_init(8000000);
    
    // Hardware reset
    st7735_reset();
    
    // Software reset
    st7735_write_command(ST7735_SWRESET);
    delay_ms(150);
    
    // Sleep out
    st7735_write_command(ST7735_SLPOUT);
    delay_ms(500);
    
    // Color mode: 16-bit color
    st7735_write_command(ST7735_COLMOD);
    st7735_write_data(0x05);  // 16-bit/pixel
    
    // Memory access control (rotation, RGB/BGR)
    st7735_write_command(ST7735_MADCTL);
    st7735_write_data(0x00);  // Normal orientation
    
    // Display on
    st7735_write_command(ST7735_DISPON);
    delay_ms(100);
}

// Set address window for drawing
void st7735_set_addr_window(uint16_t x0, uint16_t y0, 
                           uint16_t x1, uint16_t y1) {
    // Column address set
    st7735_write_command(ST7735_CASET);
    st7735_write_data(x0 >> 8);
    st7735_write_data(x0 & 0xFF);
    st7735_write_data(x1 >> 8);
    st7735_write_data(x1 & 0xFF);
    
    // Row address set
    st7735_write_command(ST7735_RASET);
    st7735_write_data(y0 >> 8);
    st7735_write_data(y0 & 0xFF);
    st7735_write_data(y1 >> 8);
    st7735_write_data(y1 & 0xFF);
    
    // Write to RAM
    st7735_write_command(ST7735_RAMWR);
}

// Draw a pixel
void st7735_draw_pixel(uint16_t x, uint16_t y, uint16_t color) {
    if (x >= ST7735_WIDTH || y >= ST7735_HEIGHT) return;
    
    st7735_set_addr_window(x, y, x, y);
    
    dc_data();
    cs_low();
    spi_transfer(color >> 8);
    spi_transfer(color & 0xFF);
    cs_high();
}

// Fill rectangle
void st7735_fill_rect(uint16_t x, uint16_t y, uint16_t w, 
                      uint16_t h, uint16_t color) {
    if (x >= ST7735_WIDTH || y >= ST7735_HEIGHT) return;
    if (x + w > ST7735_WIDTH) w = ST7735_WIDTH - x;
    if (y + h > ST7735_HEIGHT) h = ST7735_HEIGHT - y;
    
    st7735_set_addr_window(x, y, x + w - 1, y + h - 1);
    
    uint8_t color_hi = color >> 8;
    uint8_t color_lo = color & 0xFF;
    
    dc_data();
    cs_low();
    for (uint32_t i = 0; i < (uint32_t)w * h; i++) {
        spi_transfer(color_hi);
        spi_transfer(color_lo);
    }
    cs_high();
}

// Clear screen
void st7735_clear(uint16_t color) {
    st7735_fill_rect(0, 0, ST7735_WIDTH, ST7735_HEIGHT, color);
}

// Example usage
int main(void) {
    st7735_init();
    
    // Clear screen to black
    st7735_clear(COLOR_BLACK);
    
    // Draw some rectangles
    st7735_fill_rect(10, 10, 50, 30, COLOR_RED);
    st7735_fill_rect(70, 10, 50, 30, COLOR_GREEN);
    st7735_fill_rect(10, 50, 50, 30, COLOR_BLUE);
    st7735_fill_rect(70, 50, 50, 30, COLOR_WHITE);
    
    while (1) {
        // Main loop
    }
    
    return 0;
}
```

### C++ Example: SSD1306 OLED Driver (Object-Oriented)

```cpp
#include <cstdint>
#include <cstring>

class SSD1306 {
private:
    static constexpr uint8_t WIDTH = 128;
    static constexpr uint8_t HEIGHT = 64;
    static constexpr uint16_t BUFFER_SIZE = (WIDTH * HEIGHT) / 8;
    
    // SSD1306 Commands
    static constexpr uint8_t CMD_DISPLAY_OFF = 0xAE;
    static constexpr uint8_t CMD_DISPLAY_ON = 0xAF;
    static constexpr uint8_t CMD_SET_CONTRAST = 0x81;
    static constexpr uint8_t CMD_SET_MEMORY_MODE = 0x20;
    static constexpr uint8_t CMD_SET_COLUMN_ADDR = 0x21;
    static constexpr uint8_t CMD_SET_PAGE_ADDR = 0x22;
    
    uint8_t cs_pin;
    uint8_t dc_pin;
    uint8_t rst_pin;
    uint8_t framebuffer[BUFFER_SIZE];
    
    // Hardware abstraction
    void spiTransfer(uint8_t data);
    void pinWrite(uint8_t pin, bool state);
    void delayMs(uint32_t ms);
    
    void writeCommand(uint8_t cmd) {
        pinWrite(dc_pin, false);
        pinWrite(cs_pin, false);
        spiTransfer(cmd);
        pinWrite(cs_pin, true);
    }
    
    void writeData(uint8_t data) {
        pinWrite(dc_pin, true);
        pinWrite(cs_pin, false);
        spiTransfer(data);
        pinWrite(cs_pin, true);
    }
    
public:
    SSD1306(uint8_t cs, uint8_t dc, uint8_t rst)
        : cs_pin(cs), dc_pin(dc), rst_pin(rst) {
        memset(framebuffer, 0, BUFFER_SIZE);
    }
    
    void init() {
        // Hardware reset
        pinWrite(rst_pin, true);
        delayMs(1);
        pinWrite(rst_pin, false);
        delayMs(10);
        pinWrite(rst_pin, true);
        delayMs(10);
        
        // Initialization sequence
        writeCommand(CMD_DISPLAY_OFF);
        
        writeCommand(0xD5); // Set display clock divide
        writeCommand(0x80);
        
        writeCommand(0xA8); // Set multiplex ratio
        writeCommand(0x3F);
        
        writeCommand(0xD3); // Set display offset
        writeCommand(0x00);
        
        writeCommand(0x40); // Set start line
        
        writeCommand(0x8D); // Charge pump
        writeCommand(0x14);
        
        writeCommand(CMD_SET_MEMORY_MODE);
        writeCommand(0x00); // Horizontal addressing mode
        
        writeCommand(0xA1); // Segment remap
        writeCommand(0xC8); // COM scan direction
        
        writeCommand(0xDA); // COM pins configuration
        writeCommand(0x12);
        
        writeCommand(CMD_SET_CONTRAST);
        writeCommand(0xCF);
        
        writeCommand(0xD9); // Set precharge period
        writeCommand(0xF1);
        
        writeCommand(0xDB); // VCOMH deselect level
        writeCommand(0x40);
        
        writeCommand(0xA4); // Display all on resume
        writeCommand(0xA6); // Normal display (not inverted)
        
        writeCommand(CMD_DISPLAY_ON);
    }
    
    void setPixel(uint8_t x, uint8_t y, bool on) {
        if (x >= WIDTH || y >= HEIGHT) return;
        
        if (on) {
            framebuffer[x + (y / 8) * WIDTH] |= (1 << (y & 7));
        } else {
            framebuffer[x + (y / 8) * WIDTH] &= ~(1 << (y & 7));
        }
    }
    
    void clear() {
        memset(framebuffer, 0, BUFFER_SIZE);
    }
    
    void display() {
        writeCommand(CMD_SET_COLUMN_ADDR);
        writeCommand(0);
        writeCommand(WIDTH - 1);
        
        writeCommand(CMD_SET_PAGE_ADDR);
        writeCommand(0);
        writeCommand((HEIGHT / 8) - 1);
        
        pinWrite(dc_pin, true);
        pinWrite(cs_pin, false);
        for (uint16_t i = 0; i < BUFFER_SIZE; i++) {
            spiTransfer(framebuffer[i]);
        }
        pinWrite(cs_pin, true);
    }
    
    void drawLine(uint8_t x0, uint8_t y0, uint8_t x1, uint8_t y1) {
        int dx = abs(x1 - x0);
        int dy = abs(y1 - y0);
        int sx = (x0 < x1) ? 1 : -1;
        int sy = (y0 < y1) ? 1 : -1;
        int err = dx - dy;
        
        while (true) {
            setPixel(x0, y0, true);
            
            if (x0 == x1 && y0 == y1) break;
            
            int e2 = 2 * err;
            if (e2 > -dy) {
                err -= dy;
                x0 += sx;
            }
            if (e2 < dx) {
                err += dx;
                y0 += sy;
            }
        }
    }
    
    void drawRect(uint8_t x, uint8_t y, uint8_t w, uint8_t h) {
        drawLine(x, y, x + w - 1, y);
        drawLine(x + w - 1, y, x + w - 1, y + h - 1);
        drawLine(x + w - 1, y + h - 1, x, y + h - 1);
        drawLine(x, y + h - 1, x, y);
    }
};
```

### Rust Example: SPI Display Driver with embedded-hal

```rust
#![no_std]

use embedded_hal::blocking::spi::Write;
use embedded_hal::digital::v2::OutputPin;
use embedded_hal::blocking::delay::DelayMs;

/// ST7735 Display Driver
pub struct ST7735<SPI, CS, DC, RST> {
    spi: SPI,
    cs: CS,
    dc: DC,
    rst: RST,
    width: u16,
    height: u16,
}

// ST7735 Commands
const CMD_SWRESET: u8 = 0x01;
const CMD_SLPOUT: u8 = 0x11;
const CMD_DISPON: u8 = 0x29;
const CMD_CASET: u8 = 0x2A;
const CMD_RASET: u8 = 0x2B;
const CMD_RAMWR: u8 = 0x2C;
const CMD_COLMOD: u8 = 0x3A;
const CMD_MADCTL: u8 = 0x36;

/// RGB565 Color type
#[derive(Copy, Clone)]
pub struct Color(pub u16);

impl Color {
    pub const BLACK: Color = Color(0x0000);
    pub const WHITE: Color = Color(0xFFFF);
    pub const RED: Color = Color(0xF800);
    pub const GREEN: Color = Color(0x07E0);
    pub const BLUE: Color = Color(0x001F);
    
    /// Create color from RGB components (0-255)
    pub fn from_rgb(r: u8, g: u8, b: u8) -> Self {
        let r5 = (r as u16 >> 3) & 0x1F;
        let g6 = (g as u16 >> 2) & 0x3F;
        let b5 = (b as u16 >> 3) & 0x1F;
        Color((r5 << 11) | (g6 << 5) | b5)
    }
}

impl<SPI, CS, DC, RST, E> ST7735<SPI, CS, DC, RST>
where
    SPI: Write<u8, Error = E>,
    CS: OutputPin,
    DC: OutputPin,
    RST: OutputPin,
{
    pub fn new(spi: SPI, cs: CS, dc: DC, rst: RST) -> Self {
        ST7735 {
            spi,
            cs,
            dc,
            rst,
            width: 128,
            height: 160,
        }
    }
    
    fn write_command(&mut self, cmd: u8) -> Result<(), E> {
        self.dc.set_low().ok();
        self.cs.set_low().ok();
        self.spi.write(&[cmd])?;
        self.cs.set_high().ok();
        Ok(())
    }
    
    fn write_data(&mut self, data: &[u8]) -> Result<(), E> {
        self.dc.set_high().ok();
        self.cs.set_low().ok();
        self.spi.write(data)?;
        self.cs.set_high().ok();
        Ok(())
    }
    
    pub fn init<DELAY>(&mut self, delay: &mut DELAY) -> Result<(), E>
    where
        DELAY: DelayMs<u16>,
    {
        // Hardware reset
        self.rst.set_high().ok();
        delay.delay_ms(5);
        self.rst.set_low().ok();
        delay.delay_ms(20);
        self.rst.set_high().ok();
        delay.delay_ms(150);
        
        // Software reset
        self.write_command(CMD_SWRESET)?;
        delay.delay_ms(150);
        
        // Sleep out
        self.write_command(CMD_SLPOUT)?;
        delay.delay_ms(500);
        
        // Color mode: 16-bit
        self.write_command(CMD_COLMOD)?;
        self.write_data(&[0x05])?;
        
        // Memory access control
        self.write_command(CMD_MADCTL)?;
        self.write_data(&[0x00])?;
        
        // Display on
        self.write_command(CMD_DISPON)?;
        delay.delay_ms(100);
        
        Ok(())
    }
    
    pub fn set_addr_window(&mut self, x0: u16, y0: u16, 
                          x1: u16, y1: u16) -> Result<(), E> {
        // Column address
        self.write_command(CMD_CASET)?;
        self.write_data(&[
            (x0 >> 8) as u8,
            (x0 & 0xFF) as u8,
            (x1 >> 8) as u8,
            (x1 & 0xFF) as u8,
        ])?;
        
        // Row address
        self.write_command(CMD_RASET)?;
        self.write_data(&[
            (y0 >> 8) as u8,
            (y0 & 0xFF) as u8,
            (y1 >> 8) as u8,
            (y1 & 0xFF) as u8,
        ])?;
        
        // Write to RAM
        self.write_command(CMD_RAMWR)?;
        
        Ok(())
    }
    
    pub fn draw_pixel(&mut self, x: u16, y: u16, 
                     color: Color) -> Result<(), E> {
        if x >= self.width || y >= self.height {
            return Ok(());
        }
        
        self.set_addr_window(x, y, x, y)?;
        self.write_data(&[
            (color.0 >> 8) as u8,
            (color.0 & 0xFF) as u8,
        ])?;
        
        Ok(())
    }
    
    pub fn fill_rect(&mut self, x: u16, y: u16, w: u16, h: u16, 
                    color: Color) -> Result<(), E> {
        if x >= self.width || y >= self.height {
            return Ok(());
        }
        
        let w = w.min(self.width - x);
        let h = h.min(self.height - y);
        
        self.set_addr_window(x, y, x + w - 1, y + h - 1)?;
        
        let color_bytes = [
            (color.0 >> 8) as u8,
            (color.0 & 0xFF) as u8,
        ];
        
        self.dc.set_high().ok();
        self.cs.set_low().ok();
        
        for _ in 0..(w as u32 * h as u32) {
            self.spi.write(&color_bytes)?;
        }
        
        self.cs.set_high().ok();
        
        Ok(())
    }
    
    pub fn clear(&mut self, color: Color) -> Result<(), E> {
        self.fill_rect(0, 0, self.width, self.height, color)
    }
}

    let peripherals = /* initialize peripherals */;

        // Main loop
    }
}
```

## Key Programming Concepts

### 1. **Command vs Data Mode**
The DC (Data/Command) pin distinguishes between configuration commands and pixel data, requiring careful state management.

### 2. **Address Windowing**
Efficiently update specific screen regions by setting column and row address ranges before writing pixel data.

### 3. **Color Encoding**
RGB565 format packs 16-bit color: 5 bits red, 6 bits green, 5 bits blue, optimizing memory and bandwidth.

### 4. **Double Buffering**
For OLED displays like SSD1306, maintain a framebuffer in RAM and transfer it in one operation to prevent tearing and flicker.

### 5. **SPI Speed Optimization**
Higher SPI frequencies (10-40 MHz) enable smoother animations, but verify display and MCU capabilities.

### 6. **DMA Integration**
Using Direct Memory Access for large data transfers frees the CPU during screen updates.

## Summary

SPI display drivers provide an efficient interface for controlling LCD and OLED displays in embedded systems. The synchronous, high-speed nature of SPI makes it ideal for graphics applications requiring frequent updates. Key considerations include proper initialization sequences, command/data pin management, color format encoding, and optimizing data transfer through techniques like address windowing and DMA.

The examples demonstrate implementation patterns across C, C++, and Rust, showing both low-level control and abstraction through object-oriented and trait-based designs. Modern embedded frameworks like `embedded-hal` in Rust enable portable driver development across different microcontroller platforms while maintaining type safety and zero-cost abstractions.

Understanding SPI display drivers is foundational for building user interfaces in resource-constrained embedded systems, from simple status indicators to complex graphical applications.
