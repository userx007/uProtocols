# SPI (Serial Peripheral Interface) Programming Guide

## Overview

SPI is a synchronous serial communication protocol commonly used for short-distance communication between microcontrollers and peripheral devices like sensors, displays, SD cards, and ADCs. It operates in a master-slave configuration with separate lines for data transmission and clock synchronization.

## SPI Fundamentals

### Key Characteristics

**Four-Wire Interface:**
- **SCLK (Serial Clock)**: Generated by master, synchronizes data transmission
- **MOSI (Master Out Slave In)**: Data line from master to slave
- **MISO (Master In Slave Out)**: Data line from slave to master
- **CS/SS (Chip Select/Slave Select)**: Selects which slave device is active (active low)

**Operating Modes:**
SPI supports four modes based on Clock Polarity (CPOL) and Clock Phase (CPHA):
- **Mode 0**: CPOL=0, CPHA=0 (Clock idle low, sample on rising edge)
- **Mode 1**: CPOL=0, CPHA=1 (Clock idle low, sample on falling edge)
- **Mode 2**: CPOL=1, CPHA=0 (Clock idle high, sample on falling edge)
- **Mode 3**: CPOL=1, CPHA=1 (Clock idle high, sample on rising edge)

**Advantages:**
- Full-duplex communication
- High-speed data transfer (can exceed 10 Mbps)
- Simple hardware implementation
- No start/stop bits required

**Disadvantages:**
- Requires more pins than I2C
- No acknowledgment mechanism
- Single master limitation in basic configurations
- No built-in error checking

## C/C++ Implementation Examples

### Example 1: Basic SPI Driver (Embedded C)

```c
#include <stdint.h>
#include <stdbool.h>

// Hardware register definitions (example for generic MCU)
#define SPI_CR1     (*(volatile uint32_t*)0x40013000)
#define SPI_CR2     (*(volatile uint32_t*)0x40013004)
#define SPI_SR      (*(volatile uint32_t*)0x40013008)
#define SPI_DR      (*(volatile uint32_t*)0x4001300C)

// SPI Control Register bits
#define SPI_CR1_ENABLE      (1 << 6)
#define SPI_CR1_MASTER      (1 << 2)
#define SPI_CR1_BR_DIV16    (3 << 3)
#define SPI_CR1_CPOL_HIGH   (1 << 1)
#define SPI_CR1_CPHA_2EDGE  (1 << 0)

// Status Register bits
#define SPI_SR_TXE          (1 << 1)  // Transmit buffer empty
#define SPI_SR_RXNE         (1 << 0)  // Receive buffer not empty
#define SPI_SR_BSY          (1 << 7)  // Busy flag

typedef enum {
    SPI_MODE_0 = 0,  // CPOL=0, CPHA=0
    SPI_MODE_1 = 1,  // CPOL=0, CPHA=1
    SPI_MODE_2 = 2,  // CPOL=1, CPHA=0
    SPI_MODE_3 = 3   // CPOL=1, CPHA=1
} spi_mode_t;

typedef struct {
    uint32_t frequency;
    spi_mode_t mode;
    bool msb_first;
} spi_config_t;

// Initialize SPI peripheral
void spi_init(const spi_config_t* config) {
    uint32_t cr1 = 0;
    
    // Configure as master
    cr1 |= SPI_CR1_MASTER;
    
    // Set clock divider (simplified - would calculate from frequency)
    cr1 |= SPI_CR1_BR_DIV16;
    
    // Set mode
    if (config->mode & 0x02) cr1 |= SPI_CR1_CPOL_HIGH;
    if (config->mode & 0x01) cr1 |= SPI_CR1_CPHA_2EDGE;
    
    // Write configuration
    SPI_CR1 = cr1;
    
    // Enable SPI
    SPI_CR1 |= SPI_CR1_ENABLE;
}

// Transfer single byte (blocking)
uint8_t spi_transfer_byte(uint8_t data) {
    // Wait for TX buffer empty
    while (!(SPI_SR & SPI_SR_TXE));
    
    // Send data
    SPI_DR = data;
    
    // Wait for RX buffer not empty
    while (!(SPI_SR & SPI_SR_RXNE));
    
    // Read received data
    return (uint8_t)SPI_DR;
}

// Transfer multiple bytes
void spi_transfer(const uint8_t* tx_data, uint8_t* rx_data, size_t len) {
    for (size_t i = 0; i < len; i++) {
        uint8_t tx_byte = tx_data ? tx_data[i] : 0xFF;
        uint8_t rx_byte = spi_transfer_byte(tx_byte);
        if (rx_data) {
            rx_data[i] = rx_byte;
        }
    }
    
    // Wait until not busy
    while (SPI_SR & SPI_SR_BSY);
}

// Example: Reading from an SPI sensor
typedef struct {
    uint8_t cs_pin;
} spi_device_t;

void cs_low(uint8_t pin) {
    // GPIO implementation to set CS low
}

void cs_high(uint8_t pin) {
    // GPIO implementation to set CS high
}

uint8_t read_sensor_register(spi_device_t* device, uint8_t reg_addr) {
    uint8_t tx_buf[2] = {reg_addr | 0x80, 0x00};  // Read bit set
    uint8_t rx_buf[2];
    
    cs_low(device->cs_pin);
    spi_transfer(tx_buf, rx_buf, 2);
    cs_high(device->cs_pin);
    
    return rx_buf[1];
}

void write_sensor_register(spi_device_t* device, uint8_t reg_addr, 
                           uint8_t value) {
    uint8_t tx_buf[2] = {reg_addr & 0x7F, value};  // Write bit clear
    
    cs_low(device->cs_pin);
    spi_transfer(tx_buf, NULL, 2);
    cs_high(device->cs_pin);
}
```

### Example 2: Linux User-Space SPI (C++)

```cpp
#include <iostream>
#include <fcntl.h>
#include <unistd.h>
#include <sys/ioctl.h>
#include <linux/spi/spidev.h>
#include <cstring>
#include <stdexcept>

class SPIDevice {
private:
    int fd;
    uint32_t speed;
    uint8_t mode;
    uint8_t bits_per_word;
    
public:
    SPIDevice(const char* device, uint32_t speed_hz, uint8_t spi_mode)
        : speed(speed_hz), mode(spi_mode), bits_per_word(8) {
        
        fd = open(device, O_RDWR);
        if (fd < 0) {
            throw std::runtime_error("Failed to open SPI device");
        }
        
        // Set SPI mode
        if (ioctl(fd, SPI_IOC_WR_MODE, &mode) < 0) {
            close(fd);
            throw std::runtime_error("Failed to set SPI mode");
        }
        
        // Set bits per word
        if (ioctl(fd, SPI_IOC_WR_BITS_PER_WORD, &bits_per_word) < 0) {
            close(fd);
            throw std::runtime_error("Failed to set bits per word");
        }
        
        // Set max speed
        if (ioctl(fd, SPI_IOC_WR_MAX_SPEED_HZ, &speed) < 0) {
            close(fd);
            throw std::runtime_error("Failed to set max speed");
        }
    }
    
    ~SPIDevice() {
        if (fd >= 0) {
            close(fd);
        }
    }
    
    void transfer(const uint8_t* tx_buffer, uint8_t* rx_buffer, size_t length) {
        struct spi_ioc_transfer transfer_data;
        memset(&transfer_data, 0, sizeof(transfer_data));
        
        transfer_data.tx_buf = reinterpret_cast<uintptr_t>(tx_buffer);
        transfer_data.rx_buf = reinterpret_cast<uintptr_t>(rx_buffer);
        transfer_data.len = length;
        transfer_data.speed_hz = speed;
        transfer_data.bits_per_word = bits_per_word;
        
        if (ioctl(fd, SPI_IOC_MESSAGE(1), &transfer_data) < 0) {
            throw std::runtime_error("SPI transfer failed");
        }
    }
    
    void write(const uint8_t* data, size_t length) {
        transfer(data, nullptr, length);
    }
    
    void read(uint8_t* data, size_t length) {
        uint8_t dummy[length];
        memset(dummy, 0xFF, length);
        transfer(dummy, data, length);
    }
};

// Example usage with an accelerometer
int main() {
    try {
        // Open SPI device at 1 MHz, Mode 0
        SPIDevice spi("/dev/spidev0.0", 1000000, SPI_MODE_0);
        
        // Read WHO_AM_I register (example for ADXL345)
        uint8_t tx_data[2] = {0x80 | 0x00, 0x00};  // Read register 0x00
        uint8_t rx_data[2] = {0};
        
        spi.transfer(tx_data, rx_data, 2);
        
        std::cout << "Device ID: 0x" << std::hex 
                  << static_cast<int>(rx_data[1]) << std::endl;
        
        // Configure device - write to POWER_CTL register
        uint8_t power_config[2] = {0x2D, 0x08};  // Enable measurement mode
        spi.write(power_config, 2);
        
        // Read acceleration data
        uint8_t read_cmd[7] = {0x80 | 0x40 | 0x32};  // Multi-byte read from 0x32
        uint8_t accel_data[7] = {0};
        
        spi.transfer(read_cmd, accel_data, 7);
        
        int16_t x = (accel_data[2] << 8) | accel_data[1];
        int16_t y = (accel_data[4] << 8) | accel_data[3];
        int16_t z = (accel_data[6] << 8) | accel_data[5];
        
        std::cout << "Acceleration - X: " << x 
                  << " Y: " << y 
                  << " Z: " << z << std::endl;
        
    } catch (const std::exception& e) {
        std::cerr << "Error: " << e.what() << std::endl;
        return 1;
    }
    
    return 0;
}
```

## Rust Implementation Examples

### Example 1: Embedded Rust with embedded-hal

```rust
#![no_std]
#![no_main]

use embedded_hal::blocking::spi::{Transfer, Write};
use embedded_hal::digital::v2::OutputPin;
use panic_halt as _;

// Generic SPI device abstraction
pub struct SpiDevice<SPI, CS> {
    spi: SPI,
    cs: CS,
}

impl<SPI, CS, E> SpiDevice<SPI, CS>
where
    SPI: Transfer<u8, Error = E> + Write<u8, Error = E>,
    CS: OutputPin,
{
    pub fn new(spi: SPI, cs: CS) -> Self {
        Self { spi, cs }
    }
    
    pub fn write_register(&mut self, register: u8, value: u8) -> Result<(), E> {
        let _ = self.cs.set_low();
        
        let mut buffer = [register & 0x7F, value];
        let result = self.spi.write(&buffer);
        
        let _ = self.cs.set_high();
        result
    }
    
    pub fn read_register(&mut self, register: u8) -> Result<u8, E> {
        let _ = self.cs.set_low();
        
        let mut buffer = [register | 0x80, 0x00];
        self.spi.transfer(&mut buffer)?;
        
        let _ = self.cs.set_high();
        Ok(buffer[1])
    }
    
    pub fn read_multiple(&mut self, start_register: u8, 
                         buffer: &mut [u8]) -> Result<(), E> {
        let _ = self.cs.set_low();
        
        // Send read command with auto-increment
        let mut cmd = [start_register | 0x80 | 0x40];
        self.spi.write(&cmd)?;
        
        // Read data
        self.spi.transfer(buffer)?;
        
        let _ = self.cs.set_high();
        Ok(())
    }
}

// Example: Accelerometer driver
pub struct ADXL345<SPI, CS> {
    device: SpiDevice<SPI, CS>,
}

#[derive(Debug, Clone, Copy)]
pub struct AccelData {
    pub x: i16,
    pub y: i16,
    pub z: i16,
}

impl<SPI, CS, E> ADXL345<SPI, CS>
where
    SPI: Transfer<u8, Error = E> + Write<u8, Error = E>,
    CS: OutputPin,
{
    const DEVID: u8 = 0x00;
    const POWER_CTL: u8 = 0x2D;
    const DATAX0: u8 = 0x32;
    
    pub fn new(spi: SPI, cs: CS) -> Result<Self, E> {
        let mut device = SpiDevice::new(spi, cs);
        
        // Verify device ID
        let id = device.read_register(Self::DEVID)?;
        if id != 0xE5 {
            // In embedded, we might panic or return a specific error
            panic!("Invalid device ID");
        }
        
        // Enable measurement mode
        device.write_register(Self::POWER_CTL, 0x08)?;
        
        Ok(Self { device })
    }
    
    pub fn read_accel(&mut self) -> Result<AccelData, E> {
        let mut data = [0u8; 6];
        self.device.read_multiple(Self::DATAX0, &mut data)?;
        
        Ok(AccelData {
            x: i16::from_le_bytes([data[0], data[1]]),
            y: i16::from_le_bytes([data[2], data[3]]),
            z: i16::from_le_bytes([data[4], data[5]]),
        })
    }
}
```

### Example 2: Linux SPI in Rust

```rust
use std::fs::{File, OpenOptions};
use std::os::unix::io::AsRawFd;
use std::io::{self, Error, ErrorKind};

const SPI_IOC_MAGIC: u8 = b'k';
const SPI_IOC_MESSAGE_1: u8 = 0;

#[repr(C)]
struct SpiIocTransfer {
    tx_buf: u64,
    rx_buf: u64,
    len: u32,
    speed_hz: u32,
    delay_usecs: u16,
    bits_per_word: u8,
    cs_change: u8,
    tx_nbits: u8,
    rx_nbits: u8,
    pad: u16,
}

pub struct LinuxSpiDevice {
    file: File,
    speed_hz: u32,
    mode: u8,
}

impl LinuxSpiDevice {
    pub fn new(device_path: &str, speed_hz: u32, mode: u8) -> io::Result<Self> {
        let file = OpenOptions::new()
            .read(true)
            .write(true)
            .open(device_path)?;
        
        let fd = file.as_raw_fd();
        
        // Set SPI mode
        unsafe {
            if libc::ioctl(fd, request_code!(b'k', 1, 1), &mode) < 0 {
                return Err(Error::last_os_error());
            }
            
            // Set bits per word
            let bits: u8 = 8;
            if libc::ioctl(fd, request_code!(b'k', 3, 1), &bits) < 0 {
                return Err(Error::last_os_error());
            }
            
            // Set max speed
            if libc::ioctl(fd, request_code!(b'k', 4, 4), &speed_hz) < 0 {
                return Err(Error::last_os_error());
            }
        }
        
        Ok(Self { file, speed_hz, mode })
    }
    
    pub fn transfer(&self, tx_data: &[u8], rx_data: &mut [u8]) -> io::Result<()> {
        if tx_data.len() != rx_data.len() {
            return Err(Error::new(ErrorKind::InvalidInput, 
                                  "Buffer length mismatch"));
        }
        
        let transfer = SpiIocTransfer {
            tx_buf: tx_data.as_ptr() as u64,
            rx_buf: rx_data.as_mut_ptr() as u64,
            len: tx_data.len() as u32,
            speed_hz: self.speed_hz,
            delay_usecs: 0,
            bits_per_word: 8,
            cs_change: 0,
            tx_nbits: 0,
            rx_nbits: 0,
            pad: 0,
        };
        
        unsafe {
            if libc::ioctl(self.file.as_raw_fd(), 
                          request_code!(b'k', 0, 32), 
                          &transfer) < 0 {
                return Err(Error::last_os_error());
            }
        }
        
        Ok(())
    }
    
    pub fn write(&self, data: &[u8]) -> io::Result<()> {
        let mut dummy = vec![0u8; data.len()];
        self.transfer(data, &mut dummy)
    }
    
    pub fn read(&self, buffer: &mut [u8]) -> io::Result<()> {
        let dummy = vec![0xFFu8; buffer.len()];
        self.transfer(&dummy, buffer)
    }
}

// Helper macro for ioctl request codes
macro_rules! request_code {
    ($ty:expr, $nr:expr, $size:expr) => {
        (2u32 << 30) | (($ty as u32) << 8) | ($nr as u32) | (($size as u32) << 16)
    };
}

// Example usage
fn main() -> io::Result<()> {
    let spi = LinuxSpiDevice::new("/dev/spidev0.0", 1_000_000, 0)?;
    
    // Read device ID
    let tx = [0x80 | 0x00, 0x00];
    let mut rx = [0u8; 2];
    
    spi.transfer(&tx, &mut rx)?;
    println!("Device ID: 0x{:02X}", rx[1]);
    
    // Write configuration
    let config = [0x2D, 0x08];
    spi.write(&config)?;
    
    // Read sensor data
    let read_cmd = [0x80 | 0x40 | 0x32, 0, 0, 0, 0, 0, 0];
    let mut data = [0u8; 7];
    
    spi.transfer(&read_cmd, &mut data)?;
    
    let x = i16::from_le_bytes([data[1], data[2]]);
    let y = i16::from_le_bytes([data[3], data[4]]);
    let z = i16::from_le_bytes([data[5], data[6]]);
    
    println!("Acceleration: X={}, Y={}, Z={}", x, y, z);
    
    Ok(())
}
```

## Logic Analyzer Usage for SPI Debugging

Logic analyzers are essential tools for debugging SPI communication issues. They capture and visualize bus signals to identify timing problems, protocol violations, and data corruption.

### What to Capture

When debugging SPI, monitor all four signals:
- **SCLK**: Verify clock frequency and duty cycle
- **MOSI**: Master-to-slave data
- **MISO**: Slave-to-master data
- **CS**: Chip select timing and duration

### Common Issues Detected

1. **Clock Mode Mismatch**: Logic analyzer shows data sampled at wrong edge
2. **Timing Violations**: Setup/hold time violations visible in signal transitions
3. **CS Timing**: CS deasserted too early, cutting off data
4. **Data Corruption**: Glitches or noise on data lines during transmission
5. **Speed Issues**: Clock frequency too high for slave device capabilities

### Analysis Techniques

**Protocol Decoding**: Modern logic analyzers automatically decode SPI frames, displaying hex/binary values alongside raw signals.

**Trigger Setup**: Trigger on CS falling edge to capture complete transactions or on specific data patterns to isolate problematic transfers.

**Timing Measurements**: Measure clock-to-data delays, CS pulse widths, and inter-transaction gaps to verify they meet device specifications.

### Example Debug Scenario

When an SPI sensor returns incorrect data, a logic analyzer reveals:
- Clock idles at wrong level (CPOL mismatch)
- Data changes while CS is low between bytes (protocol violation)
- MISO transitions don't align with SCLK edges (timing issue)

These visualizations pinpoint the exact problem, whether it's incorrect mode configuration, buggy firmware, or hardware signal integrity issues.

## Summary

SPI is a versatile, high-speed serial communication protocol ideal for interfacing microcontrollers with peripheral devices. Its full-duplex operation and simple four-wire interface make it popular in embedded systems, though it trades pin efficiency for speed compared to alternatives like I2C.

**Key Programming Considerations:**
- Always configure clock polarity and phase (mode) to match slave device requirements
- Implement proper chip select control with adequate setup/hold times
- Handle both transmit and receive operations simultaneously in full-duplex mode
- Use DMA for high-throughput applications to reduce CPU overhead
- Implement timeout mechanisms for robust error handling

**Language-Specific Insights:**
- **C/C++**: Direct hardware register access in bare-metal systems; use Linux spidev driver for user-space applications
- **Rust**: Leverage embedded-hal traits for portable drivers; strong type system prevents common configuration errors
- Both languages support efficient zero-copy transfers and interrupt-driven operation

**Debugging Best Practices:**
- Use logic analyzers to verify timing, protocol compliance, and signal integrity
- Start with low clock speeds during development, then optimize
- Verify device datasheets for maximum clock rates and timing requirements
- Monitor power supply quality, as SPI is sensitive to voltage fluctuations

With proper implementation and debugging techniques, SPI provides reliable, high-performance communication for a wide range of embedded applications.
