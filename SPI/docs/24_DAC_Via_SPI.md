# DAC via SPI: Controlling Digital-to-Analog Converters

## Overview

A Digital-to-Analog Converter (DAC) is a device that converts digital values into analog voltage outputs. When controlled via SPI (Serial Peripheral Interface), DACs provide a precise and efficient way to generate analog signals from microcontrollers. This is essential for applications like audio generation, motor control, sensor calibration, and waveform synthesis.

## Understanding SPI Communication

SPI is a synchronous serial communication protocol that uses four main signals:

- **MOSI** (Master Out Slave In): Data from microcontroller to DAC
- **MISO** (Master In Slave Out): Data from DAC to microcontroller (often unused with DACs)
- **SCK** (Serial Clock): Clock signal generated by master
- **CS/SS** (Chip Select/Slave Select): Selects the specific DAC device

## DAC Basics

DACs typically have:
- **Resolution**: Number of bits (8-bit, 10-bit, 12-bit, 16-bit). Higher resolution means finer voltage steps
- **Reference Voltage**: Maximum output voltage (often Vref or VDD)
- **Output Range**: Typically 0V to Vref
- **Settling Time**: Time required for output to stabilize after a value change

The output voltage is calculated as:
```
Vout = (Digital_Value / Max_Digital_Value) × Vref
```

For a 12-bit DAC with 3.3V reference:
```
Vout = (Digital_Value / 4095) × 3.3V
```

## Common SPI DAC Examples

Popular SPI DACs include:
- **MCP4921/MCP4922**: 12-bit single/dual channel DACs
- **DAC8552**: 16-bit dual channel DAC
- **AD5206**: Digital potentiometer (acts like a DAC)
- **MCP4728**: 12-bit quad channel DAC with I2C (for comparison)

## C/C++ Implementation

Here's a comprehensive example using the MCP4921 12-bit DAC with an Arduino-compatible platform:

```c
#include <SPI.h>

// Pin definitions
#define DAC_CS_PIN 10
#define DAC_LDAC_PIN 9  // Load DAC pin (optional, can be tied to GND)

// MCP4921 Configuration bits
#define MCP4921_WRITE_CMD   0x3000  // Write command with buffered, 1x gain, active
#define MCP4921_SHUTDOWN    0x0000  // Shutdown bit
#define MCP4921_ACTIVE      0x1000  // Active mode bit
#define MCP4921_GAIN_1X     0x2000  // 1x gain (0-Vref)
#define MCP4921_GAIN_2X     0x0000  // 2x gain (0-2*Vref)
#define MCP4921_UNBUFFERED  0x4000  // Unbuffered Vref

class MCP4921_DAC {
private:
    uint8_t cs_pin;
    uint8_t ldac_pin;
    bool use_ldac;
    
public:
    MCP4921_DAC(uint8_t cs, uint8_t ldac = 255) 
        : cs_pin(cs), ldac_pin(ldac), use_ldac(ldac != 255) {}
    
    void begin() {
        pinMode(cs_pin, OUTPUT);
        digitalWrite(cs_pin, HIGH);
        
        if (use_ldac) {
            pinMode(ldac_pin, OUTPUT);
            digitalWrite(ldac_pin, HIGH);
        }
        
        SPI.begin();
        SPI.setDataMode(SPI_MODE0);
        SPI.setBitOrder(MSBFIRST);
        SPI.setClockDivider(SPI_CLOCK_DIV2); // Fast clock
    }
    
    // Write 12-bit value to DAC (0-4095)
    void write(uint16_t value, bool gain_2x = false, bool buffered = false) {
        // Ensure value is within 12-bit range
        value &= 0x0FFF;
        
        // Build command word
        uint16_t command = MCP4921_ACTIVE;
        
        if (!gain_2x) {
            command |= MCP4921_GAIN_1X;
        }
        
        if (!buffered) {
            command |= MCP4921_UNBUFFERED;
        }
        
        command |= value;
        
        // Send to DAC
        digitalWrite(cs_pin, LOW);
        SPI.transfer16(command);
        digitalWrite(cs_pin, HIGH);
        
        // Latch the output if LDAC is used
        if (use_ldac) {
            digitalWrite(ldac_pin, LOW);
            delayMicroseconds(1);
            digitalWrite(ldac_pin, HIGH);
        }
    }
    
    // Write voltage (in millivolts) assuming Vref = 3300mV
    void writeVoltage(uint16_t millivolts, uint16_t vref_mv = 3300) {
        uint16_t value = ((uint32_t)millivolts * 4095) / vref_mv;
        write(value);
    }
    
    // Shutdown DAC to save power
    void shutdown() {
        digitalWrite(cs_pin, LOW);
        SPI.transfer16(MCP4921_SHUTDOWN);
        digitalWrite(cs_pin, HIGH);
    }
};

// Example usage
MCP4921_DAC dac(DAC_CS_PIN, DAC_LDAC_PIN);

void setup() {
    Serial.begin(115200);
    dac.begin();
    
    // Set DAC to mid-scale (1.65V with 3.3V reference)
    dac.write(2048);
    
    Serial.println("DAC initialized");
}

void loop() {
    // Example 1: Generate a sine wave
    static uint16_t phase = 0;
    float angle = (phase * 2.0 * PI) / 360.0;
    float sine_val = (sin(angle) + 1.0) / 2.0; // Normalize to 0-1
    uint16_t dac_value = (uint16_t)(sine_val * 4095);
    
    dac.write(dac_value);
    
    phase = (phase + 1) % 360;
    delay(10);
}

// Example 2: Generate sawtooth wave
void generateSawtooth() {
    for (uint16_t i = 0; i < 4096; i++) {
        dac.write(i);
        delayMicroseconds(100);
    }
}

// Example 3: Generate triangle wave
void generateTriangle() {
    // Rising edge
    for (uint16_t i = 0; i < 4096; i++) {
        dac.write(i);
        delayMicroseconds(50);
    }
    // Falling edge
    for (uint16_t i = 4095; i > 0; i--) {
        dac.write(i);
        delayMicroseconds(50);
    }
}
```

## Rust Implementation

Here's a Rust implementation using embedded-hal traits for portability across different microcontrollers:

```rust
#![no_std]

use embedded_hal::blocking::spi::Write;
use embedded_hal::digital::v2::OutputPin;

/// MCP4921 12-bit DAC driver
pub struct Mcp4921<SPI, CS> {
    spi: SPI,
    cs: CS,
    config: u16,
}

/// Configuration options for MCP4921
pub struct Config {
    /// Buffered Vref input
    pub buffered: bool,
    /// Gain selection (true = 1x, false = 2x)
    pub gain_1x: bool,
    /// Active mode (false = shutdown)
    pub active: bool,
}

impl Default for Config {
    fn default() -> Self {
        Config {
            buffered: false,
            gain_1x: true,
            active: true,
        }
    }
}

impl<SPI, CS, E> Mcp4921<SPI, CS>
where
    SPI: Write<u8, Error = E>,
    CS: OutputPin,
{
    /// Create a new MCP4921 driver instance
    pub fn new(spi: SPI, cs: CS) -> Self {
        Self {
            spi,
            cs,
            config: 0x3000, // Default: active, 1x gain, unbuffered
        }
    }

    /// Initialize the DAC with custom configuration
    pub fn with_config(spi: SPI, cs: CS, config: Config) -> Self {
        let mut dac = Self::new(spi, cs);
        dac.set_config(config);
        dac
    }

    /// Update configuration
    pub fn set_config(&mut self, config: Config) {
        self.config = 0x0000;
        
        if config.active {
            self.config |= 0x1000; // Active bit
        }
        
        if config.gain_1x {
            self.config |= 0x2000; // Gain bit
        }
        
        if !config.buffered {
            self.config |= 0x4000; // Unbuffered bit
        }
    }

    /// Write a 12-bit value to the DAC (0-4095)
    pub fn write(&mut self, value: u16) -> Result<(), E> {
        // Mask to 12 bits and combine with configuration
        let data = self.config | (value & 0x0FFF);
        
        // Convert to two bytes (MSB first)
        let bytes = [
            ((data >> 8) & 0xFF) as u8,
            (data & 0xFF) as u8,
        ];

        // Send SPI transaction
        self.cs.set_low().ok();
        let result = self.spi.write(&bytes);
        self.cs.set_high().ok();
        
        result
    }

    /// Write voltage in millivolts (assumes 3.3V reference)
    pub fn write_voltage(&mut self, millivolts: u16, vref_mv: u16) -> Result<(), E> {
        let value = ((millivolts as u32 * 4095) / vref_mv as u32) as u16;
        self.write(value)
    }

    /// Shutdown the DAC to save power
    pub fn shutdown(&mut self) -> Result<(), E> {
        let bytes = [0x00, 0x00];
        
        self.cs.set_low().ok();
        let result = self.spi.write(&bytes);
        self.cs.set_high().ok();
        
        result
    }

    /// Release the SPI and CS pin
    pub fn release(self) -> (SPI, CS) {
        (self.spi, self.cs)
    }
}

/// Waveform generator using DAC
pub struct WaveformGenerator<SPI, CS> {
    dac: Mcp4921<SPI, CS>,
}

impl<SPI, CS, E> WaveformGenerator<SPI, CS>
where
    SPI: Write<u8, Error = E>,
    CS: OutputPin,
{
    pub fn new(dac: Mcp4921<SPI, CS>) -> Self {
        Self { dac }
    }

    /// Generate a single point on a sine wave
    pub fn sine_point(&mut self, phase: u16, amplitude: u16, offset: u16) -> Result<(), E> {
        // Use lookup table or approximation for no_std
        // Simple approximation using phase (0-360 degrees scaled to 0-4095)
        let normalized_phase = (phase as f32 / 4095.0) * 2.0 * core::f32::consts::PI;
        let sine_val = libm::sinf(normalized_phase);
        let value = offset + ((sine_val * amplitude as f32) as i32) as u16;
        
        self.dac.write(value.min(4095))
    }

    /// Generate sawtooth value
    pub fn sawtooth_point(&mut self, phase: u16) -> Result<(), E> {
        self.dac.write(phase & 0x0FFF)
    }

    /// Generate triangle value
    pub fn triangle_point(&mut self, phase: u16) -> Result<(), E> {
        let value = if phase < 2048 {
            phase * 2
        } else {
            4095 - ((phase - 2048) * 2)
        };
        
        self.dac.write(value)
    }

    /// Generate square wave value
    pub fn square_point(&mut self, phase: u16, duty_cycle: u16) -> Result<(), E> {
        let value = if phase < duty_cycle { 4095 } else { 0 };
        self.dac.write(value)
    }
}

// Example usage with cortex-m embedded system
#[cfg(feature = "example")]
mod example {
    use super::*;
    use cortex_m_rt::entry;
    use stm32f4xx_hal::{pac, prelude::*, spi::Spi};

    #[entry]
    fn main() -> ! {
        let dp = pac::Peripherals::take().unwrap();
        let rcc = dp.RCC.constrain();
        let clocks = rcc.cfgr.freeze();

        let gpioa = dp.GPIOA.split();
        let gpiob = dp.GPIOB.split();

        // SPI pins
        let sck = gpioa.pa5.into_alternate();
        let miso = gpioa.pa6.into_alternate();
        let mosi = gpioa.pa7.into_alternate();
        let cs = gpiob.pb6.into_push_pull_output();

        // Initialize SPI
        let spi = Spi::new(
            dp.SPI1,
            (sck, miso, mosi),
            embedded_hal::spi::MODE_0,
            1.MHz(),
            &clocks,
        );

        // Create DAC instance
        let mut dac = Mcp4921::new(spi, cs);
        let mut waveform_gen = WaveformGenerator::new(dac);

        let mut phase: u16 = 0;

        loop {
            // Generate sine wave
            waveform_gen.sine_point(phase, 2047, 2048).ok();
            
            phase = (phase + 10) & 0x0FFF;
            
            // Simple delay
            cortex_m::asm::delay(10000);
        }
    }
}
```

```c
void writeMultipleDACs(uint16_t* values, uint8_t* cs_pins, uint8_t count) {
    for (uint8_t i = 0; i < count; i++) {
        digitalWrite(cs_pins[i], LOW);
```

## Summary

DAC control via SPI enables precise analog output generation from digital systems. Key points include:

**Technical aspects**: SPI DACs use serial communication to receive digital values that are converted to proportional analog voltages. Resolution (8-16 bits) determines output precision, while configuration bits control gain, buffering, and operational modes.

**Implementation considerations**: Both C/C++ and Rust implementations require proper SPI configuration (clock polarity, bit order, speed), chip select management, and command word formatting. Higher-level abstractions can simplify waveform generation and multi-channel control.

**Common applications**: Audio synthesis, motor control, sensor calibration, programmable power supplies, test equipment, and arbitrary waveform generation all benefit from SPI-controlled DACs.

**Performance factors**: Update rates depend on SPI clock speed and settling time. DMA can enable continuous high-speed waveform playback without CPU intervention. Proper reference voltage selection and PCB layout affect accuracy and noise performance.

This technology bridges the digital-analog divide, making it essential for embedded systems requiring precise analog control from microcontroller platforms.
