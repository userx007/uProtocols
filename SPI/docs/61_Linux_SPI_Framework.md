## 1. Introduction to SPI

**Serial Peripheral Interface (SPI)** is a synchronous, full-duplex serial communication protocol widely used in embedded systems to connect microcontrollers and processors to peripherals such as ADCs, DACs, display controllers, flash memory chips, sensors, and RF transceivers.

### 1.1 SPI Signal Lines

| Signal | Direction        | Description                              |
|--------|------------------|------------------------------------------|
| SCLK   | Master → Slave   | Serial clock generated by the master     |
| MOSI   | Master → Slave   | Master Out, Slave In (data from master)  |
| MISO   | Slave → Master   | Master In, Slave Out (data to master)    |
| CS/SS  | Master → Slave   | Chip Select, active-low per slave device |

### 1.2 SPI Modes

The clock polarity (CPOL) and clock phase (CPHA) define four SPI modes:

| Mode | CPOL | CPHA | Clock idle | Data sampled on        |
|------|------|------|-----------|-------------------------|
| 0    | 0    | 0    | Low       | Rising edge             |
| 1    | 0    | 1    | Low       | Falling edge            |
| 2    | 1    | 0    | High      | Falling edge            |
| 3    | 1    | 1    | High      | Rising edge             |

## 2. Linux SPI Subsystem Architecture

The Linux SPI subsystem resides under `drivers/spi/` and is built around a clean separation of concerns:

```
┌─────────────────────────────────────────────────────────┐
│                  User Space (spidev)                    │
└────────────────────────┬────────────────────────────────┘
                         │ ioctl / read / write
┌────────────────────────▼────────────────────────────────┐
│              SPI Core (drivers/spi/spi.c)               │
│  - spi_register_controller()                            │
│  - spi_register_driver()                                │
│  - spi_sync() / spi_async()                             │
│  - spi_message_init() / spi_transfer                    │
└──────────┬──────────────────────────┬───────────────────┘
           │                          │
┌──────────▼──────────┐   ┌───────────▼───────────────────┐
│   SPI Controller    │   │   SPI Protocol/Device Driver  │
│   Driver            │   │   (e.g., spidev, mcp3204,     │
│   (spi-bcm2835,     │   │    ili9341, w25q128, etc.)    │
│    spi-stm32, etc.) │   └───────────────────────────────┘
└──────────┬──────────┘
           │ Hardware Abstraction
┌──────────▼──────────┐
│   SPI Hardware      │
│   (BCM2835, STM32,  │
│    i.MX6, etc.)     │
└─────────────────────┘
```

The SPI core provides:
- A bus abstraction (`spi_bus_type`) that binds controllers to device drivers.
- Message queuing, either via a kernel thread or workqueue.
- DMA mapping and bounce buffer management.
- A unified API for both synchronous and asynchronous transfers.

## 3. Key Data Structures

### 3.1 `struct spi_controller` (formerly `spi_master`)

Represents a hardware SPI bus controller. Defined in `include/linux/spi/spi.h`.

```c
struct spi_controller {
    struct device   dev;
    struct list_head list;

    s16             bus_num;          /* SPI bus number */
    u16             num_chipselect;   /* number of chip-selects */
    u16             dma_alignment;    /* DMA buffer alignment */
    u32             bits_per_word_mask;
    u32             min_speed_hz;
    u32             max_speed_hz;

    /* Transfer initiation */
    int (*transfer_one_message)(struct spi_controller *ctlr,
                                struct spi_message *mesg);
    int (*transfer_one)(struct spi_controller *ctlr,
                        struct spi_device *spi,
                        struct spi_transfer *transfer);

    /* Optional hooks */
    int  (*setup)(struct spi_device *spi);
    void (*set_cs)(struct spi_device *spi, bool enable);
    int  (*prepare_transfer_hardware)(struct spi_controller *ctlr);
    int  (*unprepare_transfer_hardware)(struct spi_controller *ctlr);

    /* Internal queuing (populated by spi_core) */
    struct kthread_worker   *kworker;
    struct kthread_work      pump_messages;
    struct list_head         queue;
    /* ... many more fields ... */
};
```

### 3.2 `struct spi_device`

Represents a single slave device on the SPI bus.

```c
struct spi_device {
    struct device       dev;
    struct spi_controller *controller;
    u32                 max_speed_hz;   /* maximum clock rate */
    u8                  chip_select;    /* chipselect index */
    u8                  bits_per_word;  /* bits per word (default: 8) */
    u16                 mode;           /* SPI_MODE_x flags */
    int                 irq;
    void               *controller_state;
    void               *controller_data;
    char                modalias[SPI_NAME_SIZE]; /* driver name */
    /* ... */
};
```

### 3.3 `struct spi_transfer`

Describes a single data transfer (read, write, or read+write).

```c
struct spi_transfer {
    const void  *tx_buf;        /* data to send (or NULL) */
    void        *rx_buf;        /* data to receive (or NULL) */
    unsigned    len;            /* in bytes */

    dma_addr_t  tx_dma;        /* DMA address for tx_buf */
    dma_addr_t  rx_dma;        /* DMA address for rx_buf */

    unsigned    cs_change:1;   /* deselect CS after this transfer? */
    unsigned    tx_nbits;      /* bus width for TX */
    unsigned    rx_nbits;      /* bus width for RX */
    u8          bits_per_word;
    u16         delay_usecs;   /* delay after this transfer */
    u32         speed_hz;      /* per-transfer speed override */

    struct list_head transfer_list; /* link in spi_message.transfers */
};
```

### 3.4 `struct spi_message`

A sequence of transfers that are executed atomically (CS held asserted throughout).

```c
struct spi_message {
    struct list_head    transfers;      /* list of spi_transfer */
    struct spi_device  *spi;
    unsigned            is_dma_mapped:1;
    void (*complete)(void *context);    /* async completion callback */
    void               *context;
    unsigned            frame_length;
    unsigned            actual_length;
    int                 status;         /* 0 = success */
    struct list_head    queue;          /* link in controller queue */
};
```

## 4. SPI Controller Driver (Master Side)

A controller driver registers a `spi_controller` with the SPI core and implements the low-level hardware operations.

### 4.1 Minimal Controller Driver in C

```c
// drivers/spi/spi-mycontroller.c
#include <linux/module.h>
#include <linux/platform_device.h>
#include <linux/spi/spi.h>
#include <linux/clk.h>
#include <linux/io.h>
#include <linux/interrupt.h>

/* Register offsets (example) */
#define SPI_CTRL    0x00
#define SPI_STATUS  0x04
#define SPI_DATA    0x08
#define SPI_CS      0x0C

#define SPI_CTRL_EN     BIT(0)
#define SPI_CTRL_TX_EN  BIT(1)
#define SPI_STATUS_BUSY BIT(0)
#define SPI_STATUS_DONE BIT(1)

struct myctlr_data {
    void __iomem        *base;
    struct clk          *clk;
    struct spi_controller *ctlr;
    int                  irq;
};

/* Write a single byte and read back the response byte */
static int myctlr_transfer_one(struct spi_controller *ctlr,
                                struct spi_device *spi,
                                struct spi_transfer *xfer)
{
    struct myctlr_data *priv = spi_controller_get_devdata(ctlr);
    const u8 *tx = xfer->tx_buf;
    u8       *rx = xfer->rx_buf;
    unsigned  len = xfer->len;
    u32       status;

    for (unsigned i = 0; i < len; i++) {
        /* Write TX byte */
        writel(tx ? tx[i] : 0xFF, priv->base + SPI_DATA);
        /* Trigger transfer */
        writel(SPI_CTRL_EN | SPI_CTRL_TX_EN, priv->base + SPI_CTRL);

        /* Poll for completion (production: use interrupt + completion) */
        if (readl_poll_timeout(priv->base + SPI_STATUS, status,
                               status & SPI_STATUS_DONE,
                               10, 100 * USEC_PER_MSEC)) {
            dev_err(&ctlr->dev, "SPI transfer timed out\n");
            return -ETIMEDOUT;
        }

        /* Read RX byte */
        if (rx)
            rx[i] = (u8)readl(priv->base + SPI_DATA);

        /* Clear done flag */
        writel(SPI_STATUS_DONE, priv->base + SPI_STATUS);
    }

    /* Signal to SPI core that transfer is complete */
    spi_finalize_current_transfer(ctlr);
    return 0;
}

static int myctlr_setup(struct spi_device *spi)
{
    struct myctlr_data *priv =
        spi_controller_get_devdata(spi->controller);
    u32 ctrl = 0;

    /* Configure CPOL/CPHA */
    if (spi->mode & SPI_CPOL)
        ctrl |= BIT(4);
    if (spi->mode & SPI_CPHA)
        ctrl |= BIT(5);

    writel(ctrl, priv->base + SPI_CTRL);
    return 0;
}

static int myctlr_probe(struct platform_device *pdev)
{
    struct spi_controller *ctlr;
    struct myctlr_data    *priv;
    struct resource       *res;
    int ret;

    /* Allocate controller + private data in one call */
    ctlr = spi_alloc_master(&pdev->dev, sizeof(*priv));
    if (!ctlr)
        return -ENOMEM;

    priv = spi_controller_get_devdata(ctlr);
    platform_set_drvdata(pdev, ctlr);

    /* Map hardware registers */
    res = platform_get_resource(pdev, IORESOURCE_MEM, 0);
    priv->base = devm_ioremap_resource(&pdev->dev, res);
    if (IS_ERR(priv->base)) {
        ret = PTR_ERR(priv->base);
        goto err_put;
    }

    /* Enable clock */
    priv->clk = devm_clk_get_enabled(&pdev->dev, NULL);
    if (IS_ERR(priv->clk)) {
        ret = PTR_ERR(priv->clk);
        goto err_put;
    }

    /* Fill in controller ops */
    ctlr->bus_num         = pdev->id;
    ctlr->num_chipselect  = 4;
    ctlr->max_speed_hz    = clk_get_rate(priv->clk) / 2;
    ctlr->bits_per_word_mask = SPI_BPW_MASK(8);
    ctlr->mode_bits       = SPI_CPOL | SPI_CPHA | SPI_CS_HIGH;
    ctlr->setup           = myctlr_setup;
    ctlr->transfer_one    = myctlr_transfer_one;

    /* Register with the SPI core — spawns kernel thread for queue */
    ret = devm_spi_register_controller(&pdev->dev, ctlr);
    if (ret) {
        dev_err(&pdev->dev, "Failed to register SPI controller: %d\n", ret);
        goto err_put;
    }

    dev_info(&pdev->dev, "myctlr SPI controller registered\n");
    return 0;

err_put:
    spi_controller_put(ctlr);
    return ret;
}

static int myctlr_remove(struct platform_device *pdev)
{
    /* devm_spi_register_controller handles unregistration automatically */
    return 0;
}

static const struct of_device_id myctlr_of_match[] = {
    { .compatible = "myvendor,myctlr-spi" },
    { }
};
MODULE_DEVICE_TABLE(of, myctlr_of_match);

static struct platform_driver myctlr_driver = {
    .probe  = myctlr_probe,
    .remove = myctlr_remove,
    .driver = {
        .name           = "myctlr-spi",
        .of_match_table = myctlr_of_match,
    },
};
module_platform_driver(myctlr_driver);

MODULE_AUTHOR("Embedded Developer");
MODULE_DESCRIPTION("Example SPI Controller Driver");
MODULE_LICENSE("GPL v2");
```

## 5. SPI Device Driver (Protocol/Client Side)

A protocol driver binds to a specific `spi_device` and uses the SPI core API to communicate with the hardware peripheral.

### 5.1 SPI Device Driver Skeleton in C

```c
// drivers/misc/mydevice-spi.c
#include <linux/module.h>
#include <linux/spi/spi.h>
#include <linux/iio/iio.h>
#include <linux/regmap.h>

#define MYDEV_REG_ID        0x00
#define MYDEV_REG_CONFIG    0x01
#define MYDEV_REG_DATA_H    0x02
#define MYDEV_REG_DATA_L    0x03

#define MYDEV_CHIP_ID       0xA5

struct mydev_data {
    struct spi_device *spi;
    struct mutex       lock;
    u8                 tx_buf[4] ____cacheline_aligned;
    u8                 rx_buf[4] ____cacheline_aligned;
};

/* ----------------------------------------------------------
 * Low-level register helpers
 * ---------------------------------------------------------- */

/**
 * mydev_reg_read - Read a single register byte via SPI
 *
 * SPI frame format: [0x80 | reg_addr] [dummy byte]
 *                   → two-byte transfer, read byte comes back on 2nd byte
 */
static int mydev_reg_read(struct mydev_data *dev, u8 reg, u8 *val)
{
    struct spi_transfer xfers[2] = {};
    struct spi_message  msg;
    int ret;

    mutex_lock(&dev->lock);

    dev->tx_buf[0] = 0x80 | reg;   /* read bit set */
    dev->tx_buf[1] = 0x00;         /* dummy */

    /* First transfer: send command */
    xfers[0].tx_buf  = &dev->tx_buf[0];
    xfers[0].len     = 1;
    xfers[0].cs_change = 0;

    /* Second transfer: receive data */
    xfers[1].rx_buf  = &dev->rx_buf[1];
    xfers[1].len     = 1;

    spi_message_init(&msg);
    spi_message_add_tail(&xfers[0], &msg);
    spi_message_add_tail(&xfers[1], &msg);

    ret = spi_sync(dev->spi, &msg);
    if (ret == 0)
        *val = dev->rx_buf[1];

    mutex_unlock(&dev->lock);
    return ret;
}

/**
 * mydev_reg_write - Write a single register byte via SPI
 *
 * SPI frame format: [reg_addr & 0x7F] [data_byte]
 */
static int mydev_reg_write(struct mydev_data *dev, u8 reg, u8 val)
{
    int ret;

    mutex_lock(&dev->lock);

    dev->tx_buf[0] = reg & 0x7F;   /* write: MSB = 0 */
    dev->tx_buf[1] = val;

    ret = spi_write(dev->spi, dev->tx_buf, 2);

    mutex_unlock(&dev->lock);
    return ret;
}

/**
 * mydev_read_sample - Read a 16-bit ADC sample (two registers)
 */
static int mydev_read_sample(struct mydev_data *dev, s16 *sample)
{
    u8  hi, lo;
    int ret;

    ret = mydev_reg_read(dev, MYDEV_REG_DATA_H, &hi);
    if (ret)
        return ret;
    ret = mydev_reg_read(dev, MYDEV_REG_DATA_L, &lo);
    if (ret)
        return ret;

    *sample = (s16)((hi << 8) | lo);
    return 0;
}

/* ----------------------------------------------------------
 * Probe / Remove
 * ---------------------------------------------------------- */

static int mydev_probe(struct spi_device *spi)
{
    struct mydev_data *data;
    u8 chip_id;
    int ret;

    /* Configure SPI parameters */
    spi->bits_per_word = 8;
    spi->mode          = SPI_MODE_0;
    spi->max_speed_hz  = 10000000;   /* 10 MHz */
    ret = spi_setup(spi);
    if (ret < 0)
        return ret;

    data = devm_kzalloc(&spi->dev, sizeof(*data), GFP_KERNEL);
    if (!data)
        return -ENOMEM;

    data->spi = spi;
    mutex_init(&data->lock);
    spi_set_drvdata(spi, data);

    /* Verify chip identity */
    ret = mydev_reg_read(data, MYDEV_REG_ID, &chip_id);
    if (ret) {
        dev_err(&spi->dev, "Failed to read chip ID: %d\n", ret);
        return ret;
    }
    if (chip_id != MYDEV_CHIP_ID) {
        dev_err(&spi->dev, "Unexpected chip ID 0x%02X (expected 0x%02X)\n",
                chip_id, MYDEV_CHIP_ID);
        return -ENODEV;
    }

    /* Put device into operational mode */
    ret = mydev_reg_write(data, MYDEV_REG_CONFIG, 0x01);
    if (ret) {
        dev_err(&spi->dev, "Failed to configure device: %d\n", ret);
        return ret;
    }

    dev_info(&spi->dev, "mydevice found, chip_id=0x%02X\n", chip_id);
    return 0;
}

static void mydev_remove(struct spi_device *spi)
{
    struct mydev_data *data = spi_get_drvdata(spi);
    /* Put device into power-down mode */
    mydev_reg_write(data, MYDEV_REG_CONFIG, 0x00);
}

static const struct spi_device_id mydev_id_table[] = {
    { "mydevice", 0 },
    { }
};
MODULE_DEVICE_TABLE(spi, mydev_id_table);

static const struct of_device_id mydev_of_match[] = {
    { .compatible = "myvendor,mydevice" },
    { }
};
MODULE_DEVICE_TABLE(of, mydev_of_match);

static struct spi_driver mydev_driver = {
    .driver = {
        .name           = "mydevice",
        .of_match_table = mydev_of_match,
    },
    .id_table = mydev_id_table,
    .probe    = mydev_probe,
    .remove   = mydev_remove,
};
module_spi_driver(mydev_driver);

MODULE_AUTHOR("Embedded Developer");
MODULE_DESCRIPTION("Example SPI Protocol Driver");
MODULE_LICENSE("GPL v2");
```

## 6. SPI Transfers and Messages

### 6.1 Synchronous Transfer Helpers

The SPI core provides convenience wrappers for common transfer patterns:

```c
/* Write N bytes, ignore RX */
static inline int spi_write(struct spi_device *spi,
                             const void *buf, size_t len);

/* Read N bytes, TX is all zeroes */
static inline int spi_read(struct spi_device *spi,
                            void *buf, size_t len);

/* Write txbuf, then read rxbuf (two separate CS assertions) */
static inline int spi_write_then_read(struct spi_device *spi,
    const void *txbuf, unsigned n_tx,
    void *rxbuf,       unsigned n_rx);

/* Full-duplex: TX and RX same length, one CS assertion */
static inline ssize_t spi_w8r8(struct spi_device *spi, u8 cmd);
static inline ssize_t spi_w8r16(struct spi_device *spi, u8 cmd);
static inline ssize_t spi_w8r16be(struct spi_device *spi, u8 cmd);
```

### 6.2 Manual Message Construction

For complex transactions (e.g., command + payload in one CS assertion):

```c
static int mydev_burst_read(struct mydev_data *dev,
                             u8 start_reg, u8 *buf, size_t len)
{
    /*
     * Build a two-transfer message:
     *  xfer[0]: send 1-byte register address
     *  xfer[1]: receive 'len' data bytes
     * Both transfers share one CS assertion.
     */
    u8 cmd = 0x80 | start_reg;

    struct spi_transfer xfers[2] = {
        {
            .tx_buf = &cmd,
            .len    = 1,
        },
        {
            .rx_buf = buf,
            .len    = len,
        },
    };
    struct spi_message msg;

    spi_message_init_with_transfers(&msg, xfers, ARRAY_SIZE(xfers));
    return spi_sync(dev->spi, &msg);
}
```

### 6.3 Asynchronous Transfer

For non-blocking operation (e.g., from an interrupt context or a time-sensitive path):

```c
struct async_ctx {
    struct spi_message msg;
    struct spi_transfer xfer;
    u8 tx[4];
    u8 rx[4];
    struct completion done;
};

static void async_complete(void *ctx_ptr)
{
    struct async_ctx *ctx = ctx_ptr;
    complete(&ctx->done);
}

static int mydev_async_read(struct spi_device *spi)
{
    struct async_ctx ctx;
    int ret;

    init_completion(&ctx.done);
    ctx.tx[0] = 0x80 | MYDEV_REG_DATA_H;

    memset(&ctx.xfer, 0, sizeof(ctx.xfer));
    ctx.xfer.tx_buf = ctx.tx;
    ctx.xfer.rx_buf = ctx.rx;
    ctx.xfer.len    = 2;

    spi_message_init(&ctx.msg);
    ctx.msg.complete = async_complete;
    ctx.msg.context  = &ctx;
    spi_message_add_tail(&ctx.xfer, &ctx.msg);

    /* Queue and return immediately */
    ret = spi_async(spi, &ctx.msg);
    if (ret)
        return ret;

    /* Wait for completion (or use a workqueue / tasklet) */
    wait_for_completion(&ctx.done);
    return ctx.msg.status;
}
```

## 7. Device Tree Configuration

### 7.1 Controller Node

```dts
/* arch/arm/boot/dts/myboard.dts */
spi0: spi@f0040000 {
    compatible    = "myvendor,myctlr-spi";
    reg           = <0xf0040000 0x1000>;
    interrupts    = <GIC_SPI 25 IRQ_TYPE_LEVEL_HIGH>;
    clocks        = <&pclk>;
    clock-names   = "pclk";
    #address-cells = <1>;
    #size-cells    = <0>;
    status        = "okay";

    /* Child node: SPI slave device at chipselect 0 */
    mydevice@0 {
        compatible    = "myvendor,mydevice";
        reg           = <0>;           /* chip-select index */
        spi-max-frequency = <10000000>; /* 10 MHz */
        spi-cpol;                       /* CPOL = 1 → mode 2 or 3 */
        spi-cpha;                       /* CPHA = 1 → mode 1 or 3 */
        /* Optional: GPIO-based chip select */
        cs-gpios      = <&gpio0 5 GPIO_ACTIVE_LOW>;
        interrupts-extended = <&gpio0 6 IRQ_TYPE_EDGE_FALLING>;
        interrupt-names = "drdy";
    };

    /* Second device at chipselect 1 */
    flash@1 {
        compatible    = "jedec,spi-nor";
        reg           = <1>;
        spi-max-frequency = <50000000>;
    };
};
```

### 7.2 Useful DT Properties

| Property              | Meaning                                           |
|-----------------------|---------------------------------------------------|
| `spi-max-frequency`   | Maximum clock frequency in Hz                     |
| `spi-cpol`            | Set CPOL=1 (clock idle high)                      |
| `spi-cpha`            | Set CPHA=1 (data sampled on second edge)          |
| `spi-cs-high`         | Chip select is active high (inverted)             |
| `spi-3wire`           | Half-duplex 3-wire mode (MOSI and MISO shared)    |
| `spi-lsb-first`       | Transmit LSB first                                |
| `cs-gpios`            | GPIO-emulated chip selects (controller node)      |

## 8. SPI in Userspace (spidev)

The `spidev` driver exposes raw SPI access from userspace via `/dev/spidevB.C` (B = bus, C = chipselect).

### 8.1 Enable in Device Tree

```dts
spi0: spi@... {
    spidev@0 {
        compatible        = "linux,spidev";
        reg               = <0>;
        spi-max-frequency = <1000000>;
    };
};
```

### 8.2 Userspace C Example

```c
// tools/spidev_test_custom.c
#include <stdio.h>
#include <stdlib.h>
#include <stdint.h>
#include <string.h>
#include <fcntl.h>
#include <unistd.h>
#include <sys/ioctl.h>
#include <linux/spi/spidev.h>

#define SPI_DEVICE  "/dev/spidev0.0"
#define SPI_SPEED   10000000   /* 10 MHz */
#define SPI_BITS    8
#define SPI_MODE    SPI_MODE_0

static int spi_fd;

static int spi_open(void)
{
    uint8_t  mode  = SPI_MODE;
    uint8_t  bits  = SPI_BITS;
    uint32_t speed = SPI_SPEED;

    spi_fd = open(SPI_DEVICE, O_RDWR);
    if (spi_fd < 0) { perror("open"); return -1; }

    if (ioctl(spi_fd, SPI_IOC_WR_MODE, &mode)        < 0 ||
        ioctl(spi_fd, SPI_IOC_WR_BITS_PER_WORD, &bits) < 0 ||
        ioctl(spi_fd, SPI_IOC_WR_MAX_SPEED_HZ, &speed) < 0) {
        perror("ioctl config"); return -1;
    }
    return 0;
}

static int spi_transfer(const uint8_t *tx, uint8_t *rx, size_t len)
{
    struct spi_ioc_transfer tr = {
        .tx_buf        = (unsigned long)tx,
        .rx_buf        = (unsigned long)rx,
        .len           = len,
        .speed_hz      = SPI_SPEED,
        .delay_usecs   = 0,
        .bits_per_word = SPI_BITS,
        .cs_change     = 0,
    };
    return ioctl(spi_fd, SPI_IOC_MESSAGE(1), &tr);
}

static uint8_t spi_read_reg(uint8_t reg)
{
    uint8_t tx[2] = { 0x80 | reg, 0x00 };
    uint8_t rx[2] = { 0 };
    spi_transfer(tx, rx, sizeof(tx));
    return rx[1];
}

static void spi_write_reg(uint8_t reg, uint8_t val)
{
    uint8_t tx[2] = { reg & 0x7F, val };
    spi_transfer(tx, NULL, sizeof(tx));
}

int main(void)
{
    if (spi_open() < 0) return EXIT_FAILURE;

    uint8_t id = spi_read_reg(0x00);
    printf("Chip ID: 0x%02X\n", id);

    spi_write_reg(0x01, 0x01);      /* configure device */

    /* Read 16-bit sample */
    uint8_t hi = spi_read_reg(0x02);
    uint8_t lo = spi_read_reg(0x03);
    int16_t sample = (int16_t)((hi << 8) | lo);
    printf("Sample: %d\n", sample);

    close(spi_fd);
    return EXIT_SUCCESS;
}
```

## 9. Rust in the Linux Kernel — SPI Bindings

As of Linux 6.x, the kernel's Rust support (`rust/` directory) provides safe abstractions for many subsystems. SPI bindings are actively being developed. The following examples reflect the current upstream approach and the direction Rust-in-Linux is taking.

### 9.1 Rust SPI Device Driver Skeleton

```rust
// rust/kernel/spi.rs (simplified binding)
// drivers/misc/mydevice_spi_rs.rs

use kernel::prelude::*;
use kernel::spi;
use kernel::sync::Mutex;

module! {
    type: MyDeviceDriver,
    name: "mydevice_spi",
    author: "Embedded Developer",
    description: "Example SPI driver in Rust",
    license: "GPL",
}

/// Private data stored per SPI device instance.
struct MyDeviceData {
    // Mutex protects concurrent access from multiple contexts.
    chip_id: u8,
}

struct MyDeviceDriver;

#[vtable]
impl spi::Driver for MyDeviceDriver {
    type Data = Mutex<MyDeviceData>;
    type IdInfo = ();

    /// Device/firmware compatibility table
    const OF_DEVICE_ID_TABLE: Option<&'static [spi::of::DeviceId<Self::IdInfo>]> = Some(&[
        spi::of::DeviceId::new(b"myvendor,mydevice\0", &()),
    ]);

    fn probe(dev: &mut spi::Device, _id_info: Option<&()>) -> Result<Self::Data> {
        dev_info!(dev, "Probing mydevice SPI driver\n");

        // Configure SPI parameters (mirrors spi_setup() in C)
        dev.set_max_speed_hz(10_000_000);
        dev.set_bits_per_word(8);
        dev.set_mode(spi::Mode::MODE_0);
        dev.setup()?;

        // Read the chip-ID register (register 0x00, read-mode bit set)
        let tx: [u8; 2] = [0x80, 0x00];
        let mut rx: [u8; 2] = [0u8; 2];
        dev.write_then_read(&tx, &mut rx)?;
        let chip_id = rx[1];

        if chip_id != 0xA5 {
            dev_err!(dev, "Unexpected chip ID: {:#04x}\n", chip_id);
            return Err(ENODEV);
        }

        dev_info!(dev, "Found device, chip_id={:#04x}\n", chip_id);

        // Return the driver data wrapped in a Mutex
        Ok(Mutex::new(MyDeviceData { chip_id }))
    }

    fn remove(dev: &mut spi::Device, data: &Self::Data) {
        let d = data.lock();
        dev_info!(dev, "Removing mydevice (chip_id={:#04x})\n", d.chip_id);
        // Power down device: write 0x00 to config register
        let _ = dev.write(&[0x01u8, 0x00]);
    }
}
```

### 9.2 Performing SPI Transfers in Rust

```rust
use kernel::spi;
use kernel::prelude::*;

/// Perform a register read: send [0x80|reg, 0x00] and return byte [1]
fn reg_read(dev: &spi::Device, reg: u8) -> Result<u8> {
    let tx: [u8; 2] = [0x80 | reg, 0x00];
    let mut rx: [u8; 2] = [0u8; 2];
    dev.write_then_read(&tx, &mut rx)?;
    Ok(rx[1])
}

/// Perform a register write: send [reg & 0x7F, val]
fn reg_write(dev: &spi::Device, reg: u8, val: u8) -> Result {
    let tx: [u8; 2] = [reg & 0x7F, val];
    dev.write(&tx)?;
    Ok(())
}

/// Read a 16-bit ADC sample from two consecutive registers
fn read_sample(dev: &spi::Device) -> Result<i16> {
    let hi = reg_read(dev, 0x02)?;
    let lo = reg_read(dev, 0x03)?;
    Ok(i16::from_be_bytes([hi, lo]))
}

/// Burst read: command byte followed by N receive bytes, one CS assertion
fn burst_read(dev: &spi::Device, start_reg: u8, buf: &mut [u8]) -> Result {
    // Construct a two-transfer SPI message using the transfer builder
    let cmd: [u8; 1] = [0x80 | start_reg];
    let xfers = &mut [
        spi::Transfer::new_write(&cmd),
        spi::Transfer::new_read(buf),
    ];
    let mut msg = spi::Message::new(xfers);
    dev.sync(&mut msg)?;
    Ok(())
}
```

### 9.3 Rust Userspace SPI Access

For userspace tooling in Rust, the `spidev` crate wraps the Linux `spidev` ioctl interface:

```toml
# Cargo.toml
[dependencies]

```

```rust
// src/main.rs — Rust userspace SPI example
use spidev::{Spidev, SpidevOptions, SpiModeFlags, SpidevTransfer};
use std::io::{Read, Write};

fn main() -> std::io::Result<()> {
    let mut spi = Spidev::open("/dev/spidev0.0")?;

    // Configure the SPI device
    let options = SpidevOptions::new()
        .bits_per_word(8)
        .max_speed_hz(10_000_000)
        .mode(SpiModeFlags::SPI_MODE_0)
        .build();
    spi.configure(&options)?;

    // ---- Simple write + read ----
    let tx = [0x80u8, 0x00]; // Read register 0x00
    let mut rx = [0u8; 2];
    {
        let mut transfer = SpidevTransfer::read_write(&tx, &mut rx);
        spi.transfer(&mut transfer)?;
    }
    let chip_id = rx[1];
    println!("Chip ID: {:#04x}", chip_id);

    // ---- Write a register ----
    spi.write_all(&[0x01u8, 0x01])?; // Write 0x01 to register 0x01

    // ---- Burst read: command then data in one CS assertion ----
    let cmd = [0x80u8 | 0x02]; // Read from register 0x02
    let mut data = [0u8; 4];
    let mut transfers = [
        SpidevTransfer::write(&cmd),
        SpidevTransfer::read(&mut data),
    ];
    spi.transfer_multiple(&mut transfers)?;

    let sample = i16::from_be_bytes([data[0], data[1]]);
    println!("Sample: {}", sample);

    Ok(())
}
```

### 10.4 Logic Analyzer Tips

When capturing with a logic analyzer (e.g., Sigrok/PulseView):

- Decode with the **SPI protocol decoder**, setting the correct CPOL/CPHA.

## 11. Summary

The Linux SPI framework provides a robust, layered architecture for interfacing with SPI hardware at every level — from bare-metal controller drivers to high-level protocol drivers and userspace access.

### Architecture at a Glance

| Layer               | Role                                                         | Key API / Files                        |
|---------------------|--------------------------------------------------------------|----------------------------------------|
| **SPI Core**        | Bus abstraction, message queuing, DMA management             | `drivers/spi/spi.c`, `spi.h`          |
| **Controller Driver** | Hardware register access, CS/clock management             | `spi_alloc_master()`, `transfer_one()` |
| **Protocol Driver** | Device-specific logic, register maps, IIO/input integration  | `spi_sync()`, `spi_async()`            |
| **spidev**          | Raw userspace access via `/dev/spidevB.C`                    | `SPI_IOC_MESSAGE()` ioctl              |
| **Rust bindings**   | Safe, idiomatic kernel driver authoring                      | `rust/kernel/spi.rs`                   |

### Key Takeaways

**Separation of concerns** is enforced throughout: controller drivers deal only with moving bits, protocol drivers focus on device logic, and the SPI core glues them together with safe message queuing and resource management.

**The message/transfer model** allows atomic multi-transfer transactions (CS held throughout) which is critical for devices that require a command byte followed immediately by a data phase without a CS glitch.

**Device Tree** is the canonical way to describe SPI topology on embedded Linux targets; the compatible string drives probe matching for both controllers and devices.

**DMA-safe buffers** are essential for performance: always declare TX/RX buffers with `____cacheline_aligned` in kernel drivers or ensure userspace buffers are page-aligned for DMA path efficiency.

**Rust** is a first-class citizen in the Linux kernel as of 6.1+. While SPI bindings are still maturing, the existing safe abstractions already enable writing correct, memory-safe SPI drivers with clear ownership semantics — eliminating an entire class of use-after-free and race condition bugs common in C drivers.

**Debugging** is well-supported: dynamic debug, ftrace tracepoints, and sysfs statistics give full visibility into the runtime behavior of the SPI stack without modifying source code.
