# ADC via SPI: Comprehensive Guide

## Overview

ADC (Analog-to-Digital Converter) via SPI (Serial Peripheral Interface) is a common technique for reading analog sensor values in embedded systems. SPI-based ADCs offer higher precision, faster sampling rates, and better noise immunity compared to built-in microcontroller ADCs, making them ideal for industrial, medical, and high-precision measurement applications.

## SPI Protocol Fundamentals

SPI is a synchronous, full-duplex communication protocol that uses four main signals:

- **MOSI** (Master Out Slave In): Data line from master to slave
- **MISO** (Master In Slave Out): Data line from slave to master
- **SCLK** (Serial Clock): Clock signal generated by master
- **CS/SS** (Chip Select/Slave Select): Device selection line (active low)

The master controls all communication, generating the clock and selecting which slave device to communicate with. Data is typically transferred MSB-first, and both devices can transmit and receive simultaneously.

## Common SPI ADC ICs

Popular SPI ADC chips include:

- **MCP3008/3208**: 10/12-bit, 8-channel ADCs (widely used with Raspberry Pi)
- **ADS1118**: 16-bit, 4-channel with integrated temperature sensor
- **MCP3201/3202**: 12-bit, 1-2 channel high-speed ADCs
- **MAX11100**: 16-bit, single-channel with internal reference

## Communication Protocol

A typical SPI ADC read cycle involves:

1. Assert CS low to select the device
2. Send configuration/command bytes (channel selection, mode)
3. Clock out the ADC result (usually 12-16 bits)
4. Deassert CS high to end transaction

## C/C++ Implementation

Here's a detailed implementation for the MCP3008 ADC:

```c
#include <stdint.h>
#include <stdio.h>

// Hardware abstraction - platform specific
// These functions would be implemented for your specific hardware
void spi_init(void);
void cs_low(void);
void cs_high(void);
uint8_t spi_transfer(uint8_t data);
void delay_us(uint32_t us);

// MCP3008 specific definitions
#define MCP3008_CHANNELS 8
#define MCP3008_MAX_VALUE 1023  // 10-bit ADC

/**
 * Read from MCP3008 ADC channel
 * 
 * @param channel: Channel number (0-7)
 * @param single_ended: 1 for single-ended, 0 for differential
 * @return: 10-bit ADC value (0-1023) or -1 on error
 */
int16_t mcp3008_read_channel(uint8_t channel, uint8_t single_ended) {
    if (channel >= MCP3008_CHANNELS) {
        return -1;  // Invalid channel
    }
    
    // Build command byte
    // Start bit = 1, SGL/DIFF bit, D2 D1 D0 (channel select)
    uint8_t command_byte = 0x01;  // Start bit
    if (single_ended) {
        command_byte |= 0x80;  // Single-ended mode
    }
    command_byte |= (channel << 4);  // Channel selection
    
    // Perform SPI transaction
    cs_low();
    delay_us(1);  // Small delay for CS setup
    
    // Send start bit
    spi_transfer(0x01);
    
    // Send command byte and receive high bits
    uint8_t high_byte = spi_transfer(command_byte);
    
    // Receive low byte
    uint8_t low_byte = spi_transfer(0x00);
    
    delay_us(1);  // Small delay before CS release
    cs_high();
    
    // Combine bytes into 10-bit result
    // MCP3008 returns: null bit, B9, B8 in first byte
    //                  B7-B0 in second byte
    uint16_t result = ((high_byte & 0x03) << 8) | low_byte;
    
    return (int16_t)result;
}

/**
 * Convert ADC value to voltage
 * 
 * @param adc_value: Raw ADC reading (0-1023)
 * @param vref: Reference voltage (typically 3.3V or 5.0V)
 * @return: Voltage in volts
 */
float adc_to_voltage(uint16_t adc_value, float vref) {
    return ((float)adc_value / (float)MCP3008_MAX_VALUE) * vref;
}

// Example usage
int main(void) {
    spi_init();
    
    printf("MCP3008 ADC Reading Example\n");
    printf("============================\n\n");
    
    while (1) {
        // Read all 8 channels
        for (uint8_t ch = 0; ch < MCP3008_CHANNELS; ch++) {
            int16_t raw_value = mcp3008_read_channel(ch, 1);
            
            if (raw_value >= 0) {
                float voltage = adc_to_voltage(raw_value, 3.3);
                printf("CH%d: %4d (%.3fV)  ", ch, raw_value, voltage);
            } else {
                printf("CH%d: ERROR  ", ch);
            }
        }
        printf("\n");
        
        // Delay between readings
        for (uint32_t i = 0; i < 1000000; i++);  // Simple delay
    }
    
    return 0;
}
```

## Rust Implementation

Here's a comprehensive Rust implementation using the `embedded-hal` traits:

```rust
#![no_std]

use embedded_hal::blocking::spi::Transfer;
use embedded_hal::digital::v2::OutputPin;

/// Error types for ADC operations
#[derive(Debug)]
pub enum AdcError<SpiError, PinError> {
    SpiError(SpiError),
    PinError(PinError),
    InvalidChannel,
    InvalidData,
}

/// MCP3008 10-bit 8-channel ADC driver
pub struct Mcp3008<SPI, CS> {
    spi: SPI,
    cs: CS,
    vref: f32,
}

impl<SPI, CS, SpiError, PinError> Mcp3008<SPI, CS>
where
    SPI: Transfer<u8, Error = SpiError>,
    CS: OutputPin<Error = PinError>,
{
    /// Create new MCP3008 instance
    pub fn new(spi: SPI, cs: CS, vref: f32) -> Self {
        Self { spi, cs, vref }
    }
    
    /// Read a single channel in single-ended mode
    pub fn read_channel(&mut self, channel: u8) 
        -> Result<u16, AdcError<SpiError, PinError>> 
    {
        if channel >= 8 {
            return Err(AdcError::InvalidChannel);
        }
        
        // Build command byte
        // Start bit (1) + SGL/DIFF (1 for single-ended) + D2 D1 D0
        let command: u8 = 0x01;  // Start bit
        let command = command | 0x80;  // Single-ended mode
        let command = command | (channel << 4);
        
        // Prepare buffer for SPI transaction
        let mut buffer = [0x01u8, command, 0x00];
        
        // Assert CS low
        self.cs.set_low().map_err(AdcError::PinError)?;
        
        // Perform SPI transfer
        self.spi.transfer(&mut buffer).map_err(AdcError::SpiError)?;
        
        // Deassert CS high
        self.cs.set_high().map_err(AdcError::PinError)?;
        
        // Extract 10-bit result
        // buffer[1] contains null bit + B9 + B8
        // buffer[2] contains B7-B0
        let result = (((buffer[1] & 0x03) as u16) << 8) | (buffer[2] as u16);
        
        Ok(result)
    }
    
    /// Read channel and convert to voltage
    pub fn read_voltage(&mut self, channel: u8) 
        -> Result<f32, AdcError<SpiError, PinError>> 
    {
        let raw = self.read_channel(channel)?;
        Ok(self.raw_to_voltage(raw))
    }
    
    /// Convert raw ADC value to voltage
    pub fn raw_to_voltage(&self, raw: u16) -> f32 {
        (raw as f32 / 1023.0) * self.vref
    }
    
    /// Read all 8 channels
    pub fn read_all_channels(&mut self) 
        -> Result<[u16; 8], AdcError<SpiError, PinError>> 
    {
        let mut readings = [0u16; 8];
        for (i, reading) in readings.iter_mut().enumerate() {
            *reading = self.read_channel(i as u8)?;
        }
        Ok(readings)
    }
}

/// ADC reading with metadata
#[derive(Debug, Clone, Copy)]
pub struct AdcReading {
    pub channel: u8,
    pub raw: u16,
    pub voltage: f32,
}

impl AdcReading {
    pub fn new(channel: u8, raw: u16, voltage: f32) -> Self {
        Self { channel, raw, voltage }
    }
}

// Example usage in an embedded context
#[cfg(feature = "example")]
mod example {
    use super::*;
    
    pub fn example_usage<SPI, CS, SpiError, PinError>(
        spi: SPI, 
        cs: CS
    ) -> Result<(), AdcError<SpiError, PinError>>
    where
        SPI: Transfer<u8, Error = SpiError>,
        CS: OutputPin<Error = PinError>,
    {
        // Create ADC instance with 3.3V reference
        let mut adc = Mcp3008::new(spi, cs, 3.3);

        let raw = adc.read_channel(0)?;

        // Or read voltage directly

        Ok(())
    }
}
```

```rust
pub struct MovingAverageFilter {
    buffer: [u16; 16],
    index: usize,
    count: usize,
}

impl MovingAverageFilter {
    pub fn new() -> Self {
        Self {
            buffer: [0; 16],
            index: 0,
            count: 0,
        }
    }
    
    pub fn add_sample(&mut self, sample: u16) -> u16 {
        self.buffer[self.index] = sample;
        self.index = (self.index + 1) % self.buffer.len();
        
        if self.count < self.buffer.len() {
            self.count += 1;
        }
        
        self.get_average()
    }
    
    pub fn get_average(&self) -> u16 {
        if self.count == 0 {
            return 0;
        }
        
        let sum: u32 = self.buffer[..self.count]
            .iter()
            .map(|&x| x as u32)
            .sum();
        
        (sum / self.count as u32) as u16
    }
```

## Practical Considerations

### Timing and Speed

- SPI clock frequency depends on the ADC specification (typically 1-10 MHz)
- Conversion time varies by ADC (MCP3008: ~200 kHz sampling rate)
- Add appropriate delays between CS assertion and data transfer

### Noise Reduction

1. **Hardware**: Use bypass capacitors, proper grounding, twisted pair cables
2. **Software**: Implement moving average or median filters
3. **Oversampling**: Take multiple readings and average them

### Error Handling

- Validate channel numbers before communication
- Check SPI transaction success
- Verify data integrity (some ADCs provide checksums)
- Handle CS pin state errors

### Power Considerations

- Many SPI ADCs support low-power modes
- Disable ADC between readings for battery-powered applications
- Consider ADC reference voltage stability

## Summary

ADC via SPI provides a robust, scalable solution for analog measurements in embedded systems. The SPI protocol's synchronous nature ensures reliable data transfer, while external ADCs offer superior performance compared to internal MCU ADCs. The implementations shown demonstrate key concepts across different programming paradigms:

**C/C++** implementations focus on direct hardware control with bit manipulation and flexible function-based or object-oriented designs suitable for resource-constrained systems.

**Rust** implementations leverage the embedded-hal ecosystem for hardware abstraction, providing type-safe interfaces with zero-cost abstractions, compile-time guarantees, and excellent error handling through the Result type.

Common challenges include proper timing management, noise filtering, and ensuring data integrity during high-speed transfers. When implementing SPI ADC systems, consider sample rate requirements, precision needs, channel count, power consumption, and whether advanced features like DMA or hardware filtering are necessary for your application.
