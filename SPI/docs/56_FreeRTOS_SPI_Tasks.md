## Introduction

The Serial Peripheral Interface (SPI) is a synchronous, full-duplex serial communication protocol widely used in embedded systems to connect microcontrollers to peripherals such as sensors, displays, flash memory, ADCs, and DACs. When combined with FreeRTOS—a popular real-time operating system—SPI communication must be carefully integrated into the task scheduling model to ensure correctness, predictable timing, and safe concurrent access.

This document explores how to implement SPI communication within FreeRTOS task architecture. It covers the fundamental concepts, design patterns, and provides complete, annotated code examples in both **C/C++** and **Rust**.

---

## SPI Protocol Fundamentals

SPI uses four signals:

| Signal | Direction     | Description                                  |
|--------|---------------|----------------------------------------------|
| SCLK   | Master → Slave | Serial clock generated by the master         |
| MOSI   | Master → Slave | Master Out Slave In — data from master       |
| MISO   | Slave → Master | Master In Slave Out — data from slave        |
| CS/SS  | Master → Slave | Chip Select / Slave Select (active low)      |

Key characteristics:
- **Full-duplex**: MOSI and MISO transfer simultaneously
- **Synchronous**: All data is clocked by SCLK
- **Multi-slave**: Multiple devices share MOSI/MISO/SCLK but each has its own CS line
- **Configurable**: Four modes (CPOL × CPHA) control clock polarity and phase

In FreeRTOS, SPI transactions may be initiated from multiple tasks, so the SPI bus must be treated as a **shared resource** requiring careful synchronization.

---

## FreeRTOS and SPI: Design Considerations

When implementing SPI within a FreeRTOS application, several key concerns arise:

### 1. Bus Arbitration
Multiple tasks may need to use the SPI bus. Without protection, two tasks could simultaneously assert different CS lines and corrupt data. A **mutex** (or binary semaphore) must serialize access.

### 2. Blocking vs Non-Blocking Transfers
Polling (`while (!transfer_done)`) wastes CPU time in a multitasking environment. FreeRTOS tasks should **block** during SPI transfers—either on a semaphore signaled by an interrupt/DMA completion, or via a task notification.

### 3. DMA Usage
On capable hardware, SPI transfers via DMA allow the CPU to run other tasks during a transfer. The DMA-complete interrupt unblocks the waiting task.

### 4. Message Queues
A dedicated **SPI manager task** can own the bus and process transfer requests via a FreeRTOS queue. This decouples producer tasks from the physical bus.

### 5. Priority Inversion
If a low-priority task holds the SPI mutex and a high-priority task needs the bus, priority inversion can occur. FreeRTOS mutexes support **priority inheritance** to mitigate this.

### 6. Transaction Atomicity
A complete SPI transaction (CS assert → transfer → CS deassert) must not be interrupted by another device's transaction. The mutex scope must cover the full transaction.

## Architecture Patterns

Three common patterns are used for FreeRTOS SPI integration:

```
Pattern 1: Direct Access with Mutex
─────────────────────────────────────
 [Task A] ──┐
            ├──► [SPI Mutex] ──► [SPI Hardware]
 [Task B] ──┘

Pattern 2: Centralized SPI Manager Task
─────────────────────────────────────────
 [Task A] ──┐
            ├──► [Request Queue] ──► [SPI Manager Task] ──► [SPI Hardware]
 [Task B] ──┘
                                         │
                                    [Callback / Semaphore]
                                         │
                                   [Caller unblocks]

Pattern 3: DMA + Task Notification
─────────────────────────────────────
 [Task A] ──► [SPI Start DMA] ──► [Task blocks on notification]
                                         │
                                   [DMA ISR fires]
                                         │
                               [vTaskNotifyGiveFromISR]
                                         │
                                   [Task A resumes]
```

## C/C++ Implementation

### Basic SPI Task in C

This example shows a simple FreeRTOS task that performs periodic SPI reads from a sensor, using a mutex for bus protection and a semaphore for ISR-to-task synchronization.

```c
/*
 * 56_freertos_spi_basic.c
 * Basic FreeRTOS SPI task with ISR-driven synchronization.
 * Target: STM32-like HAL abstraction (adapt as needed).
 */

#include "FreeRTOS.h"
#include "task.h"
#include "semphr.h"
#include "spi_hal.h"   /* Platform SPI HAL */

/* ── Configuration ─────────────────────────────────────────── */
#define SPI_TASK_STACK_SIZE   512u   /* words */
#define SPI_TASK_PRIORITY     3u
#define SPI_TRANSFER_TIMEOUT  pdMS_TO_TICKS(100)
#define SENSOR_READ_PERIOD    pdMS_TO_TICKS(10)  /* 100 Hz */

/* ── FreeRTOS handles ───────────────────────────────────────── */
static SemaphoreHandle_t s_spiMutex     = NULL;  /* bus ownership */
static SemaphoreHandle_t s_transferDone = NULL;  /* ISR → task sync */

/* ── SPI transfer buffer ────────────────────────────────────── */
static uint8_t s_txBuf[4];
static uint8_t s_rxBuf[4];

/* ────────────────────────────────────────────────────────────
 * SPI Transfer Complete Callback (called from ISR context)
 * ──────────────────────────────────────────────────────────── */
void SPI_TransferCompleteCallback(void)
{
    BaseType_t xHigherPriorityTaskWoken = pdFALSE;

    /* Signal the waiting task */
    xSemaphoreGiveFromISR(s_transferDone, &xHigherPriorityTaskWoken);

    /* Yield to higher-priority task if one was unblocked */
    portYIELD_FROM_ISR(xHigherPriorityTaskWoken);
}

/* ────────────────────────────────────────────────────────────
 * Helper: Perform a complete SPI transaction atomically
 *   - Acquires mutex, asserts CS, transfers, deasserts CS,
 *     releases mutex.
 *   - Returns pdTRUE on success, pdFALSE on timeout/error.
 * ──────────────────────────────────────────────────────────── */
static BaseType_t SPI_Transaction(const uint8_t *txData,
                                   uint8_t       *rxData,
                                   size_t         len)
{
    /* Acquire the SPI bus mutex (blocks if another task owns it) */
    if (xSemaphoreTake(s_spiMutex, SPI_TRANSFER_TIMEOUT) != pdTRUE)
    {
        /* Bus not available within timeout */
        return pdFALSE;
    }

    /* Assert chip select */
    SPI_HAL_AssertCS();

    /* Initiate non-blocking (interrupt-driven) transfer */
    SPI_HAL_TransmitReceive_IT(txData, rxData, len);

    /* Block until ISR signals completion */
    BaseType_t result = xSemaphoreTake(s_transferDone,
                                        SPI_TRANSFER_TIMEOUT);

    /* Deassert chip select regardless of outcome */
    SPI_HAL_DeassertCS();

    /* Release the bus mutex */
    xSemaphoreGive(s_spiMutex);

    return result;
}

/* ────────────────────────────────────────────────────────────
 * SPI Sensor Task
 * ──────────────────────────────────────────────────────────── */
static void vSpiSensorTask(void *pvParameters)
{
    (void)pvParameters;

    TickType_t xLastWakeTime = xTaskGetTickCount();

    for (;;)
    {
        /* Prepare read command for sensor register 0x28 */
        s_txBuf[0] = 0x80 | 0x28;   /* Read bit | register address */
        s_txBuf[1] = 0x00;           /* Dummy byte to clock out data */

        if (SPI_Transaction(s_txBuf, s_rxBuf, 2) == pdTRUE)
        {
            /* s_rxBuf[1] contains the sensor value */
            int16_t rawValue = (int16_t)s_rxBuf[1];
            /* ... process rawValue ... */
        }
        else
        {
            /* Handle timeout / error */
        }

        /* Wait until next period (precise 10 ms cadence) */
        vTaskDelayUntil(&xLastWakeTime, SENSOR_READ_PERIOD);
    }
}

/* ────────────────────────────────────────────────────────────
 * Initialization — call from main() before scheduler starts
 * ──────────────────────────────────────────────────────────── */
void SpiTasks_Init(void)
{
    /* Create synchronization objects */
    s_spiMutex     = xSemaphoreCreateMutex();
    s_transferDone = xSemaphoreCreateBinary();

    configASSERT(s_spiMutex     != NULL);
    configASSERT(s_transferDone != NULL);

    /* Create the sensor task */
    xTaskCreate(vSpiSensorTask,
                "SpiSensor",
                SPI_TASK_STACK_SIZE,
                NULL,
                SPI_TASK_PRIORITY,
                NULL);
}
```

### SPI Bus Manager with Queue

This pattern uses a dedicated manager task that owns the SPI bus. Other tasks post `SpiRequest_t` messages to a queue and block on a response semaphore.

```c
/*
 * 56_freertos_spi_manager.c
 * SPI Manager Task — centralized bus ownership via FreeRTOS queue.
 */

#include "FreeRTOS.h"
#include "task.h"
#include "queue.h"
#include "semphr.h"
#include "spi_hal.h"

/* ── Request / Response Types ───────────────────────────────── */
typedef struct
{
    uint8_t          *txBuf;        /* transmit buffer          */
    uint8_t          *rxBuf;        /* receive buffer           */
    size_t            len;          /* transfer length in bytes */
    uint8_t           deviceId;     /* selects CS line          */
    SemaphoreHandle_t doneSem;      /* caller blocks on this    */
    BaseType_t       *resultOut;    /* pdTRUE / pdFALSE result  */
} SpiRequest_t;

/* ── Module State ───────────────────────────────────────────── */
#define SPI_QUEUE_DEPTH        8u
#define SPI_MANAGER_STACK      768u
#define SPI_MANAGER_PRIORITY   5u    /* Higher than client tasks */
#define SPI_TRANSFER_TIMEOUT   pdMS_TO_TICKS(200)

static QueueHandle_t     s_spiQueue      = NULL;
static SemaphoreHandle_t s_transferDone  = NULL;

/* ── ISR Callback ───────────────────────────────────────────── */
void SPI_TransferCompleteCallback(void)
{
    BaseType_t xHPTW = pdFALSE;
    xSemaphoreGiveFromISR(s_transferDone, &xHPTW);
    portYIELD_FROM_ISR(xHPTW);
}

/* ── SPI Manager Task ───────────────────────────────────────── */
static void vSpiManagerTask(void *pvParameters)
{
    (void)pvParameters;
    SpiRequest_t req;

    for (;;)
    {
        /* Block indefinitely waiting for a request */
        if (xQueueReceive(s_spiQueue, &req, portMAX_DELAY) != pdTRUE)
            continue;

        /* Assert correct CS based on device ID */
        SPI_HAL_AssertCS_Device(req.deviceId);

        /* Start interrupt-driven transfer */
        SPI_HAL_TransmitReceive_IT(req.txBuf, req.rxBuf, req.len);

        /* Wait for DMA/ISR completion */
        BaseType_t ok = xSemaphoreTake(s_transferDone,
                                        SPI_TRANSFER_TIMEOUT);

        SPI_HAL_DeassertCS_Device(req.deviceId);

        /* Report result back to caller */
        if (req.resultOut != NULL)
            *req.resultOut = ok;

        /* Unblock the requesting task */
        xSemaphoreGive(req.doneSem);
    }
}

/* ── Public API: submit a blocking SPI transfer ─────────────── */
BaseType_t SPI_Submit(uint8_t        deviceId,
                       const uint8_t *txBuf,
                       uint8_t       *rxBuf,
                       size_t         len)
{
    /* Each caller needs its own done-semaphore.
     * For static tasks this can be pre-allocated. */
    SemaphoreHandle_t doneSem = xSemaphoreCreateBinary();
    if (doneSem == NULL) return pdFALSE;

    BaseType_t result = pdFALSE;

    SpiRequest_t req = {
        .txBuf    = (uint8_t *)txBuf,
        .rxBuf    = rxBuf,
        .len      = len,
        .deviceId = deviceId,
        .doneSem  = doneSem,
        .resultOut = &result,
    };

    /* Post request; if queue is full return error immediately */
    if (xQueueSend(s_spiQueue, &req, pdMS_TO_TICKS(50)) != pdTRUE)
    {
        vSemaphoreDelete(doneSem);
        return pdFALSE;
    }

    /* Block until manager task signals completion */
    xSemaphoreTake(doneSem, portMAX_DELAY);
    vSemaphoreDelete(doneSem);

    return result;
}

/* ── Initialization ─────────────────────────────────────────── */
void SpiManager_Init(void)
{
    s_spiQueue     = xQueueCreate(SPI_QUEUE_DEPTH, sizeof(SpiRequest_t));
    s_transferDone = xSemaphoreCreateBinary();

    configASSERT(s_spiQueue     != NULL);
    configASSERT(s_transferDone != NULL);

    xTaskCreate(vSpiManagerTask, "SpiMgr",
                SPI_MANAGER_STACK, NULL,
                SPI_MANAGER_PRIORITY, NULL);
}
```

### DMA-driven SPI with Notifications

Task notifications are lighter than semaphores and ideal for single-consumer DMA completion signaling.

```c
/*
 * 56_freertos_spi_dma_notify.c
 * DMA-driven SPI using FreeRTOS task notifications (faster than semaphores).
 */

#include "FreeRTOS.h"
#include "task.h"
#include "semphr.h"
#include "spi_hal.h"

/* Store the handle of the task waiting for DMA completion */
static TaskHandle_t s_spiOwnerTask = NULL;
static SemaphoreHandle_t s_busMutex = NULL;

/* ── DMA Complete ISR callback ──────────────────────────────── */
void SPI_DMA_CompleteCallback(void)
{
    BaseType_t xHPTW = pdFALSE;

    if (s_spiOwnerTask != NULL)
    {
        /* Direct-to-task notification — no semaphore overhead */
        vTaskNotifyGiveFromISR(s_spiOwnerTask, &xHPTW);
    }

    portYIELD_FROM_ISR(xHPTW);
}

/* ── DMA SPI Transfer (blocking for caller) ─────────────────── */
BaseType_t SPI_TransferDMA(uint8_t deviceId,
                            const uint8_t *tx,
                            uint8_t       *rx,
                            size_t         len)
{
    if (xSemaphoreTake(s_busMutex, pdMS_TO_TICKS(100)) != pdTRUE)
        return pdFALSE;

    /* Register this task as the DMA owner */
    s_spiOwnerTask = xTaskGetCurrentTaskHandle();

    SPI_HAL_AssertCS_Device(deviceId);
    SPI_HAL_StartDMA(tx, rx, len);

    /* Block: wait for DMA ISR to send notification */
    BaseType_t ok = (ulTaskNotifyTake(pdTRUE,
                                       pdMS_TO_TICKS(200)) != 0)
                     ? pdTRUE : pdFALSE;

    SPI_HAL_DeassertCS_Device(deviceId);

    s_spiOwnerTask = NULL;
    xSemaphoreGive(s_busMutex);

    return ok;
}

/* ── Example usage task ─────────────────────────────────────── */
static void vDisplayTask(void *pvParams)
{
    (void)pvParams;

    static uint8_t displayFrame[128];  /* e.g. 128-byte LCD command */
    static uint8_t dummy[128];

    /* Prepare display frame ... */
    displayFrame[0] = 0x40;  /* data command */

    for (;;)
    {
        /* Ship frame over SPI DMA; task blocks until done */
        SPI_TransferDMA(/*deviceId=*/1, displayFrame, dummy,
                         sizeof(displayFrame));

        vTaskDelay(pdMS_TO_TICKS(16));  /* ~60 fps */
    }
}

void SpiDMA_Init(void)
{
    s_busMutex = xSemaphoreCreateMutex();
    configASSERT(s_busMutex != NULL);

    xTaskCreate(vDisplayTask, "Display", 512, NULL, 3, NULL);
}
```

### Multi-Device SPI with Mutex

When multiple peripherals share the SPI bus and each is accessed from independent tasks, a single mutex ensures safe interleaving.

```c
/*
 * 56_freertos_spi_multidevice.c
 * Multiple devices on one SPI bus, each accessed from separate tasks.
 */

#include "FreeRTOS.h"
#include "task.h"
#include "semphr.h"
#include "spi_hal.h"

typedef enum {
    SPI_DEV_SENSOR  = 0,
    SPI_DEV_DISPLAY = 1,
    SPI_DEV_FLASH   = 2,
} SpiDevice_t;

static SemaphoreHandle_t s_busMutex    = NULL;
static SemaphoreHandle_t s_xferDone    = NULL;

void SPI_TransferCompleteCallback(void)
{
    BaseType_t hp = pdFALSE;
    xSemaphoreGiveFromISR(s_xferDone, &hp);
    portYIELD_FROM_ISR(hp);
}

/* ── Generic transfer with device selection ─────────────────── */
static BaseType_t SPI_DeviceTransfer(SpiDevice_t device,
                                      const uint8_t *tx,
                                      uint8_t       *rx,
                                      size_t         len)
{
    if (xSemaphoreTake(s_busMutex, pdMS_TO_TICKS(500)) != pdTRUE)
        return pdFALSE;

    SPI_HAL_AssertCS_Device((uint8_t)device);
    SPI_HAL_TransmitReceive_IT(tx, rx, len);

    BaseType_t ok = xSemaphoreTake(s_xferDone, pdMS_TO_TICKS(200));

    SPI_HAL_DeassertCS_Device((uint8_t)device);
    xSemaphoreGive(s_busMutex);
    return ok;
}

/* ── Sensor task: reads at 100 Hz ───────────────────────────── */
static void vSensorTask(void *pv)
{
    (void)pv;
    TickType_t t = xTaskGetTickCount();
    uint8_t tx[2] = {0x80 | 0x28, 0x00};
    uint8_t rx[2];

    for (;;)
    {
        SPI_DeviceTransfer(SPI_DEV_SENSOR, tx, rx, 2);
        vTaskDelayUntil(&t, pdMS_TO_TICKS(10));
    }
}

/* ── Display task: refreshes at ~30 Hz ──────────────────────── */
static void vDisplayTask(void *pv)
{
    (void)pv;
    static uint8_t frame[256];
    static uint8_t discard[256];
    frame[0] = 0x40;  /* write data command */

    for (;;)
    {
        SPI_DeviceTransfer(SPI_DEV_DISPLAY, frame, discard,
                            sizeof(frame));
        vTaskDelay(pdMS_TO_TICKS(33));
    }
}

/* ── Flash task: on-demand write ────────────────────────────── */
static void vFlashTask(void *pv)
{
    (void)pv;
    uint8_t writePage[258];   /* cmd + addr + 256 bytes */
    uint8_t discard[258];
    writePage[0] = 0x02;      /* Page Program command */
    writePage[1] = 0x00;      /* address high byte */
    writePage[2] = 0x00;      /* address low byte */
    /* Fill writePage[3..258] with data ... */

    for (;;)
    {
        /* Write only when needed */
        SPI_DeviceTransfer(SPI_DEV_FLASH, writePage, discard,
                            sizeof(writePage));

        /* Wait for a trigger (e.g., notification from another task) */
        ulTaskNotifyTake(pdTRUE, portMAX_DELAY);
    }
}

void MultiSpi_Init(void)
{
    s_busMutex = xSemaphoreCreateMutex();
    s_xferDone = xSemaphoreCreateBinary();

    xTaskCreate(vSensorTask,  "Sensor",  512, NULL, 3, NULL);
    xTaskCreate(vDisplayTask, "Display", 512, NULL, 2, NULL);
    xTaskCreate(vFlashTask,   "Flash",   768, NULL, 1, NULL);
}
```

## Rust Implementation

Rust's ownership model and type system provide compile-time safety guarantees for concurrent SPI access that C cannot. The examples below use the `embassy` async executor and `embedded-hal` traits, which map naturally onto FreeRTOS-style task architectures.

### Basic SPI Task in Rust

```rust
// 56_freertos_spi_basic.rs
// Basic SPI task using Embassy async runtime (FreeRTOS-equivalent for Rust).
// Uses embassy-stm32 HAL; adapt embassy_stm32::spi::Spi for your target.

#![no_std]
#![no_main]

use embassy_executor::Spawner;
use embassy_stm32::{
    gpio::{Level, Output, Speed},
    spi::{Config, Spi},
    time::Hertz,
};
use embassy_time::{Duration, Ticker};
use embedded_hal_async::spi::SpiBus;

/// Read a single register from an SPI sensor.
/// CS is manually controlled to keep the transaction atomic.
async fn sensor_read_register<SPI, CS>(
    spi: &mut SPI,
    cs:  &mut CS,
    reg: u8,
) -> Result<u8, SPI::Error>
where
    SPI: SpiBus,
    CS:  embedded_hal::digital::OutputPin,
{
    let tx = [0x80 | reg, 0x00u8];   // read bit | address, dummy
    let mut rx = [0u8; 2];

    cs.set_low().ok();
    let result = spi.transfer(&mut rx, &tx).await;
    cs.set_high().ok();

    result.map(|_| rx[1])
}

/// Sensor task — runs at 100 Hz, reads register 0x28
#[embassy_executor::task]
async fn spi_sensor_task(
    mut spi: Spi<'static, embassy_stm32::peripherals::SPI1,
                 embassy_stm32::peripherals::DMA2_CH3,
                 embassy_stm32::peripherals::DMA2_CH2>,
    mut cs: Output<'static>,
) {
    let mut ticker = Ticker::every(Duration::from_millis(10)); // 100 Hz

    loop {
        match sensor_read_register(&mut spi, &mut cs, 0x28).await {
            Ok(value) => {
                let raw: i16 = value as i16;
                // Process raw sensor value...
                let _ = raw;
            }
            Err(_) => {
                // Handle error — e.g., log or increment error counter
            }
        }

        ticker.next().await;
    }
}

#[embassy_executor::main]
async fn main(spawner: Spawner) {
    let p = embassy_stm32::init(Default::default());

    // Configure SPI1 at 8 MHz, Mode 0
    let mut spi_config = Config::default();
    spi_config.frequency = Hertz(8_000_000);

    let spi = Spi::new(
        p.SPI1,
        p.PA5,  // SCLK
        p.PA7,  // MOSI
        p.PA6,  // MISO
        p.DMA2_CH3,
        p.DMA2_CH2,
        spi_config,
    );

    let cs = Output::new(p.PA4, Level::High, Speed::VeryHigh);

    spawner.spawn(spi_sensor_task(spi, cs)).unwrap();
}
```

### Shared SPI Bus with Mutex in Rust

Rust's `embassy_sync::mutex::Mutex` provides async-aware mutual exclusion for sharing the SPI bus across multiple tasks safely.

```rust
// 56_freertos_spi_shared_bus.rs
// Sharing an SPI bus between multiple Embassy tasks using an async Mutex.

#![no_std]
#![no_main]

use embassy_executor::Spawner;
use embassy_stm32::spi::Spi;
use embassy_sync::{blocking_mutex::raw::CriticalSectionRawMutex, mutex::Mutex};
use embassy_time::{Duration, Timer};
use embedded_hal_async::spi::SpiBus;
use static_cell::StaticCell;

/// Type alias for the shared bus — CriticalSection mutex wrapping the SPI driver.
type SharedSpi = Mutex<CriticalSectionRawMutex,
                       Spi<'static,
                           embassy_stm32::peripherals::SPI1,
                           embassy_stm32::peripherals::DMA2_CH3,
                           embassy_stm32::peripherals::DMA2_CH2>>;

/// Static storage for the shared bus (lives for 'static lifetime)
static SPI_BUS: StaticCell<SharedSpi> = StaticCell::new();

/// Sensor task: acquires bus lock, transfers, releases automatically.
#[embassy_executor::task]
async fn sensor_task(bus: &'static SharedSpi) {
    loop {
        {
            // Lock is held only for the duration of this scope
            let mut spi = bus.lock().await;

            let tx = [0x80u8 | 0x28, 0x00];
            let mut rx = [0u8; 2];

            // RAII: lock released when `spi` drops at end of block
            let _ = spi.transfer(&mut rx, &tx).await;

            let value = rx[1];
            let _ = value; // process...
        }

        Timer::after(Duration::from_millis(10)).await;
    }
}

/// Display task: shares the same bus, at a different rate.
#[embassy_executor::task]
async fn display_task(bus: &'static SharedSpi) {
    let mut frame = [0u8; 128];
    frame[0] = 0x40; // write data command

    loop {
        {
            let mut spi = bus.lock().await;
            let mut discard = [0u8; 128];
            let _ = spi.transfer(&mut discard, &frame).await;
        }

        Timer::after(Duration::from_millis(33)).await; // ~30 fps
    }
}

#[embassy_executor::main]
async fn main(spawner: Spawner) {
    let p = embassy_stm32::init(Default::default());

    let spi = Spi::new(
        p.SPI1, p.PA5, p.PA7, p.PA6,
        p.DMA2_CH3, p.DMA2_CH2,
        Default::default(),
    );

    // Initialize the static shared bus
    let bus = SPI_BUS.init(Mutex::new(spi));

    spawner.spawn(sensor_task(bus)).unwrap();
    spawner.spawn(display_task(bus)).unwrap();
}
```

### SPI Message Queue Pattern in Rust

A channel-based approach mirrors the FreeRTOS queue pattern. An `SpiManager` task owns the hardware and processes requests from other tasks.

```rust
// 56_freertos_spi_channel.rs
// SPI manager task using Embassy channel — mirrors FreeRTOS queue pattern.

#![no_std]
#![no_main]

use embassy_executor::Spawner;
use embassy_stm32::{
    gpio::{Level, Output, Speed},
    spi::Spi,
};
use embassy_sync::{
    blocking_mutex::raw::CriticalSectionRawMutex,
    channel::{Channel, Sender, Receiver},
    signal::Signal,
};
use embassy_time::{Duration, Timer};
use embedded_hal_async::spi::SpiBus;
use static_cell::StaticCell;

/// Maximum number of pending SPI requests in the channel
const QUEUE_DEPTH: usize = 8;

/// A request sent to the SPI manager task.
/// Uses a Signal for the response so the caller can await it.
pub struct SpiRequest {
    pub tx:       [u8; 4],
    pub len:      usize,
    pub device:   u8,
    pub response: &'static Signal<CriticalSectionRawMutex, SpiResponse>,
}

/// The response returned to the caller.
pub struct SpiResponse {
    pub rx:  [u8; 4],
    pub ok:  bool,
}

type SpiChannel = Channel<CriticalSectionRawMutex, SpiRequest, QUEUE_DEPTH>;

static CHANNEL: StaticCell<SpiChannel> = StaticCell::new();

/// SPI Manager Task — sole owner of the SPI hardware.
#[embassy_executor::task]
async fn spi_manager_task(
    mut spi: Spi<'static,
                 embassy_stm32::peripherals::SPI1,
                 embassy_stm32::peripherals::DMA2_CH3,
                 embassy_stm32::peripherals::DMA2_CH2>,
    mut cs0: Output<'static>,
    mut cs1: Output<'static>,
    rx: Receiver<'static, CriticalSectionRawMutex, SpiRequest, QUEUE_DEPTH>,
) {
    loop {
        // Block until a request arrives
        let req = rx.receive().await;

        // Select the correct CS line
        let cs: &mut Output = match req.device {
            0 => &mut cs0,
            _ => &mut cs1,
        };

        let mut rx_buf = [0u8; 4];

        cs.set_low();
        let ok = spi
            .transfer(&mut rx_buf[..req.len], &req.tx[..req.len])
            .await
            .is_ok();
        cs.set_high();

        // Signal caller with result
        req.response.signal(SpiResponse { rx: rx_buf, ok });
    }
}

/// Example client task that submits SPI requests via the channel.
#[embassy_executor::task]
async fn client_task(
    tx: Sender<'static, CriticalSectionRawMutex, SpiRequest, QUEUE_DEPTH>,
) {
    // Each task needs its own static Signal for the response
    static RESP_SIGNAL: StaticCell<Signal<CriticalSectionRawMutex, SpiResponse>>
        = StaticCell::new();
    let signal = RESP_SIGNAL.init(Signal::new());

    loop {
        let req = SpiRequest {
            tx:       [0x80 | 0x28, 0x00, 0x00, 0x00],
            len:      2,
            device:   0,
            response: signal,
        };

        // Submit request (async, waits if queue is full)
        tx.send(req).await;

        // Await the response
        let resp = signal.wait().await;

        if resp.ok {
            let value = resp.rx[1];
            let _ = value; // process sensor value...
        }

        Timer::after(Duration::from_millis(10)).await;
    }
}

#[embassy_executor::main]
async fn main(spawner: Spawner) {
    let p = embassy_stm32::init(Default::default());

    let channel = CHANNEL.init(Channel::new());
    let (tx, rx) = (channel.sender(), channel.receiver());

    let spi = Spi::new(
        p.SPI1, p.PA5, p.PA7, p.PA6,
        p.DMA2_CH3, p.DMA2_CH2,
        Default::default(),
    );

    let cs0 = Output::new(p.PA4, Level::High, Speed::VeryHigh);
    let cs1 = Output::new(p.PB0, Level::High, Speed::VeryHigh);

    spawner.spawn(spi_manager_task(spi, cs0, cs1, rx)).unwrap();
    spawner.spawn(client_task(tx)).unwrap();
}
```

## Comparison: C/C++ vs Rust

| Aspect | C/C++ (FreeRTOS) | Rust (Embassy / embedded-hal) |
|---|---|---|
| **Bus safety** | Runtime: mutex must be manually acquired/released | Compile-time: ownership prevents concurrent access |
| **Memory** | Manual buffer management; stack corruption possible | Ownership enforced at compile time; no dangling pointers |
| **Synchronization** | Semaphores, queues, task notifications | Async/await, Mutex, Channel, Signal |
| **ISR integration** | `xSemaphoreGiveFromISR` / `vTaskNotifyGiveFromISR` | Async DMA via hardware abstraction layer |
| **Error handling** | Return codes (`pdTRUE`/`pdFALSE`) | `Result<T, E>` enforced by type system |
| **Portability** | FreeRTOS port required per platform | `embedded-hal` traits abstract hardware |
| **Code size** | Small, predictable | Slightly larger due to async state machines |
| **Learning curve** | Lower — familiar C idioms | Higher — ownership/lifetimes/async model |

## Common Pitfalls and Best Practices

### 1. Never Access SPI from ISR Context Directly
SPI transactions involve blocking (mutex, semaphore). ISR callbacks should only signal tasks via `xSemaphoreGiveFromISR` or `vTaskNotifyGiveFromISR`, never perform the transfer themselves.

### 2. Always Scope the Mutex to the Full Transaction
The mutex must be held from CS assert through CS deassert. Releasing it between the start of a transfer and CS deassert allows another task to corrupt the transaction.

```c
// ❌ WRONG: mutex released before transfer completes
xSemaphoreTake(s_busMutex, ...);
SPI_HAL_AssertCS();
xSemaphoreGive(s_busMutex);      // Bug: another task can interleave here!
SPI_HAL_TransmitReceive_IT(...);
```

```c
// ✅ CORRECT: mutex covers the full transaction
xSemaphoreTake(s_busMutex, ...);
SPI_HAL_AssertCS();
SPI_HAL_TransmitReceive_IT(...);
xSemaphoreTake(s_xferDone, ...);
SPI_HAL_DeassertCS();
xSemaphoreGive(s_busMutex);
```

### 3. Use `vTaskDelayUntil` for Periodic Sensor Tasks
`vTaskDelay` introduces drift over time. `vTaskDelayUntil` maintains a precise period regardless of how long the SPI transfer took.

### 4. Size Stacks Generously for SPI Tasks
SPI manager tasks handle buffers on the stack. Insufficient stack depth causes hard-to-debug stack overflow corruptions. Use `uxTaskGetStackHighWaterMark()` to profile.

### 5. Enable Priority Inheritance on Mutexes
Use `xSemaphoreCreateMutex()` (not `xSemaphoreCreateBinary()`) for bus arbitration. Mutexes support priority inheritance; binary semaphores do not.

### 6. Handle Transfer Timeouts Gracefully
Always pass a finite timeout to `xSemaphoreTake`. An infinite wait masks hardware failures (stuck CS, disconnected MISO). Log and recover.

### 7. In Rust, Prefer `lock()` Scope Minimization
Hold the async `Mutex` lock only for the transfer itself. Long-held locks block other tasks during their `lock().await` calls.

## Summary

Implementing SPI communication within FreeRTOS task architecture requires balancing hardware access safety, timing predictability, and CPU efficiency. The key takeaways are:

**Core Principles:**
- The SPI bus is a shared resource — always protect it with a mutex or channel-based ownership pattern.
- Avoid polling; use interrupt/DMA-driven transfers with semaphores or task notifications so the CPU can run other tasks during transfers.
- Always wrap a complete transaction (CS assert → transfer → CS deassert) within a single mutex hold to prevent interleaving.
- Use `vTaskDelayUntil` for precise periodic task scheduling to prevent drift.
- Size task stacks generously and profile with `uxTaskGetStackHighWaterMark`.

**Pattern Selection Guide:**

| Scenario | Recommended Pattern |
|---|---|
| Single task, one device | Direct transfer + binary semaphore for ISR sync |
| Multiple tasks, multiple devices | Mutex protecting full transaction |
| Decoupled producer/consumer | SPI Manager Task with request queue |
| High-throughput, DMA required | DMA + task notification (`ulTaskNotifyTake`) |
| Rust async environment | Embassy `Mutex<Spi>` or `Channel`-based manager |

**C/C++** with FreeRTOS gives fine-grained control and low overhead but requires careful manual management of mutexes, semaphores, and buffer lifetimes. **Rust** with Embassy provides compile-time safety guarantees that eliminate entire classes of concurrency bugs at the cost of a steeper learning curve.

Together, these patterns form the foundation for reliable, real-time SPI communication in multitasked embedded systems.
