# SPI (Serial Peripheral Interface) - Comprehensive Guide

## Overview

SPI (Serial Peripheral Interface) is a synchronous serial communication protocol widely used for short-distance communication between microcontrollers and peripheral devices like sensors, displays, SD cards, and ADCs. It was developed by Motorola and has become one of the most common embedded communication standards.

## Core Concepts

### Bus Architecture

SPI uses a master-slave architecture with four main signal lines:

- **MOSI** (Master Out Slave In): Data line from master to slave
- **MISO** (Master In Slave Out): Data line from slave to master
- **SCK** (Serial Clock): Clock signal generated by master
- **SS/CS** (Slave Select/Chip Select): Slave selection line (active low)

### Key Characteristics

- **Full-duplex communication**: Simultaneous bidirectional data transfer
- **Synchronous**: Clock signal coordinates data transmission
- **No fixed protocol**: Flexible but requires prior knowledge of device specifications
- **High speed**: Can operate at MHz frequencies (commonly 1-50 MHz)
- **Short distance**: Typically used for on-board communication

### Operating Modes

SPI supports four modes based on Clock Polarity (CPOL) and Clock Phase (CPHA):

| Mode | CPOL | CPHA | Clock Idle | Data Sampling |
|------|------|------|------------|---------------|
| 0    | 0    | 0    | Low        | Leading edge  |
| 1    | 0    | 1    | Low        | Trailing edge |
| 2    | 1    | 0    | High       | Leading edge  |
| 3    | 1    | 1    | High       | Trailing edge |

## C/C++ Implementation Examples

### Basic SPI Communication (Bare Metal - AVR)

```c
#include <avr/io.h>
#include <stdint.h>

// SPI initialization as master
void SPI_MasterInit(void) {
    // Set MOSI, SCK, and SS as output
    DDRB |= (1 << PB3) | (1 << PB5) | (1 << PB2);
    
    // Set MISO as input
    DDRB &= ~(1 << PB4);
    
    // Enable SPI, Master mode, clock rate fck/16
    SPCR = (1 << SPE) | (1 << MSTR) | (1 << SPR0);
}

// Transmit and receive a byte
uint8_t SPI_Transceive(uint8_t data) {
    // Load data into buffer
    SPDR = data;
    
    // Wait for transmission complete
    while (!(SPSR & (1 << SPIF)));
    
    // Return received data
    return SPDR;
}

// Write to specific register
void SPI_WriteRegister(uint8_t reg, uint8_t value) {
    // Pull CS low
    PORTB &= ~(1 << PB2);
    
    // Send register address
    SPI_Transceive(reg);
    
    // Send data
    SPI_Transceive(value);
    
    // Pull CS high
    PORTB |= (1 << PB2);
}

// Read from specific register
uint8_t SPI_ReadRegister(uint8_t reg) {
    uint8_t result;
    
    // Pull CS low
    PORTB &= ~(1 << PB2);
    
    // Send register address with read bit
    SPI_Transceive(reg | 0x80);
    
    // Read data
    result = SPI_Transceive(0x00);
    
    // Pull CS high
    PORTB |= (1 << PB2);
    
    return result;
}
```

### Linux userspace SPI (C)

```c
#include <stdio.h>
#include <stdint.h>
#include <fcntl.h>
#include <unistd.h>
#include <sys/ioctl.h>
#include <linux/spi/spidev.h>

typedef struct {
    int fd;
    uint8_t mode;
    uint8_t bits;
    uint32_t speed;
} SPI_Device;

// Initialize SPI device
int SPI_Init(SPI_Device *dev, const char *device) {
    dev->fd = open(device, O_RDWR);
    if (dev->fd < 0) {
        perror("Error opening SPI device");
        return -1;
    }
    
    // Set SPI mode
    dev->mode = SPI_MODE_0;
    if (ioctl(dev->fd, SPI_IOC_WR_MODE, &dev->mode) < 0) {
        perror("Error setting SPI mode");
        return -1;
    }
    
    // Set bits per word
    dev->bits = 8;
    if (ioctl(dev->fd, SPI_IOC_WR_BITS_PER_WORD, &dev->bits) < 0) {
        perror("Error setting bits per word");
        return -1;
    }
    
    // Set max speed
    dev->speed = 1000000; // 1 MHz
    if (ioctl(dev->fd, SPI_IOC_WR_MAX_SPEED_HZ, &dev->speed) < 0) {
        perror("Error setting speed");
        return -1;
    }
    
    return 0;
}

// Transfer data
int SPI_Transfer(SPI_Device *dev, uint8_t *tx_buf, uint8_t *rx_buf, size_t len) {
    struct spi_ioc_transfer transfer = {
        .tx_buf = (unsigned long)tx_buf,
        .rx_buf = (unsigned long)rx_buf,
        .len = len,
        .speed_hz = dev->speed,
        .bits_per_word = dev->bits,
    };
    
    if (ioctl(dev->fd, SPI_IOC_MESSAGE(1), &transfer) < 0) {
        perror("SPI transfer failed");
        return -1;
    }
    
    return 0;
}

// Example usage
int main() {
    SPI_Device spi;
    uint8_t tx_data[3] = {0x01, 0x02, 0x03};
    uint8_t rx_data[3] = {0};
    
    if (SPI_Init(&spi, "/dev/spidev0.0") < 0) {
        return -1;
    }
    
    if (SPI_Transfer(&spi, tx_data, rx_data, 3) == 0) {
        printf("Received: 0x%02X 0x%02X 0x%02X\n", 
               rx_data[0], rx_data[1], rx_data[2]);
    }
    
    close(spi.fd);
    return 0;
}
```

### C++ Object-Oriented Approach

```cpp
#include <cstdint>
#include <vector>
#include <stdexcept>

class SPIDevice {
private:
    int fd_;
    uint8_t mode_;
    uint32_t speed_;
    uint8_t bits_per_word_;

public:
    SPIDevice(const char* device, uint8_t mode = 0, 
              uint32_t speed = 1000000, uint8_t bits = 8)
        : mode_(mode), speed_(speed), bits_per_word_(bits) {
        
        fd_ = open(device, O_RDWR);
        if (fd_ < 0) {
            throw std::runtime_error("Failed to open SPI device");
        }
        
        configure();
    }
    
    ~SPIDevice() {
        if (fd_ >= 0) {
            close(fd_);
        }
    }
    
    void configure() {
        if (ioctl(fd_, SPI_IOC_WR_MODE, &mode_) < 0 ||
            ioctl(fd_, SPI_IOC_WR_BITS_PER_WORD, &bits_per_word_) < 0 ||
            ioctl(fd_, SPI_IOC_WR_MAX_SPEED_HZ, &speed_) < 0) {
            throw std::runtime_error("SPI configuration failed");
        }
    }
    
    std::vector<uint8_t> transfer(const std::vector<uint8_t>& tx_data) {
        std::vector<uint8_t> rx_data(tx_data.size());
        
        struct spi_ioc_transfer transfer = {};
        transfer.tx_buf = reinterpret_cast<uint64_t>(tx_data.data());
        transfer.rx_buf = reinterpret_cast<uint64_t>(rx_data.data());
        transfer.len = tx_data.size();
        transfer.speed_hz = speed_;
        transfer.bits_per_word = bits_per_word_;
        
        if (ioctl(fd_, SPI_IOC_MESSAGE(1), &transfer) < 0) {
            throw std::runtime_error("SPI transfer failed");
        }
        
        return rx_data;
    }
    
    void setSpeed(uint32_t speed) {
        speed_ = speed;
        configure();
    }
};

// Example: Interfacing with an accelerometer
class ADXL345 {
private:
    SPIDevice spi_;
    static constexpr uint8_t READ_BIT = 0x80;
    static constexpr uint8_t MULTI_BYTE = 0x40;
    
public:
    ADXL345(const char* device) : spi_(device, 3, 5000000) {
        // SPI mode 3, 5 MHz
    }
    
    uint8_t readRegister(uint8_t reg) {
        std::vector<uint8_t> tx = {static_cast<uint8_t>(reg | READ_BIT), 0x00};
        auto rx = spi_.transfer(tx);
        return rx[1];
    }
    
    void writeRegister(uint8_t reg, uint8_t value) {
        std::vector<uint8_t> tx = {reg, value};
        spi_.transfer(tx);
    }
    
    struct AccelData {
        int16_t x, y, z;
    };
    
    AccelData readAcceleration() {
        std::vector<uint8_t> tx = {
            static_cast<uint8_t>(0x32 | READ_BIT | MULTI_BYTE),
            0, 0, 0, 0, 0, 0
        };
        auto rx = spi_.transfer(tx);
        
        AccelData data;
        data.x = static_cast<int16_t>((rx[2] << 8) | rx[1]);
        data.y = static_cast<int16_t>((rx[4] << 8) | rx[3]);
        data.z = static_cast<int16_t>((rx[6] << 8) | rx[5]);
        
        return data;
    }
};
```

## Rust Implementation Examples

### Embedded HAL (embedded-hal crate)

```rust
use embedded_hal::spi::{Mode, Phase, Polarity};
use embedded_hal::digital::v2::OutputPin;

// SPI Mode 0
pub const MODE_0: Mode = Mode {
    polarity: Polarity::IdleLow,
    phase: Phase::CaptureOnFirstTransition,
};

/// Generic SPI device wrapper
pub struct SpiDevice<SPI, CS> {
    spi: SPI,
    cs: CS,
}

impl<SPI, CS, E> SpiDevice<SPI, CS>
where
    SPI: embedded_hal::blocking::spi::Transfer<u8, Error = E>
        + embedded_hal::blocking::spi::Write<u8, Error = E>,
    CS: OutputPin,
{
    pub fn new(spi: SPI, cs: CS) -> Self {
        Self { spi, cs }
    }
    
    /// Write a single byte
    pub fn write_byte(&mut self, byte: u8) -> Result<(), E> {
        self.cs.set_low().ok();
        let result = self.spi.write(&[byte]);
        self.cs.set_high().ok();
        result
    }
    
    /// Transfer data (full duplex)
    pub fn transfer(&mut self, buffer: &mut [u8]) -> Result<(), E> {
        self.cs.set_low().ok();
        let result = self.spi.transfer(buffer);
        self.cs.set_high().ok();
        result.map(|_| ())
    }
    
    /// Write to a register
    pub fn write_register(&mut self, reg: u8, value: u8) -> Result<(), E> {
        self.cs.set_low().ok();
        self.spi.write(&[reg, value])?;
        self.cs.set_high().ok();
        Ok(())
    }
    
    /// Read from a register
    pub fn read_register(&mut self, reg: u8) -> Result<u8, E> {
        self.cs.set_low().ok();
        let mut buffer = [reg | 0x80, 0x00];
        self.spi.transfer(&mut buffer)?;
        self.cs.set_high().ok();
        Ok(buffer[1])
    }
}

// Example for STM32 microcontroller
#[cfg(feature = "stm32")]
use stm32f4xx_hal::{prelude::*, spi::Spi};

pub fn init_spi_stm32(
    spi: stm32f4xx_hal::pac::SPI1,
    clocks: &stm32f4xx_hal::rcc::Clocks,
    sck: stm32f4xx_hal::gpio::gpioa::PA5<stm32f4xx_hal::gpio::Alternate<5>>,
    miso: stm32f4xx_hal::gpio::gpioa::PA6<stm32f4xx_hal::gpio::Alternate<5>>,
    mosi: stm32f4xx_hal::gpio::gpioa::PA7<stm32f4xx_hal::gpio::Alternate<5>>,
) -> Spi<stm32f4xx_hal::pac::SPI1> {
    Spi::new(
        spi,
        (sck, miso, mosi),
        MODE_0,
        1.MHz(),
        clocks,
    )
}
```

### Linux spidev (using spidev crate)

```rust
use spidev::{Spidev, SpidevOptions, SpidevTransfer, SpiModeFlags};
use std::io;

pub struct LinuxSpiDevice {
    spi: Spidev,
}

impl LinuxSpiDevice {
    /// Create a new SPI device
    pub fn new(device: &str, speed_hz: u32, mode: u8) -> io::Result<Self> {
        let mut spi = Spidev::open(device)?;
        
        let options = SpidevOptions::new()
            .bits_per_word(8)
            .max_speed_hz(speed_hz)
            .mode(SpiModeFlags::from_bits_truncate(mode))
            .build();
        
        spi.configure(&options)?;
        
        Ok(Self { spi })
    }
    
    /// Perform a full-duplex transfer
    pub fn transfer(&mut self, tx_buf: &[u8]) -> io::Result<Vec<u8>> {
        let mut rx_buf = vec![0u8; tx_buf.len()];
        
        let mut transfer = SpidevTransfer::read_write(tx_buf, &mut rx_buf);
        self.spi.transfer(&mut transfer)?;
        
        Ok(rx_buf)
    }
    
    /// Write data only
    pub fn write(&mut self, data: &[u8]) -> io::Result<()> {
        let mut transfer = SpidevTransfer::write(data);
        self.spi.transfer(&mut transfer)?;
        Ok(())
    }
    
    /// Read data only
    pub fn read(&mut self, len: usize) -> io::Result<Vec<u8>> {
        let mut buffer = vec![0u8; len];
        let mut transfer = SpidevTransfer::read(&mut buffer);
        self.spi.transfer(&mut transfer)?;
        Ok(buffer)
    }
}

/// Example: MAX31855 thermocouple interface
pub struct MAX31855 {
    spi: LinuxSpiDevice,
}

impl MAX31855 {
    pub fn new(device: &str) -> io::Result<Self> {
        // MAX31855 uses SPI Mode 0, max 5 MHz
        let spi = LinuxSpiDevice::new(device, 5_000_000, 0)?;
        Ok(Self { spi })
    }
    
    pub fn read_temperature(&mut self) -> io::Result<f32> {
        let data = self.spi.read(4)?;
        
        // Check for fault
        if data[3] & 0x01 != 0 {
            return Err(io::Error::new(
                io::ErrorKind::Other,
                "Thermocouple fault detected"
            ));
        }
        
        // Extract temperature (14-bit, signed)
        let raw = ((data[0] as i16) << 6) | ((data[1] as i16) >> 2);
        let temp = (raw as f32) * 0.25;
        
        Ok(temp)
    }
}

// Usage example
fn main() -> io::Result<()> {
    let mut sensor = MAX31855::new("/dev/spidev0.0")?;
    
    loop {
        match sensor.read_temperature() {
            Ok(temp) => println!("Temperature: {:.2}°C", temp),
            Err(e) => eprintln!("Error reading temperature: {}", e),
        }
        
        std::thread::sleep(std::time::Duration::from_secs(1));
    }
}
```

### Async Rust Implementation

```rust
use embedded_hal_async::spi::SpiBus;
use embassy_time::{Duration, Timer};

/// Async SPI device driver
pub struct AsyncSpiDevice<SPI, CS> {
    spi: SPI,
    cs: CS,
}

impl<SPI, CS> AsyncSpiDevice<SPI, CS>
where
    SPI: SpiBus,
    CS: embedded_hal_async::digital::Wait,
{
    pub fn new(spi: SPI, cs: CS) -> Self {
        Self { spi, cs }
    }
    
    /// Async write operation
    pub async fn write_async(&mut self, data: &[u8]) -> Result<(), SPI::Error> {
        self.cs.set_low().await.ok();
        let result = self.spi.write(data).await;
        Timer::after(Duration::from_micros(1)).await;
        self.cs.set_high().await.ok();
        result
    }
    
    /// Async read operation
    pub async fn read_async(&mut self, buffer: &mut [u8]) -> Result<(), SPI::Error> {
        self.cs.set_low().await.ok();
        let result = self.spi.read(buffer).await;
        Timer::after(Duration::from_micros(1)).await;
        self.cs.set_high().await.ok();
        result
    }
    
    /// Async transfer operation
    pub async fn transfer_async(&mut self, 
                                  read: &mut [u8], 
                                  write: &[u8]) -> Result<(), SPI::Error> {
        self.cs.set_low().await.ok();
        let result = self.spi.transfer(read, write).await;
        Timer::after(Duration::from_micros(1)).await;
        self.cs.set_high().await.ok();
        result
    }
}
```

## Protocol Decoders

Protocol decoders are software tools that interpret captured SPI transactions from logic analyzers or oscilloscopes. They parse the raw signal data and present it in human-readable format.

### Common Protocol Decoder Tools

1. **Sigrok/PulseView** - Open-source logic analyzer software
2. **Saleae Logic** - Commercial logic analyzer with excellent SPI decoding
3. **DSView** - Open-source alternative for DreamSourceLab analyzers
4. **Custom Python decoders** - For specialized protocols

### Creating a Simple SPI Decoder (Python)

```python
class SPIDecoder:
    def __init__(self, mode=0):
        self.mode = mode
        self.cpol = (mode >> 1) & 1
        self.cpha = mode & 1
        
    def decode_transaction(self, clock_edges, mosi_data, miso_data, cs_states):
        """
        Decode SPI transaction from captured data
        
        clock_edges: list of clock transition timestamps
        mosi_data: list of MOSI states at each clock edge
        miso_data: list of MISO states at each clock edge
        cs_states: list of CS states
        """
        transactions = []
        current_byte_mosi = 0
        current_byte_miso = 0
        bit_count = 0
        
        for i, (clk_time, mosi_bit, miso_bit, cs) in enumerate(
            zip(clock_edges, mosi_data, miso_data, cs_states)):
            
            # Check if CS is active (low)
            if cs == 1:
                if bit_count > 0:
                    # Transaction ended
                    transactions.append({
                        'mosi': current_byte_mosi,
                        'miso': current_byte_miso,
                        'timestamp': clk_time
                    })
                current_byte_mosi = 0
                current_byte_miso = 0
                bit_count = 0
                continue
            
            # Sample on correct edge based on mode
            if self._should_sample(i):
                current_byte_mosi = (current_byte_mosi << 1) | mosi_bit
                current_byte_miso = (current_byte_miso << 1) | miso_bit
                bit_count += 1
                
                if bit_count == 8:
                    transactions.append({
                        'mosi': current_byte_mosi,
                        'miso': current_byte_miso,
                        'timestamp': clk_time
                    })
                    current_byte_mosi = 0
                    current_byte_miso = 0
                    bit_count = 0
        
        return transactions
    
    def _should_sample(self, edge_index):
        """Determine if this clock edge should be sampled based on SPI mode"""
        is_rising = edge_index % 2 == (0 if self.cpol == 0 else 1)
        
        if self.cpha == 0:
            return is_rising
        else:
            return not is_rising
    
    def format_transaction(self, transaction):
        """Format transaction for display"""
        return f"Time: {transaction['timestamp']:.6f}s | " \
               f"MOSI: 0x{transaction['mosi']:02X} | " \
               f"MISO: 0x{transaction['miso']:02X}"

# Example usage
decoder = SPIDecoder(mode=0)
# Assuming you have captured data from a logic analyzer
transactions = decoder.decode_transaction(clk_edges, mosi, miso, cs)
for tx in transactions:
    print(decoder.format_transaction(tx))
```

### Practical Decoder Features

**Key capabilities of SPI protocol decoders:**

- **Bit-level visualization**: Display individual clock cycles and data bits
- **Byte grouping**: Automatically group 8/16/32 bits into meaningful values
- **Register interpretation**: Decode specific device registers based on protocol
- **Timing analysis**: Measure setup/hold times, clock frequency, transaction duration
- **Error detection**: Identify protocol violations or timing issues
- **Export capabilities**: Save decoded data to CSV, JSON, or other formats
- **Trigger support**: Capture specific patterns or register accesses

## Summary

**SPI (Serial Peripheral Interface)** is a versatile, high-speed synchronous communication protocol essential for embedded systems. Its key strengths include full-duplex operation, simplicity, and high throughput, making it ideal for interfacing with sensors, memory devices, and displays.

**Key Takeaways:**

- **Four-wire interface** (MOSI, MISO, SCK, CS) with master-slave architecture
- **Four operating modes** determined by clock polarity and phase
- **Implementation varies** across platforms: bare-metal requires direct register manipulation, while Linux provides spidev interface
- **C/C++ implementations** range from low-level AVR programming to object-oriented Linux userspace drivers
- **Rust provides** both embedded HAL abstractions for bare-metal and spidev bindings for Linux
- **Protocol decoders** are invaluable for debugging, allowing visualization and interpretation of SPI transactions captured by logic analyzers

When working with SPI, always verify the target device's timing requirements, mode settings, and maximum clock speed to ensure reliable communication. Protocol decoders become essential tools for troubleshooting complex interactions and validating protocol implementations.
