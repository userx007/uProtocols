## 1. Introduction to SPI <a name="introduction"></a>

The **Serial Peripheral Interface (SPI)** is a synchronous, full-duplex, master-slave communication protocol widely used to connect microcontrollers and embedded SoCs to peripherals such as sensors, displays, ADCs, DACs, flash memory, and RF transceivers.

SPI is favoured in embedded Linux systems because:

- It achieves high data rates (typically 1–100 MHz)
- The protocol is simple and hardware-cheap (no addressing, no ACK)
- Linux provides a mature, well-structured SPI subsystem
- Both kernel-space drivers and userspace (`spidev`) access are supported

---

## 2. SPI Protocol Fundamentals <a name="protocol"></a>

### Signal Lines

| Signal | Direction       | Description                                      |
|--------|-----------------|--------------------------------------------------|
| SCLK   | Master → Slave  | Serial clock generated by the master             |
| MOSI   | Master → Slave  | Master Out Slave In — data from master to slave  |
| MISO   | Slave → Master  | Master In Slave Out — data from slave to master  |
| CS/SS  | Master → Slave  | Chip Select (active low) — selects the slave     |

### Clock Modes (CPOL / CPHA)

| Mode | CPOL | CPHA | Clock idle | Data sampled    |
|------|------|------|------------|-----------------|
| 0    | 0    | 0    | Low        | Rising edge     |
| 1    | 0    | 1    | Low        | Falling edge    |
| 2    | 1    | 0    | High       | Falling edge    |
| 3    | 1    | 1    | High       | Rising edge     |

Choosing the wrong mode is one of the most common bugs in SPI driver development. Always check the peripheral's datasheet.

### Transfer Model

SPI is inherently full-duplex: every clock cycle simultaneously shifts one bit out (MOSI) and one bit in (MISO). A "write-only" transfer still receives dummy bytes; a "read-only" transfer still sends dummy bytes (usually `0x00` or `0xFF`).

## 3. Linux SPI Subsystem Architecture <a name="architecture"></a>

```
+-------------------+       +-------------------+
|  SPI Device       |       |  SPI Device       |
|  Driver (client)  |       |  Driver (client)  |
+--------+----------+       +--------+----------+
         |                           |
         +----------+----------------+
                    |
         +----------v----------+
         |   SPI Core          |  (drivers/spi/spi.c)
         |  spi_register_driver|
         | spi_sync / spi_async|
         +----------+----------+
                    |
         +----------v----------+
         |  SPI Bus Controller |  (SPI Master/Controller driver)
         |  e.g. spi-bcm2835   |  e.g. spi-imx, spi-pl022
         +----------+----------+
                    |
              Hardware SPI
```

### Key Kernel Structures

| Structure             | Purpose                                              |
|-----------------------|------------------------------------------------------|
| `struct spi_device`   | Represents one SPI slave device                      |
| `struct spi_driver`   | The driver bound to a device (probe/remove/id_table) |
| `struct spi_transfer` | One segment of a SPI transfer (tx/rx buffer + len)   |
| `struct spi_message`  | A list of `spi_transfer` objects for one CS assertion|
| `struct spi_controller` | Represents the SPI bus master hardware             |

## 4. Device Tree Configuration <a name="device-tree"></a>

SPI devices are described in the Device Tree (DTS). The controller node already exists for your SoC; you add your device as a child node.

```dts
/* Example: custom sensor on SPI1, chip-select 0, Raspberry Pi / i.MX style */

&spi1 {
    status = "okay";
    #address-cells = <1>;
    #size-cells = <0>;

    my_sensor: my_sensor@0 {
        compatible = "myvendor,my-sensor";
        reg = <0>;                          /* chip-select 0 */
        spi-max-frequency = <10000000>;     /* 10 MHz */
        spi-cpol;                           /* CPOL=1  (omit for CPOL=0) */
        spi-cpha;                           /* CPHA=1  (omit for CPHA=0) */
        spi-cs-high;                        /* CS active HIGH (rare)      */
        /* Custom properties for your driver */
        myvendor,reset-gpios = <&gpio 42 GPIO_ACTIVE_LOW>;
        myvendor,int-gpios   = <&gpio 43 GPIO_ACTIVE_HIGH>;
    };
};
```

> **Important:** The `compatible` string must exactly match the `.compatible` entry in your driver's `of_device_id` table.

## 5. C/C++ Kernel Driver Implementation <a name="c-driver"></a>

### 5.1 Minimal Driver Skeleton

```c
// SPDX-License-Identifier: GPL-2.0
/*
 * my_sensor.c - SPI driver for MyVendor Sensor
 */

#include <linux/module.h>
#include <linux/spi/spi.h>
#include <linux/of.h>
#include <linux/regmap.h>
#include <linux/iio/iio.h>
#include <linux/gpio/consumer.h>

#define MY_SENSOR_REG_WHOAMI    0x0F
#define MY_SENSOR_REG_CTRL1     0x20
#define MY_SENSOR_REG_DATA_OUT  0x28

#define MY_SENSOR_WHOAMI_VALUE  0x42
#define MY_SENSOR_CTRL1_ENABLE  0x0F   /* ODR=100Hz, all axes on */

struct my_sensor_data {
    struct spi_device   *spi;
    struct gpio_desc    *reset_gpio;
    struct gpio_desc    *int_gpio;
    int                  irq;
    /* DMA-safe transfer buffers — must not be on the stack! */
    u8                   tx_buf[2] ____cacheline_aligned;
    u8                   rx_buf[2] ____cacheline_aligned;
};

/* ------------------------------------------------------------------ */
/* Low-level register access                                           */
/* ------------------------------------------------------------------ */

/**
 * my_sensor_read_reg - Read a single 8-bit register
 * Returns 0 on success, negative errno on failure.
 * The value is placed in *val.
 */
static int my_sensor_read_reg(struct my_sensor_data *priv, u8 reg, u8 *val)
{
    struct spi_transfer xfers[2];
    struct spi_message  msg;
    int ret;

    memset(xfers, 0, sizeof(xfers));

    /* Many sensors use bit7=1 to indicate a read */
    priv->tx_buf[0] = reg | 0x80;
    priv->rx_buf[0] = 0;
    priv->rx_buf[1] = 0;

    /* Phase 1: send command byte */
    xfers[0].tx_buf = priv->tx_buf;
    xfers[0].len    = 1;

    /* Phase 2: receive data byte (CS held low between phases) */
    xfers[1].rx_buf = &priv->rx_buf[1];
    xfers[1].len    = 1;

    spi_message_init(&msg);
    spi_message_add_tail(&xfers[0], &msg);
    spi_message_add_tail(&xfers[1], &msg);

    ret = spi_sync(priv->spi, &msg);
    if (ret)
        return ret;

    *val = priv->rx_buf[1];
    return 0;
}

/**
 * my_sensor_write_reg - Write a single 8-bit register
 */
static int my_sensor_write_reg(struct my_sensor_data *priv, u8 reg, u8 val)
{
    /* Write both bytes in a single transfer for efficiency */
    priv->tx_buf[0] = reg & 0x7F;   /* bit7=0 → write */
    priv->tx_buf[1] = val;

    return spi_write(priv->spi, priv->tx_buf, 2);
}

/* ------------------------------------------------------------------ */
/* Burst read helper (e.g. 6 bytes of XYZ data)                        */
/* ------------------------------------------------------------------ */

static int my_sensor_burst_read(struct my_sensor_data *priv,
                                u8 start_reg, u8 *buf, size_t len)
{
    /* Allocate on heap — never put SPI buffers on the kernel stack   */
    u8 *cmd = kmalloc(1 + len, GFP_KERNEL | GFP_DMA);
    u8 *rxp;
    struct spi_transfer xfer;
    struct spi_message  msg;
    int ret;

    if (!cmd)
        return -ENOMEM;

    rxp = cmd + 1;  /* reuse the same allocation for rx */

    cmd[0] = start_reg | 0xC0; /* read + auto-increment (device specific) */

    memset(&xfer, 0, sizeof(xfer));
    xfer.tx_buf = cmd;
    xfer.rx_buf = cmd;          /* in-place full-duplex */
    xfer.len    = 1 + len;

    spi_message_init(&msg);
    spi_message_add_tail(&xfer, &msg);

    ret = spi_sync(priv->spi, &msg);
    if (!ret)
        memcpy(buf, rxp, len);

    kfree(cmd);
    return ret;
}

/* ------------------------------------------------------------------ */
/* Interrupt handler (data-ready)                                      */
/* ------------------------------------------------------------------ */

static irqreturn_t my_sensor_irq_handler(int irq, void *dev_id)
{
    struct my_sensor_data *priv = dev_id;
    u8 raw[6];
    s16 x, y, z;

    if (my_sensor_burst_read(priv, MY_SENSOR_REG_DATA_OUT, raw, 6))
        return IRQ_HANDLED;

    x = (s16)((raw[1] << 8) | raw[0]);
    y = (s16)((raw[3] << 8) | raw[2]);
    z = (s16)((raw[5] << 8) | raw[4]);

    dev_dbg(&priv->spi->dev, "XYZ: %d %d %d\n", x, y, z);
    /* Push to IIO buffer, wake a consumer, etc. */

    return IRQ_HANDLED;
}

/* ------------------------------------------------------------------ */
/* probe / remove                                                      */
/* ------------------------------------------------------------------ */

static int my_sensor_probe(struct spi_device *spi)
{
    struct my_sensor_data *priv;
    u8 whoami;
    int ret;

    /* Validate SPI settings the core may have already applied */
    spi->mode       = SPI_MODE_3;       /* CPOL=1, CPHA=1 */
    spi->bits_per_word = 8;
    ret = spi_setup(spi);
    if (ret < 0) {
        dev_err(&spi->dev, "spi_setup failed: %d\n", ret);
        return ret;
    }

    priv = devm_kzalloc(&spi->dev, sizeof(*priv), GFP_KERNEL);
    if (!priv)
        return -ENOMEM;

    priv->spi = spi;
    spi_set_drvdata(spi, priv);

    /* Optional: GPIO reset */
    priv->reset_gpio = devm_gpiod_get_optional(&spi->dev, "myvendor,reset",
                                                GPIOD_OUT_HIGH);
    if (IS_ERR(priv->reset_gpio))
        return PTR_ERR(priv->reset_gpio);

    if (priv->reset_gpio) {
        gpiod_set_value_cansleep(priv->reset_gpio, 0); /* assert reset */
        usleep_range(1000, 2000);
        gpiod_set_value_cansleep(priv->reset_gpio, 1); /* deassert */
        msleep(10);
    }

    /* Verify device identity */
    ret = my_sensor_read_reg(priv, MY_SENSOR_REG_WHOAMI, &whoami);
    if (ret) {
        dev_err(&spi->dev, "WHO_AM_I read failed: %d\n", ret);
        return ret;
    }
    if (whoami != MY_SENSOR_WHOAMI_VALUE) {
        dev_err(&spi->dev, "Unexpected WHO_AM_I: 0x%02x\n", whoami);
        return -ENODEV;
    }

    /* Enable the sensor */
    ret = my_sensor_write_reg(priv, MY_SENSOR_REG_CTRL1, MY_SENSOR_CTRL1_ENABLE);
    if (ret)
        return ret;

    /* Request interrupt if provided */
    if (spi->irq > 0) {
        priv->irq = spi->irq;
        ret = devm_request_threaded_irq(&spi->dev, priv->irq,
                                        NULL, my_sensor_irq_handler,
                                        IRQF_TRIGGER_RISING | IRQF_ONESHOT,
                                        "my_sensor", priv);
        if (ret) {
            dev_err(&spi->dev, "IRQ request failed: %d\n", ret);
            return ret;
        }
    }

    dev_info(&spi->dev, "my_sensor probed successfully (WHO_AM_I=0x%02x)\n",
             whoami);
    return 0;
}

static void my_sensor_remove(struct spi_device *spi)
{
    struct my_sensor_data *priv = spi_get_drvdata(spi);

    /* Disable the sensor */
    my_sensor_write_reg(priv, MY_SENSOR_REG_CTRL1, 0x00);
    dev_info(&spi->dev, "my_sensor removed\n");
    /* devm resources are freed automatically */
}

/* ------------------------------------------------------------------ */
/* Device matching tables                                              */
/* ------------------------------------------------------------------ */

static const struct of_device_id my_sensor_of_match[] = {
    { .compatible = "myvendor,my-sensor" },
    { }
};
MODULE_DEVICE_TABLE(of, my_sensor_of_match);

static const struct spi_device_id my_sensor_id[] = {
    { "my-sensor", 0 },
    { }
};
MODULE_DEVICE_TABLE(spi, my_sensor_id);

static struct spi_driver my_sensor_driver = {
    .driver = {
        .name           = "my-sensor",
        .of_match_table = my_sensor_of_match,
    },
    .probe    = my_sensor_probe,
    .remove   = my_sensor_remove,
    .id_table = my_sensor_id,
};

module_spi_driver(my_sensor_driver);

MODULE_AUTHOR("Your Name <you@example.com>");
MODULE_DESCRIPTION("SPI driver for MyVendor Sensor");
MODULE_LICENSE("GPL v2");
```

### 5.3 SPI Regmap Backend (alternative low-level API)

The `regmap` API abstracts register access and is preferred for devices with many registers, as it supports caching and bulk operations.

```c
#include <linux/regmap.h>

static const struct regmap_config my_sensor_regmap_config = {
    .reg_bits       = 8,
    .val_bits       = 8,
    .read_flag_mask = 0x80,    /* bit7=1 signals a read */
    .max_register   = 0x7F,
    .cache_type     = REGCACHE_RBTREE,
};

static int my_sensor_probe_regmap(struct spi_device *spi)
{
    struct regmap *regmap;
    unsigned int val;
    int ret;

    regmap = devm_regmap_init_spi(spi, &my_sensor_regmap_config);
    if (IS_ERR(regmap))
        return PTR_ERR(regmap);

    ret = regmap_read(regmap, MY_SENSOR_REG_WHOAMI, &val);
    if (ret)
        return ret;

    if (val != MY_SENSOR_WHOAMI_VALUE)
        return -ENODEV;

    ret = regmap_write(regmap, MY_SENSOR_REG_CTRL1, MY_SENSOR_CTRL1_ENABLE);
    if (ret)
        return ret;

    /* Bulk read example */
    u8 raw[6];
    ret = regmap_bulk_read(regmap, MY_SENSOR_REG_DATA_OUT, raw, sizeof(raw));
    if (ret)
        return ret;

    return 0;
}
```

### 6.2 Full Rust Driver

```rust
// SPDX-License-Identifier: GPL-2.0
//! Rust SPI driver for MyVendor Sensor.

use kernel::prelude::*;
use kernel::spi;
use kernel::of;
use kernel::device::RawDevice;

module! {
    type: MySensorDriver,
    name: "my_sensor_rust",
    author: "Your Name <you@example.com>",
    description: "Rust SPI driver for MyVendor Sensor",
    license: "GPL",
}

// Register constants
const REG_WHOAMI:   u8 = 0x0F;
const REG_CTRL1:    u8 = 0x20;
const REG_DATA_OUT: u8 = 0x28;
const WHOAMI_VALUE: u8 = 0x42;
const CTRL1_ENABLE: u8 = 0x0F;

// ------------------------------------------------------------------ //
// Per-device private data                                             //
// ------------------------------------------------------------------ //

struct MySensorData {
    spi: spi::Device,
}

// ------------------------------------------------------------------ //
// Low-level helpers                                                   //
// ------------------------------------------------------------------ //

impl MySensorData {
    /// Read a single 8-bit register.
    fn read_reg(&self, reg: u8) -> Result<u8> {
        // Bit 7 = 1 signals a read to the device
        let tx: [u8; 1] = [reg | 0x80];
        let mut rx: [u8; 1] = [0u8; 1];
        self.spi.write_then_read(&tx, &mut rx)?;
        Ok(rx[0])
    }

    /// Write a single 8-bit register.
    fn write_reg(&self, reg: u8, val: u8) -> Result {
        let buf: [u8; 2] = [reg & 0x7F, val];
        self.spi.write(&buf)?;
        Ok(())
    }

    /// Burst-read `N` bytes starting at `start_reg`.
    fn burst_read<const N: usize>(&self, start_reg: u8) -> Result<[u8; N]> {
        // Many sensors use bit7=1 | bit6=1 for read + auto-increment
        let tx: [u8; 1] = [start_reg | 0xC0];
        let mut rx = [0u8; N];
        self.spi.write_then_read(&tx, &mut rx)?;
        Ok(rx)
    }

    /// Read a signed 16-bit value from two consecutive registers (LE).
    fn read_i16(&self, reg: u8) -> Result<i16> {
        let bytes = self.burst_read::<2>(reg)?;
        Ok(i16::from_le_bytes(bytes))
    }
}

// ------------------------------------------------------------------ //
// SPI driver implementation                                           //
// ------------------------------------------------------------------ //

struct MySensorDriver;

impl spi::Driver for MySensorDriver {
    type Data = Box<MySensorData>;

    kernel::define_of_id_table! {
        [()],
        [of::DeviceId::Compatible(b"myvendor,my-sensor\0")]
    }

    fn probe(spi: &mut spi::Device, _id: Option<&spi::DeviceId>) -> Result<Self::Data> {
        dev_info!(spi, "Probing MySensor Rust driver\n");

        // Configure SPI mode
        spi.set_mode(spi::Mode::MODE_3)?;
        spi.set_bits_per_word(8)?;
        spi.setup()?;

        let data = Box::try_new(MySensorData { spi: spi.clone() })?;

        // Check WHO_AM_I
        let whoami = data.read_reg(REG_WHOAMI)?;
        if whoami != WHOAMI_VALUE {
            dev_err!(spi, "Unexpected WHO_AM_I: {:#04x}\n", whoami);
            return Err(ENODEV);
        }

        // Enable sensor
        data.write_reg(REG_CTRL1, CTRL1_ENABLE)?;

        dev_info!(spi, "MySensor initialised (WHO_AM_I={:#04x})\n", whoami);
        Ok(data)
    }

    fn remove(spi: &mut spi::Device, data: &mut Self::Data) -> Result {
        dev_info!(spi, "Removing MySensor\n");
        // Disable sensor
        data.write_reg(REG_CTRL1, 0x00)?;
        Ok(())
    }
}

// ------------------------------------------------------------------ //
// Module-level registration                                           //
// ------------------------------------------------------------------ //

impl kernel::Module for MySensorDriver {
    fn init(_module: &'static ThisModule) -> Result<Self> {
        let driver = spi::Registration::<MySensorDriver>::new_pinned(
            c_str!("my_sensor_rust"),
            &THIS_MODULE,
        )?;
        // The registration object holds the driver alive.
        // Store it somewhere (here, we use `Ok` for illustration;
        // a real driver would store it in the module struct).
        Ok(Self)
    }
}
```

### 6.3 Reading XYZ Sensor Data in Rust

```rust
/// Parse and print XYZ accelerometer data (6 bytes, little-endian).
fn print_accel(data: &MySensorData) -> Result {
    let raw = data.burst_read::<6>(REG_DATA_OUT)?;

    let x = i16::from_le_bytes([raw[0], raw[1]]);
    let y = i16::from_le_bytes([raw[2], raw[3]]);
    let z = i16::from_le_bytes([raw[4], raw[5]]);

    // Each LSB = 0.061 mg for a ±2g range (example sensitivity)
    let sensitivity: f32 = 0.061;
    pr_info!(
        "Accel: X={:.3}g  Y={:.3}g  Z={:.3}g\n",
        x as f32 * sensitivity / 1000.0,
        y as f32 * sensitivity / 1000.0,
        z as f32 * sensitivity / 1000.0,
    );
    Ok(())
}
```

> **Note:** Floating-point in kernel Rust requires the `f32`/`f64` features to be explicitly enabled in the Kconfig and the target architecture to support it. For production drivers, use fixed-point arithmetic instead.

## 7. Userspace Access via `spidev` <a name="userspace"></a>

For rapid prototyping or devices that do not require a kernel driver, Linux exposes SPI buses through the generic `spidev` character device (`/dev/spidevB.C`).

### 7.1 Device Tree for spidev

```dts
&spi0 {

    };

```

### 7.2 C Userspace Example

```c
/* spidev_test.c — userspace SPI read/write via ioctl */
#include <fcntl.h>
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <sys/ioctl.h>
#include <linux/spi/spidev.h>
#include <unistd.h>
#include <stdint.h>

#define DEVICE      "/dev/spidev0.0"
#define SPEED_HZ    10000000   /* 10 MHz */
#define BITS        8
#define MODE        SPI_MODE_3

static int fd;

static void spi_init(void)
{
    uint8_t  mode  = MODE;
    uint8_t  bits  = BITS;
    uint32_t speed = SPEED_HZ;

    fd = open(DEVICE, O_RDWR);
    if (fd < 0) { perror("open"); exit(1); }

    if (ioctl(fd, SPI_IOC_WR_MODE,          &mode)  < 0) { perror("set mode");  exit(1); }
    if (ioctl(fd, SPI_IOC_WR_BITS_PER_WORD, &bits)  < 0) { perror("set bits");  exit(1); }
    if (ioctl(fd, SPI_IOC_WR_MAX_SPEED_HZ,  &speed) < 0) { perror("set speed"); exit(1); }
}

static int spi_xfer(const uint8_t *tx, uint8_t *rx, size_t len)
{
    struct spi_ioc_transfer tr = {
        .tx_buf        = (unsigned long)tx,
        .rx_buf        = (unsigned long)rx,
        .len           = len,
        .speed_hz      = SPEED_HZ,
        .bits_per_word = BITS,
        .delay_usecs   = 0,
        .cs_change     = 0,
    };
    return ioctl(fd, SPI_IOC_MESSAGE(1), &tr);
}

static int read_register(uint8_t reg, uint8_t *val)
{
    uint8_t tx[2] = { reg | 0x80, 0x00 };
    uint8_t rx[2] = { 0 };
    int ret = spi_xfer(tx, rx, 2);
    if (ret < 0) return ret;
    *val = rx[1];
    return 0;
}

static int write_register(uint8_t reg, uint8_t val)
{
    uint8_t tx[2] = { reg & 0x7F, val };
    return spi_xfer(tx, NULL, 2);
}

int main(void)
{
    uint8_t whoami;

    spi_init();

    if (read_register(0x0F, &whoami) < 0) {
        perror("read WHO_AM_I");
        close(fd);
        return 1;
    }
    printf("WHO_AM_I = 0x%02X\n", whoami);

    if (write_register(0x20, 0x0F) < 0) {
        perror("write CTRL1");
        close(fd);
        return 1;
    }
    printf("Sensor enabled\n");

    close(fd);
    return 0;
}
```

### 7.3 Rust Userspace Example (`spidev` crate)

```rust
// Cargo.toml dependency: spidev = "0.6"
use spidev::{Spidev, SpidevOptions, SpidevTransfer, SpiModeFlags};
use std::io::{self, Read, Write};

fn main() -> io::Result<()> {
    let mut spi = Spidev::open("/dev/spidev0.0")?;

    let options = SpidevOptions::new()
        .bits_per_word(8)
        .max_speed_hz(10_000_000)
        .mode(SpiModeFlags::SPI_MODE_3)
        .build();
    spi.configure(&options)?;

    // Read WHO_AM_I register
    let whoami = read_reg(&mut spi, 0x0F)?;
    println!("WHO_AM_I = {:#04X}", whoami);

    // Enable sensor (CTRL1 = 0x0F)
    write_reg(&mut spi, 0x20, 0x0F)?;
    println!("Sensor enabled");

    // Read 6 bytes of XYZ data
    let raw = burst_read(&mut spi, 0x28, 6)?;
    let x = i16::from_le_bytes([raw[0], raw[1]]);
    let y = i16::from_le_bytes([raw[2], raw[3]]);
    let z = i16::from_le_bytes([raw[4], raw[5]]);
    println!("Raw XYZ: {} {} {}", x, y, z);

    Ok(())
}

fn read_reg(spi: &mut Spidev, reg: u8) -> io::Result<u8> {
    let tx = [reg | 0x80, 0x00];
    let mut rx = [0u8; 2];
    {
        let mut transfer = SpidevTransfer::read_write(&tx, &mut rx);
        spi.transfer(&mut transfer)?;
    }
    Ok(rx[1])
}

fn write_reg(spi: &mut Spidev, reg: u8, val: u8) -> io::Result<()> {
    let tx = [reg & 0x7F, val];
    spi.write_all(&tx)?;
    Ok(())
}

fn burst_read(spi: &mut Spidev, start_reg: u8, n: usize) -> io::Result<Vec<u8>> {
    let tx_cmd = [start_reg | 0xC0];
    let mut rx = vec![0u8; n];
    // Two-transfer message: send command, then receive
    let mut cmd_buf = [0u8; 1];
    let mut transfers = [
        SpidevTransfer::read_write(&tx_cmd, &mut cmd_buf),
        SpidevTransfer::read(&mut rx),
    ];
    spi.transfer_multiple(&mut transfers)?;
    Ok(rx)
}
```

## 8. Advanced Topics <a name="advanced"></a>

### 8.1 DMA-Safe Buffers

A common pitfall: **never place SPI transfer buffers on the kernel stack**. The DMA engine requires physically contiguous, cache-aligned memory. Use `kmalloc(size, GFP_KERNEL | GFP_DMA)` or embed them as fields in your `devm_kzalloc`-allocated private data structure (which is slab-allocated and safe).

```c
/* BAD — stack buffer, may not be DMA-accessible */
u8 tx[2] = { 0x80, 0x00 };
spi_write(spi, tx, 2);  /* undefined behaviour on some SoCs */

/* GOOD — heap-allocated field in private data struct */
priv->tx_buf[0] = 0x80;
priv->tx_buf[1] = 0x00;
spi_write(spi, priv->tx_buf, 2);
```

### 8.2 Asynchronous Transfers

For high-throughput or interrupt-driven designs, use `spi_async()` to avoid blocking the calling thread:

```c
static void my_complete(void *context)
{
    struct my_sensor_data *priv = context;
    /* called in interrupt or workqueue context */
    process_data(priv->rx_buf);
}

static int start_async_read(struct my_sensor_data *priv)
{
    static struct spi_transfer xfer;
    static struct spi_message  msg;

    memset(&xfer, 0, sizeof(xfer));
    priv->tx_buf[0] = MY_SENSOR_REG_DATA_OUT | 0xC0;
    xfer.tx_buf = priv->tx_buf;
    xfer.rx_buf = priv->rx_buf;
    xfer.len    = 7;  /* 1 cmd + 6 data bytes */

    spi_message_init(&msg);
    spi_message_add_tail(&xfer, &msg);
    msg.complete = my_complete;
    msg.context  = priv;

    return spi_async(priv->spi, &msg);
}
```

### 8.3 Power Management

```c
static int my_sensor_suspend(struct device *dev)
{
    struct my_sensor_data *priv = dev_get_drvdata(dev);
    return my_sensor_write_reg(priv, MY_SENSOR_REG_CTRL1, 0x00);
}

static int my_sensor_resume(struct device *dev)
{
    struct my_sensor_data *priv = dev_get_drvdata(dev);
    return my_sensor_write_reg(priv, MY_SENSOR_REG_CTRL1, MY_SENSOR_CTRL1_ENABLE);
}

static DEFINE_SIMPLE_DEV_PM_OPS(my_sensor_pm_ops,
                                 my_sensor_suspend,
                                 my_sensor_resume);
/* then in spi_driver.driver: .pm = pm_sleep_ptr(&my_sensor_pm_ops) */
```

### 8.4 Common Pitfalls Summary

| Pitfall | Consequence | Fix |
|---------|-------------|-----|
| Stack-allocated SPI buffers | Silent data corruption or crash on DMA-capable SoCs | Use `devm_kzalloc` or heap allocation |
| Wrong CPOL/CPHA | All reads return 0xFF or 0x00 | Check datasheet; test with a logic analyser |
| Missing `spi_setup()` call | Controller ignores mode/speed settings | Always call `spi_setup()` in `probe` |
| Not checking `spi_sync` return | Silent transfer failures | Always check return value |
| Forgetting read-flag bitmask | Writes occur instead of reads | Verify device-specific protocol (bit7, RW bit) |
| Incorrect `compatible` string | Driver never probes | Must match DTS exactly, including null terminator in Rust |

## 9. Summary <a name="summary"></a>

Implementing a Linux SPI device driver involves three main layers:

1. **Hardware description** — the Device Tree node tells the kernel about the SPI slave's chip-select index, maximum clock speed, and mode (CPOL/CPHA). Custom properties pass hardware-specific configuration (GPIOs, interrupt lines) to the driver.

2. **Kernel driver (C or Rust)** — the driver registers a `spi_driver`, is matched to a device via the `of_device_id` or `spi_device_id` table, and implements `probe`/`remove`. Inside `probe`, `spi_setup()` configures the bus, `spi_sync()` or `spi_async()` performs transfers using `spi_message`/`spi_transfer` structures, and `devm_*` helpers ensure resource cleanup. The `regmap_spi` backend simplifies register-mapped devices and adds caching. For Rust drivers (Linux 6.1+), the `kernel::spi` bindings provide safe wrappers with the same fundamental model.

3. **Userspace access** — the `spidev` character device provides a convenient ioctl-based interface (`SPI_IOC_MESSAGE`) for prototyping in C or via the `spidev` crate in Rust, without writing a kernel module.

Key engineering rules to keep in mind:

- Always allocate SPI transfer buffers on the heap (never the stack) to ensure DMA safety.
- Validate the `WHO_AM_I` / device-ID register in `probe` to fail early on hardware mismatches.
- Use `spi_async()` with a completion callback for high-throughput or interrupt-driven designs.
- Implement `suspend`/`resume` callbacks for proper power management on battery-powered devices.
- Prefer the `regmap_spi` API for register-heavy peripherals to leverage caching and atomic read-modify-write helpers.

SPI driver development rewards methodical debugging: a logic analyser or oscilloscope on the SCLK/MOSI/MISO/CS lines will confirm correct clocking, mode, and framing faster than any amount of printk debugging.
