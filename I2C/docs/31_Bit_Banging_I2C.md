# Bit-banging I2C: Software-Implemented I2C Communication

## Overview

Bit-banging I2C is a technique where you implement the I2C (Inter-Integrated Circuit) protocol entirely in software using general-purpose input/output (GPIO) pins, rather than relying on dedicated I2C hardware peripherals. This approach gives you complete control over the timing and behavior of the communication protocol.

## Why Use Bit-banged I2C?

There are several scenarios where bit-banging I2C is useful:

- **Limited hardware resources**: When all hardware I2C peripherals are already in use
- **Flexible pin selection**: You can use any GPIO pins, not just those connected to I2C peripherals
- **Multiple I2C buses**: Create additional I2C buses beyond hardware limitations
- **Custom timing**: Implement non-standard timing or work around problematic devices
- **Debugging**: Better visibility into the protocol for troubleshooting
- **Hardware failures**: Workaround for damaged I2C peripherals

## I2C Protocol Basics

Before diving into implementation, let's review the I2C protocol fundamentals:

I2C uses two bidirectional lines:
- **SDA (Serial Data)**: Carries data between devices
- **SCL (Serial Clock)**: Clock signal generated by the master

Key protocol elements:
- **START condition**: SDA falls while SCL is high
- **STOP condition**: SDA rises while SCL is high
- **Data transfer**: 8 bits followed by an ACK/NACK bit
- **ACK (Acknowledge)**: SDA held low during the 9th clock pulse
- **NACK (Not Acknowledge)**: SDA held high during the 9th clock pulse

## Implementation Considerations

### Timing Requirements

Standard I2C modes have specific timing requirements:
- **Standard mode**: 100 kHz (10 μs period)
- **Fast mode**: 400 kHz (2.5 μs period)
- **Fast mode plus**: 1 MHz (1 μs period)

For bit-banging, you need to ensure your delays meet these timing constraints while accounting for code execution time.

### Open-Drain Configuration

I2C lines are open-drain (or open-collector), meaning:
- Devices can only pull the line LOW
- External pull-up resistors pull lines HIGH
- Multiple devices can share the bus safely

Your GPIO configuration must support this, typically by:
- Setting pins as inputs (high-impedance) for HIGH state
- Setting pins as outputs driving LOW for LOW state

## C/C++ Implementation

Here's a comprehensive bit-banged I2C implementation in C:

```c
#include <stdint.h>
#include <stdbool.h>

// Platform-specific GPIO functions (you'll need to implement these)
// These are examples - actual implementation depends on your hardware
extern void gpio_set_output(uint8_t pin);
extern void gpio_set_input(uint8_t pin);
extern void gpio_write_low(uint8_t pin);
extern uint8_t gpio_read(uint8_t pin);
extern void delay_us(uint32_t us);

// I2C Configuration
typedef struct {
    uint8_t sda_pin;
    uint8_t scl_pin;
    uint32_t delay_us;  // Half period delay in microseconds
} i2c_bitbang_t;

// Initialize the I2C bus
void i2c_init(i2c_bitbang_t *i2c) {
    // Set both pins as inputs (high-impedance = HIGH due to pull-ups)
    gpio_set_input(i2c->sda_pin);
    gpio_set_input(i2c->scl_pin);
    
    // Calculate delay for desired frequency
    // For 100kHz: period = 10us, half period = 5us
    if (i2c->delay_us == 0) {
        i2c->delay_us = 5;  // Default to 100kHz
    }
}

// Set SDA line state
static void sda_high(i2c_bitbang_t *i2c) {
    gpio_set_input(i2c->sda_pin);  // High-impedance, pulled high
}

static void sda_low(i2c_bitbang_t *i2c) {
    gpio_write_low(i2c->sda_pin);
    gpio_set_output(i2c->sda_pin);  // Drive low
}

// Set SCL line state
static void scl_high(i2c_bitbang_t *i2c) {
    gpio_set_input(i2c->scl_pin);  // High-impedance, pulled high
    
    // Clock stretching: wait for slave to release SCL
    uint32_t timeout = 1000;
    while (gpio_read(i2c->scl_pin) == 0 && timeout > 0) {
        delay_us(1);
        timeout--;
    }
}

static void scl_low(i2c_bitbang_t *i2c) {
    gpio_write_low(i2c->scl_pin);
    gpio_set_output(i2c->scl_pin);  // Drive low
}

// Read SDA line state
static bool sda_read(i2c_bitbang_t *i2c) {
    return gpio_read(i2c->sda_pin) != 0;
}

// Generate START condition
void i2c_start(i2c_bitbang_t *i2c) {
    sda_high(i2c);
    delay_us(i2c->delay_us);
    scl_high(i2c);
    delay_us(i2c->delay_us);
    
    // START: SDA falls while SCL is high
    sda_low(i2c);
    delay_us(i2c->delay_us);
    scl_low(i2c);
    delay_us(i2c->delay_us);
}

// Generate STOP condition
void i2c_stop(i2c_bitbang_t *i2c) {
    sda_low(i2c);
    delay_us(i2c->delay_us);
    scl_high(i2c);
    delay_us(i2c->delay_us);
    
    // STOP: SDA rises while SCL is high
    sda_high(i2c);
    delay_us(i2c->delay_us);
}

// Write a single bit
static void i2c_write_bit(i2c_bitbang_t *i2c, bool bit) {
    if (bit) {
        sda_high(i2c);
    } else {
        sda_low(i2c);
    }
    delay_us(i2c->delay_us);
    
    scl_high(i2c);
    delay_us(i2c->delay_us);
    scl_low(i2c);
    delay_us(i2c->delay_us);
}

// Read a single bit
static bool i2c_read_bit(i2c_bitbang_t *i2c) {
    sda_high(i2c);  // Release SDA to allow slave to control it
    delay_us(i2c->delay_us);
    
    scl_high(i2c);
    delay_us(i2c->delay_us);
    
    bool bit = sda_read(i2c);
    
    scl_low(i2c);
    delay_us(i2c->delay_us);
    
    return bit;
}

// Write a byte and return ACK status
bool i2c_write_byte(i2c_bitbang_t *i2c, uint8_t byte) {
    // Write 8 bits, MSB first
    for (int i = 7; i >= 0; i--) {
        i2c_write_bit(i2c, (byte >> i) & 0x01);
    }
    
    // Read ACK bit
    bool ack = !i2c_read_bit(i2c);  // ACK is active low
    return ack;
}

// Read a byte and send ACK/NACK
uint8_t i2c_read_byte(i2c_bitbang_t *i2c, bool send_ack) {
    uint8_t byte = 0;
    
    // Read 8 bits, MSB first
    for (int i = 7; i >= 0; i--) {
        if (i2c_read_bit(i2c)) {
            byte |= (1 << i);
        }
    }
    
    // Send ACK or NACK
    i2c_write_bit(i2c, !send_ack);  // ACK is low, NACK is high
    
    return byte;
}

// High-level write function
bool i2c_write(i2c_bitbang_t *i2c, uint8_t addr, 
               const uint8_t *data, uint16_t len) {
    i2c_start(i2c);
    
    // Send address with write bit (0)
    if (!i2c_write_byte(i2c, (addr << 1) | 0)) {
        i2c_stop(i2c);
        return false;  // No ACK received
    }
    
    // Send data bytes
    for (uint16_t i = 0; i < len; i++) {
        if (!i2c_write_byte(i2c, data[i])) {
            i2c_stop(i2c);
            return false;  // No ACK received
        }
    }
    
    i2c_stop(i2c);
    return true;
}

// High-level read function
bool i2c_read(i2c_bitbang_t *i2c, uint8_t addr, 
              uint8_t *data, uint16_t len) {
    i2c_start(i2c);
    
    // Send address with read bit (1)
    if (!i2c_write_byte(i2c, (addr << 1) | 1)) {
        i2c_stop(i2c);
        return false;  // No ACK received
    }
    
    // Read data bytes
    for (uint16_t i = 0; i < len; i++) {
        // Send ACK for all bytes except the last one
        data[i] = i2c_read_byte(i2c, i < (len - 1));
    }
    
    i2c_stop(i2c);
    return true;
}

// Example usage with a temperature sensor (like LM75)
void example_usage(void) {
    i2c_bitbang_t i2c = {
        .sda_pin = 2,
        .scl_pin = 3,
        .delay_us = 5  // 100kHz
    };
    
    i2c_init(&i2c);
    
    // Write to register 0x01 on device at address 0x48
    uint8_t write_data[] = {0x01, 0x50};  // Register addr + value
    if (i2c_write(&i2c, 0x48, write_data, 2)) {
        // Success
    }
    
    // Read temperature from register 0x00
    uint8_t reg_addr = 0x00;
    uint8_t temp_data[2];
    
    // Write register address
    i2c_write(&i2c, 0x48, &reg_addr, 1);
    
    // Read data
    if (i2c_read(&i2c, 0x48, temp_data, 2)) {
        int16_t temp = (temp_data[0] << 8) | temp_data[1];
        // Process temperature data
    }
}
```

## Rust Implementation

Here's a comprehensive Rust implementation with type safety and error handling:

```rust
use std::time::Duration;

// Error types for I2C operations
#[derive(Debug, Clone, Copy, PartialEq)]
pub enum I2cError {
    NoAcknowledge,
    Timeout,
    BusError,
}

// Trait for GPIO operations (platform-specific implementation)
pub trait GpioPin {
    fn set_input(&mut self);
    fn set_output_low(&mut self);
    fn read(&self) -> bool;
}

// Trait for delay operations
pub trait DelayUs {
    fn delay_us(&mut self, us: u32);
}

// Bit-banged I2C bus
pub struct I2cBitbang<SDA: GpioPin, SCL: GpioPin, D: DelayUs> {
    sda: SDA,
    scl: SCL,
    delay: D,
    half_period_us: u32,
}

impl<SDA: GpioPin, SCL: GpioPin, D: DelayUs> I2cBitbang<SDA, SCL, D> {
    /// Create a new bit-banged I2C bus
    /// frequency: desired clock frequency in Hz (e.g., 100_000 for 100kHz)
    pub fn new(mut sda: SDA, mut scl: SCL, delay: D, frequency: u32) -> Self {
        // Initialize pins as inputs (high-impedance)
        sda.set_input();
        scl.set_input();
        
        // Calculate half period in microseconds
        let half_period_us = 500_000 / frequency;  // (1_000_000 / frequency) / 2
        
        Self {
            sda,
            scl,
            delay,
            half_period_us,
        }
    }
    
    // SDA control
    fn sda_high(&mut self) {
        self.sda.set_input();
    }
    
    fn sda_low(&mut self) {
        self.sda.set_output_low();
    }
    
    fn sda_read(&self) -> bool {
        self.sda.read()
    }
    
    // SCL control with clock stretching support
    fn scl_high(&mut self) -> Result<(), I2cError> {
        self.scl.set_input();
        
        // Wait for clock stretching (slave holding SCL low)
        for _ in 0..1000 {
            if self.scl.read() {
                return Ok(());
            }
            self.delay.delay_us(1);
        }
        Err(I2cError::Timeout)
    }
    
    fn scl_low(&mut self) {
        self.scl.set_output_low();
    }
    
    /// Generate START condition
    pub fn start(&mut self) -> Result<(), I2cError> {
        self.sda_high();
        self.delay.delay_us(self.half_period_us);
        self.scl_high()?;
        self.delay.delay_us(self.half_period_us);
        
        // START: SDA falls while SCL is high
        self.sda_low();
        self.delay.delay_us(self.half_period_us);
        self.scl_low();
        self.delay.delay_us(self.half_period_us);
        
        Ok(())
    }
    
    /// Generate STOP condition
    pub fn stop(&mut self) -> Result<(), I2cError> {
        self.sda_low();
        self.delay.delay_us(self.half_period_us);
        self.scl_high()?;
        self.delay.delay_us(self.half_period_us);
        
        // STOP: SDA rises while SCL is high
        self.sda_high();
        self.delay.delay_us(self.half_period_us);
        
        Ok(())
    }
    
    /// Write a single bit
    fn write_bit(&mut self, bit: bool) -> Result<(), I2cError> {
        if bit {
            self.sda_high();
        } else {
            self.sda_low();
        }
        self.delay.delay_us(self.half_period_us);
        
        self.scl_high()?;
        self.delay.delay_us(self.half_period_us);
        self.scl_low();
        self.delay.delay_us(self.half_period_us);
        
        Ok(())
    }
    
    /// Read a single bit
    fn read_bit(&mut self) -> Result<bool, I2cError> {
        self.sda_high();  // Release SDA
        self.delay.delay_us(self.half_period_us);
        
        self.scl_high()?;
        self.delay.delay_us(self.half_period_us);
        
        let bit = self.sda_read();
        
        self.scl_low();
        self.delay.delay_us(self.half_period_us);
        
        Ok(bit)
    }
    
    /// Write a byte and return whether ACK was received
    pub fn write_byte(&mut self, byte: u8) -> Result<bool, I2cError> {
        // Write 8 bits, MSB first
        for i in (0..8).rev() {
            let bit = (byte >> i) & 0x01 != 0;
            self.write_bit(bit)?;
        }
        
        // Read ACK bit (ACK is active low)
        let ack = !self.read_bit()?;
        Ok(ack)
    }
    
    /// Read a byte and send ACK/NACK
    pub fn read_byte(&mut self, send_ack: bool) -> Result<u8, I2cError> {
        let mut byte = 0u8;
        
        // Read 8 bits, MSB first
        for i in (0..8).rev() {
            if self.read_bit()? {
                byte |= 1 << i;
            }
        }
        
        // Send ACK (low) or NACK (high)
        self.write_bit(!send_ack)?;
        
        Ok(byte)
    }
    
    /// Write data to an I2C device
    pub fn write(&mut self, addr: u8, data: &[u8]) -> Result<(), I2cError> {
        self.start()?;
        
        // Send address with write bit (0)
        if !self.write_byte((addr << 1) | 0)? {
            self.stop()?;
            return Err(I2cError::NoAcknowledge);
        }
        
        // Send data bytes
        for &byte in data {
            if !self.write_byte(byte)? {
                self.stop()?;
                return Err(I2cError::NoAcknowledge);
            }
        }
        
        self.stop()?;
        Ok(())
    }
    
    /// Read data from an I2C device
    pub fn read(&mut self, addr: u8, buffer: &mut [u8]) -> Result<(), I2cError> {
        self.start()?;
        
        // Send address with read bit (1)
        if !self.write_byte((addr << 1) | 1)? {
            self.stop()?;
            return Err(I2cError::NoAcknowledge);
        }
        
        // Read data bytes
        let len = buffer.len();
        for (i, byte) in buffer.iter_mut().enumerate() {
            // Send ACK for all bytes except the last one
            *byte = self.read_byte(i < len - 1)?;
        }
        
        self.stop()?;
        Ok(())
    }
    
    /// Write to a register and then read from it (common pattern)
    pub fn write_read(&mut self, addr: u8, write_data: &[u8], 
                      read_buffer: &mut [u8]) -> Result<(), I2cError> {
        // Write phase
        self.start()?;
        
        if !self.write_byte((addr << 1) | 0)? {
            self.stop()?;
            return Err(I2cError::NoAcknowledge);
        }
        
        for &byte in write_data {
            if !self.write_byte(byte)? {
                self.stop()?;
                return Err(I2cError::NoAcknowledge);
            }
        }
        
        // Repeated START
        self.start()?;
        
        // Read phase
        if !self.write_byte((addr << 1) | 1)? {
            self.stop()?;
            return Err(I2cError::NoAcknowledge);
        }
        
        let len = read_buffer.len();
        for (i, byte) in read_buffer.iter_mut().enumerate() {
            *byte = self.read_byte(i < len - 1)?;
        }
        
        self.stop()?;
        Ok(())
    }
}
```

## Practical Considerations

### Performance Trade-offs

Bit-banging comes with some performance considerations:

1. **CPU overhead**: The CPU is continuously occupied during I2C transactions
2. **Timing accuracy**: Depends on interrupt latency and system load
3. **Speed limitations**: Difficult to achieve fast mode (400kHz) reliably on slower systems
4. **No DMA**: All data transfer requires CPU intervention

### Debugging Tips

When debugging bit-banged I2C implementations:

- Use a logic analyzer to verify timing and protocol correctness
- Add diagnostic logging for START/STOP conditions and ACK/NACK states
- Test with known-good I2C devices
- Verify pull-up resistors are present (typically 4.7kΩ for 100kHz, 2.2kΩ for 400kHz)
- Check for electrical interference or poor connections
- Ensure your delay functions are accurate

### Common Pitfalls

Watch out for these common issues:

- **Forgetting pull-up resistors**: I2C won't work without them
- **Incorrect open-drain configuration**: May cause bus contention
- **Timing too aggressive**: Code execution time can affect delays
- **Missing clock stretching support**: Some devices require this
- **Not handling repeated START**: Needed for register read operations
- **Interrupts interfering with timing**: Consider disabling interrupts during critical sections

Bit-banging I2C gives you complete flexibility and control over I2C communication at the cost of CPU overhead and timing precision. It's an invaluable technique when hardware resources are constrained or when you need complete control over the protocol implementation.
