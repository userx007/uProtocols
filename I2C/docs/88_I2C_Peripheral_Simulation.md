## 1. Introduction

The **I2C (Inter-Integrated Circuit)** bus is one of the most widely used serial protocols in embedded systems, connecting microcontrollers to sensors, EEPROMs, RTCs, DACs, display controllers, and hundreds of other peripherals using only two wires: **SDA** (serial data) and **SCL** (serial clock).

Developing and testing firmware that communicates over I2C traditionally requires the physical peripheral to be present. This creates several problems:

- Hardware may be expensive, fragile, or scarce in CI environments.
- Deterministic testing (repeating exact sensor readings) is nearly impossible on real hardware.
- Edge cases (NACK, clock-stretch timeout, partial transfers) are hard to trigger reliably.
- Parallel development — firmware written before the hardware arrives — is blocked.

**I2C Peripheral Simulation** solves all of these by creating a software model of the slave device that behaves identically to the real component from the master's perspective.

## 2. I2C Protocol Fundamentals

Before simulating I2C, a precise understanding of the protocol's transaction model is essential.

### 2.1 Physical Layer Recap

| Signal | Role |
|--------|------|
| SDA    | Bidirectional serial data (open-drain) |
| SCL    | Clock generated by master (open-drain) |
| VCC    | Pull-up supply (typically 3.3 V or 5 V) |
| GND    | Common ground |

Both lines idle **HIGH** via pull-up resistors. Any device pulls them LOW; none can drive them HIGH actively.

### 2.2 Transaction Structure

```
START  [ADDR 7-bit] [R/W] [ACK]  [DATA_0] [ACK]  ... [DATA_n] [ACK/NACK]  STOP
  S    A6..A0       0=W    A      D7..D0    A/N              ...              P
```

Key events the simulator must faithfully reproduce:

| Event | Description |
|-------|-------------|
| **START** | SDA falls while SCL is HIGH |
| **STOP** | SDA rises while SCL is HIGH |
| **ACK** | Slave pulls SDA LOW on 9th clock |
| **NACK** | Slave releases SDA HIGH on 9th clock |
| **Clock Stretch** | Slave holds SCL LOW to pause the master |
| **Repeated START** | START without preceding STOP (register read pattern) |

### 2.3 Typical Register-Read Sequence

```
Master: START | ADDR+W | REG_ADDR | REPEATED-START | ADDR+R
Slave:                    ACK              ACK
Master:                                                     | DATA_READ | STOP
Slave:                                                         DATA
```

This **write-then-read** pattern is so common that virtually every I2C simulator must handle it explicitly.

## 3. Why Simulate I2C Peripherals?

### 3.1 Advantages

| Benefit | Detail |
|---------|--------|
| **Determinism** | Return exact, repeatable values for regression tests |
| **Speed** | Software simulation runs at CPU speed, not 100/400 kHz bus speed |
| **Error injection** | Deliberately cause NACK, timeout, bus-stuck, garbled data |
| **CI/CD compatibility** | No hardware needed in build pipelines |
| **Parallelism** | Multiple simulated buses can run concurrently |
| **Introspection** | Log every transaction without extra hardware sniffers |
| **Coverage** | Exercise code paths that require rare sensor states |

### 3.2 Simulation Fidelity Spectrum

```
Low Fidelity                                              High Fidelity
     │                                                          │
  Mock                  Register Map              Cycle-Accurate
  Object                Simulator                  RTL Model
  (unit                 (integration               (gate-level,
   tests)                tests)                     used in EDA)
     │                       │                          │
Fastest, simplest     Good balance           Slowest, most accurate
```

For firmware development and CI, the **register-map simulator** sweet spot covers 95% of use cases.

## 4. Simulation Architectures

### 4.1 Software-Only Simulation (Abstract Layer)

The I2C hardware access layer is abstracted behind an interface. In tests, a software implementation replaces the hardware driver. No kernel modules or special hardware is needed.

```
┌──────────────────────────────────────────────────────┐
│                  Application / Driver                │
├──────────────────────────────────────────────────────┤
│              I2C HAL Interface (abstract)            │
├───────────────────────┬──────────────────────────────┤
│   Real HW Driver      │    Simulated I2C Backend     │
│  (ioctl/registers)    │  (in-process register map)   │
└───────────────────────┴──────────────────────────────┘
```

### 4.2 Linux Kernel `i2c-stub` Module

Linux provides a kernel module `i2c-stub` that creates a virtual I2C adapter. Userspace tools can then act as simulated slaves using `/dev/i2c-N`.

### 4.3 QEMU I2C Device Models

QEMU supports I2C bus emulation. Custom device models can be written as QEMU C plugins. This is used for full-system firmware emulation.

### 4.4 Embedded HAL Mock Objects

In Rust's `embedded-hal` ecosystem, traits define I2C behavior. Mock objects implement these traits and record or replay transactions.

## 5. C/C++ Implementation

### 5.1 Register-Map Simulator (Bare-Metal / Linux)

This is the most portable pattern: define a register bank, implement read/write handlers, and plug the simulator into your HAL.

```c
/* i2c_sim.h — Generic I2C Peripheral Simulator */
#pragma once
#include <stdint.h>
#include <stdbool.h>
#include <string.h>

#define I2C_SIM_MAX_REGS   256
#define I2C_SIM_MAX_DEVS   8

typedef struct {
    uint8_t  address;                      /* 7-bit I2C slave address           */
    uint8_t  regs[I2C_SIM_MAX_REGS];       /* Register bank                     */
    bool     reg_readonly[I2C_SIM_MAX_REGS]; /* Write-protect mask              */
    uint8_t  reg_ptr;                      /* Current register pointer          */

    /* Optional callbacks for special register behavior */
    void (*on_write)(uint8_t reg, uint8_t val, void *ctx);
    uint8_t (*on_read)(uint8_t reg, void *ctx);
    void    *cb_ctx;

    /* Statistics */
    uint32_t read_count;
    uint32_t write_count;
    uint32_t nack_count;
} I2CSimDevice;

typedef struct {
    I2CSimDevice *devices[I2C_SIM_MAX_DEVS];
    int           num_devices;
    bool          inject_nack;          /* Force next transaction to NACK */
    bool          inject_timeout;       /* Simulate clock-stretch timeout */
    char          log_buf[4096];
    int           log_pos;
} I2CBus;

/* ── Simulator API ─────────────────────────────────── */

void i2c_bus_init(I2CBus *bus) {
    memset(bus, 0, sizeof(*bus));
}

void i2c_bus_attach(I2CBus *bus, I2CSimDevice *dev) {
    if (bus->num_devices < I2C_SIM_MAX_DEVS)
        bus->devices[bus->num_devices++] = dev;
}

static I2CSimDevice *find_device(I2CBus *bus, uint8_t addr) {
    for (int i = 0; i < bus->num_devices; i++)
        if (bus->devices[i]->address == addr)
            return bus->devices[i];
    return NULL;
}

/* Returns 0 on success, -1 on NACK / device not found */
int i2c_sim_write(I2CBus *bus, uint8_t addr,
                  const uint8_t *data, size_t len) {
    if (bus->inject_nack) { bus->inject_nack = false;
        bus->log_pos += snprintf(bus->log_buf + bus->log_pos,
            sizeof(bus->log_buf) - bus->log_pos,
            "[NACK injected] WRITE addr=0x%02X\n", addr);
        return -1;
    }
    I2CSimDevice *dev = find_device(bus, addr);
    if (!dev) return -1;   /* NACK — no device at this address */

    dev->write_count++;
    /* First byte selects the register */
    dev->reg_ptr = data[0];
    for (size_t i = 1; i < len; i++) {
        uint8_t reg = dev->reg_ptr++;
        if (!dev->reg_readonly[reg]) {
            dev->regs[reg] = data[i];
            if (dev->on_write) dev->on_write(reg, data[i], dev->cb_ctx);
        }
    }
    return 0;
}

int i2c_sim_read(I2CBus *bus, uint8_t addr,
                 uint8_t *buf, size_t len) {
    if (bus->inject_nack) { bus->inject_nack = false; return -1; }
    I2CSimDevice *dev = find_device(bus, addr);
    if (!dev) return -1;

    dev->read_count++;
    for (size_t i = 0; i < len; i++) {
        uint8_t reg = dev->reg_ptr;
        buf[i] = dev->on_read
                    ? dev->on_read(reg, dev->cb_ctx)
                    : dev->regs[reg];
        dev->reg_ptr = (dev->reg_ptr + 1) & 0xFF;  /* Auto-increment */
    }
    return 0;
}

/* Convenience: write register address, then read (repeated START) */
int i2c_sim_write_read(I2CBus *bus, uint8_t addr,
                       uint8_t reg, uint8_t *buf, size_t len) {
    if (i2c_sim_write(bus, addr, &reg, 1) != 0) return -1;
    return i2c_sim_read(bus, addr, buf, len);
}
```

### 5.2 Linux i2c-stub / i2c-dev Integration

On Linux, you can load the `i2c-stub` kernel module and then populate its registers from userspace to create a simulated slave accessible by any standard I2C master (including your driver under test).

```c
/* load_i2c_stub.c — Load i2c-stub and configure a simulated device */
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <fcntl.h>
#include <unistd.h>
#include <sys/ioctl.h>
#include <linux/i2c-dev.h>
#include <linux/i2c.h>

#define SLAVE_ADDR   0x48   /* Example: TMP102 temperature sensor */

/* Load the kernel module, specifying the chip address */
static int load_stub_module(uint8_t addr) {
    char cmd[128];
    /* chip_addr=0xNN tells the stub to respond at that address */
    snprintf(cmd, sizeof(cmd),
             "modprobe i2c-stub chip_addr=0x%02X", addr);
    return system(cmd);
}

/* Find the bus number created by i2c-stub (usually the last one) */
static int find_stub_bus(void) {
    FILE *fp = popen("ls /sys/bus/i2c/devices/ | grep i2c | tail -1", "r");
    if (!fp) return -1;
    int bus = -1;
    fscanf(fp, "i2c-%d", &bus);
    pclose(fp);
    return bus;
}

/* Use i2cset (from i2c-tools) to pre-populate registers */
static void populate_registers(int bus, uint8_t addr) {
    char cmd[256];
    /* Simulate TMP102: reg 0x00 = temperature MSB/LSB (25°C = 0x0190) */
    snprintf(cmd, sizeof(cmd),
             "i2cset -y %d 0x%02X 0x00 0x01 0x90 i", bus, addr);
    system(cmd);
    /* Config register */
    snprintf(cmd, sizeof(cmd),
             "i2cset -y %d 0x%02X 0x01 0x60 0xA0 i", bus, addr);
    system(cmd);
}

int main(void) {
    if (load_stub_module(SLAVE_ADDR) != 0) {
        fprintf(stderr, "Failed to load i2c-stub (need root?)\n");
        return 1;
    }

    int bus = find_stub_bus();
    if (bus < 0) { fprintf(stderr, "Stub bus not found\n"); return 1; }
    printf("i2c-stub active on bus /dev/i2c-%d\n", bus);

    populate_registers(bus, SLAVE_ADDR);

    /* Open the bus as a master and read back */
    char dev_path[32];
    snprintf(dev_path, sizeof(dev_path), "/dev/i2c-%d", bus);
    int fd = open(dev_path, O_RDWR);
    if (fd < 0) { perror("open"); return 1; }

    if (ioctl(fd, I2C_SLAVE, SLAVE_ADDR) < 0) {
        perror("I2C_SLAVE"); return 1;
    }

    /* Write register pointer then read 2 bytes */
    uint8_t reg = 0x00;
    write(fd, &reg, 1);
    uint8_t buf[2] = {0};
    read(fd, buf, 2);

    int16_t raw = (buf[0] << 8) | buf[1];
    float temp = (raw >> 4) * 0.0625f;   /* TMP102 12-bit format */
    printf("Simulated temperature: %.2f °C\n", temp);

    close(fd);
    system("modprobe -r i2c-stub");
    return 0;
}
```

### 5.3 Simulating a Real Sensor (BME280)

A realistic simulation of the Bosch BME280 temperature/pressure/humidity sensor, complete with calibration registers, measurement registers, and status flags.

```c
/* bme280_sim.c — Simulated BME280 environmental sensor */
#include "i2c_sim.h"
#include <math.h>

/* BME280 register addresses */
#define BME280_REG_ID        0xD0   /* Should return 0x60            */
#define BME280_REG_RESET     0xE0   /* Write 0xB6 to soft-reset      */
#define BME280_REG_CTRL_HUM  0xF2
#define BME280_REG_STATUS    0xF3
#define BME280_REG_CTRL_MEAS 0xF4
#define BME280_REG_CONFIG    0xF5
#define BME280_REG_PRESS_MSB 0xF7
#define BME280_REG_TEMP_MSB  0xFA
#define BME280_REG_HUM_MSB   0xFD

/* Calibration base address */
#define BME280_CALIB_BASE    0x88

typedef struct {
    I2CSimDevice base;
    /* Environmental setpoints — what the sensor "measures" */
    float temperature_c;
    float pressure_pa;
    float humidity_rh;
    /* Bosch calibration coefficients (fixed for this simulation) */
    int32_t dig_T1, dig_T2, dig_T3;
    int64_t t_fine;   /* Shared between T and P compensation */
} BME280Sim;

/* Encode temperature using BME280's ADC format (20-bit) */
static uint32_t encode_temperature(BME280Sim *s, float temp_c) {
    /* Simplified linear approximation — real BME280 uses polynomial */
    /* Typical: 25°C ≈ ADC raw = 0x7E5A8 */
    (void)s;
    return (uint32_t)((temp_c / 0.0059604f) + 500000.0f) & 0xFFFFF;
}

static uint32_t encode_pressure(BME280Sim *s, float press_pa) {
    (void)s;
    /* 101325 Pa (sea level) ≈ ADC raw 0x56680 */
    return (uint32_t)((press_pa / 0.0024414f)) & 0xFFFFF;
}

static uint16_t encode_humidity(BME280Sim *s, float rh) {
    (void)s;
    return (uint16_t)(rh * 1024.0f);   /* Q10.6 fixed-point */
}

static void bme280_update_meas_regs(BME280Sim *s) {
    uint32_t t_raw = encode_temperature(s, s->temperature_c);
    uint32_t p_raw = encode_pressure(s, s->pressure_pa);
    uint16_t h_raw = encode_humidity(s, s->humidity_rh);

    /* Temperature: 0xFA (MSB), 0xFB (LSB), 0xFC[7:4] (XLSB) */
    s->base.regs[0xFA] = (t_raw >> 12) & 0xFF;
    s->base.regs[0xFB] = (t_raw >> 4)  & 0xFF;
    s->base.regs[0xFC] = (t_raw & 0xF) << 4;

    /* Pressure: 0xF7, 0xF8, 0xF9[7:4] */
    s->base.regs[0xF7] = (p_raw >> 12) & 0xFF;
    s->base.regs[0xF8] = (p_raw >> 4)  & 0xFF;
    s->base.regs[0xF9] = (p_raw & 0xF) << 4;

    /* Humidity: 0xFD (MSB), 0xFE (LSB) */
    s->base.regs[0xFD] = (h_raw >> 8) & 0xFF;
    s->base.regs[0xFE] =  h_raw       & 0xFF;

    /* Status: measurement complete (bit 3 = 0) */
    s->base.regs[0xF3] = 0x00;
}

static void bme280_on_write(uint8_t reg, uint8_t val, void *ctx) {
    BME280Sim *s = (BME280Sim *)ctx;
    if (reg == BME280_REG_RESET && val == 0xB6) {
        /* Soft reset: restore power-on defaults */
        s->base.regs[BME280_REG_ID]       = 0x60;
        s->base.regs[BME280_REG_CTRL_MEAS]= 0x00;
        s->base.regs[BME280_REG_STATUS]   = 0x00;
        printf("[BME280 SIM] Soft reset performed\n");
    }
    if (reg == BME280_REG_CTRL_MEAS) {
        /* Entering forced mode: trigger measurement */
        uint8_t mode = val & 0x03;
        if (mode == 0x01 || mode == 0x02) {
            s->base.regs[0xF3] = 0x08;   /* measuring bit set */
            bme280_update_meas_regs(s);   /* instantly complete */
        }
    }
}

void bme280_sim_init(BME280Sim *s, uint8_t addr,
                     float temp, float press, float hum) {
    memset(s, 0, sizeof(*s));
    s->base.address   = addr;
    s->base.on_write  = bme280_on_write;
    s->base.cb_ctx    = s;
    s->temperature_c  = temp;
    s->pressure_pa    = press;
    s->humidity_rh    = hum;

    /* Device ID */
    s->base.regs[BME280_REG_ID]    = 0x60;
    s->base.reg_readonly[BME280_REG_ID] = true;

    /* Calibration: load fixed coefficients into 0x88..0xA1 */
    /* (Real production code would store device-specific values here) */
    s->base.regs[0x88] = 0x70; s->base.regs[0x89] = 0x6B;  /* dig_T1 */
    s->base.regs[0x8A] = 0xBF; s->base.regs[0x8B] = 0x67;  /* dig_T2 */
    s->base.regs[0x8C] = 0x32; s->base.regs[0x8D] = 0x00;  /* dig_T3 */

    bme280_update_meas_regs(s);
}

/* ── Usage example ─────────────────────────────────── */
int main(void) {
    I2CBus   bus;
    BME280Sim bme;

    i2c_bus_init(&bus);
    bme280_sim_init(&bme, 0x76, 25.3f, 101325.0f, 55.0f);
    i2c_bus_attach(&bus, &bme.base);

    /* Read device ID */
    uint8_t id = 0;
    i2c_sim_write_read(&bus, 0x76, BME280_REG_ID, &id, 1);
    printf("Device ID: 0x%02X (expect 0x60)\n", id);

    /* Trigger forced-mode measurement */
    uint8_t force_cmd[] = { BME280_REG_CTRL_MEAS, 0x27 }; /* osrs=1, forced */
    i2c_sim_write(&bus, 0x76, force_cmd, 2);

    /* Read raw temperature registers */
    uint8_t t_raw[3];
    i2c_sim_write_read(&bus, 0x76, BME280_REG_TEMP_MSB, t_raw, 3);
    printf("T raw: 0x%02X 0x%02X 0x%02X\n", t_raw[0], t_raw[1], t_raw[2]);

    return 0;
}
```

### 5.4 Event-Driven Simulation with Callbacks

For more complex scenarios — where register writes trigger state transitions (like a motor controller) — an event-driven pattern is useful.

```cpp
// i2c_device_model.hpp — C++ event-driven I2C peripheral model
#pragma once
#include <cstdint>
#include <functional>
#include <map>
#include <vector>
#include <string>
#include <sstream>

class I2CDeviceModel {
public:
    using ReadHandler  = std::function<uint8_t(uint8_t reg)>;
    using WriteHandler = std::function<void(uint8_t reg, uint8_t val)>;

    explicit I2CDeviceModel(uint8_t address) : addr_(address) {}

    /* Register custom read/write handlers per register */
    void on_read (uint8_t reg, ReadHandler  h) { read_handlers_[reg]  = h; }
    void on_write(uint8_t reg, WriteHandler h) { write_handlers_[reg] = h; }

    /* Default register access (no handler = raw byte) */
    uint8_t  read_reg (uint8_t reg) const { return regs_[reg]; }
    void     write_reg(uint8_t reg, uint8_t val) { regs_[reg] = val; }

    uint8_t address() const { return addr_; }

    /* Called by the bus simulator */
    int  handle_write(const uint8_t *data, size_t len);
    int  handle_read (uint8_t *buf, size_t len);
    void log_transaction(bool is_read, const uint8_t *data, size_t len);

    const std::string& transaction_log() const { return log_; }

private:
    uint8_t  addr_;
    uint8_t  regs_[256] = {};
    uint8_t  reg_ptr_   = 0;
    std::map<uint8_t, ReadHandler>  read_handlers_;
    std::map<uint8_t, WriteHandler> write_handlers_;
    std::string log_;
};

int I2CDeviceModel::handle_write(const uint8_t *data, size_t len) {
    reg_ptr_ = data[0];
    for (size_t i = 1; i < len; i++) {
        uint8_t reg = reg_ptr_++;
        regs_[reg] = data[i];
        auto it = write_handlers_.find(reg);
        if (it != write_handlers_.end()) it->second(reg, data[i]);
    }
    return 0;
}

int I2CDeviceModel::handle_read(uint8_t *buf, size_t len) {
    for (size_t i = 0; i < len; i++) {
        uint8_t reg = reg_ptr_;
        auto it = read_handlers_.find(reg);
        buf[i] = (it != read_handlers_.end()) ? it->second(reg) : regs_[reg];
        reg_ptr_ = (reg_ptr_ + 1) & 0xFF;
    }
    return 0;
}

/* ── Example: Simulated 16-bit DAC (MCP4725) ──────── */
#include <cstdio>

class MCP4725Sim : public I2CDeviceModel {
public:
    explicit MCP4725Sim(uint8_t addr) : I2CDeviceModel(addr), dac_value_(0) {
        /* Register write handler for DAC output registers */
        on_write(0x40, [this](uint8_t, uint8_t val) {
            /* MCP4725 write-DAC command: 2 data bytes follow */
            (void)val;
        });
    }

    int handle_write(const uint8_t *data, size_t len) {
        if (len >= 3) {
            uint8_t cmd = (data[0] >> 5) & 0x07;
            if (cmd == 0x02) {                             /* Write DAC Register */
                dac_value_ = ((uint16_t)data[1] << 4) | (data[2] >> 4);
                voltage_   = (dac_value_ / 4095.0f) * vref_;
                printf("[MCP4725 SIM] DAC output: %u  (%.3f V)\n",
                       dac_value_, voltage_);
```

### 6.3 Full Peripheral Simulator with State Machine

A complete register-map simulator in Rust, implementing the full I2C transaction state machine.

```rust
// src/i2c_sim.rs — Full-featured I2C peripheral simulator in Rust

use std::collections::HashMap;

#[derive(Debug, Clone, PartialEq)]
pub enum TransactionState {
    Idle,
    AddressedWrite,
    AddressedRead,
    RegisterSelected,
}

pub struct RegisterMap {
    pub regs:      [u8; 256],
    pub readonly:  [bool; 256],
    read_hooks:    HashMap<u8, Box<dyn Fn(u8) -> u8 + Send>>,
    write_hooks:   HashMap<u8, Box<dyn Fn(u8, u8) + Send>>,
}

impl RegisterMap {
    pub fn new() -> Self {
        Self {
            regs:       [0u8; 256],
            readonly:   [false; 256],
            read_hooks: HashMap::new(),
            write_hooks: HashMap::new(),
        }
    }

    pub fn set_readonly(&mut self, reg: u8) {
        self.readonly[reg as usize] = true;
    }

    pub fn on_read<F>(&mut self, reg: u8, f: F)
    where F: Fn(u8) -> u8 + Send + 'static {
        self.read_hooks.insert(reg, Box::new(f));
    }

    pub fn on_write<F>(&mut self, reg: u8, f: F)
    where F: Fn(u8, u8) + Send + 'static {
        self.write_hooks.insert(reg, Box::new(f));
    }

    pub fn read(&self, reg: u8) -> u8 {
        if let Some(hook) = self.read_hooks.get(&reg) {
            hook(reg)
        } else {
            self.regs[reg as usize]
        }
    }

    pub fn write(&mut self, reg: u8, val: u8) {
        if !self.readonly[reg as usize] {
            self.regs[reg as usize] = val;
        }
        if let Some(hook) = self.write_hooks.get(&reg) {
            hook(reg, val);
        }
    }
}

pub struct I2CPeripheralSim {
    pub address:   u8,
    pub regs:      RegisterMap,
    reg_ptr:       u8,
    state:         TransactionState,
    // Statistics
    pub reads:     u32,
    pub writes:    u32,
    pub errors:    u32,
    pub log:       Vec<String>,
}

#[derive(Debug)]
pub enum I2CError {
    AddressMismatch,
    WriteToReadonly(u8),
    InvalidState,
}

impl I2CPeripheralSim {
    pub fn new(address: u8) -> Self {
        Self {
            address,
            regs:    RegisterMap::new(),
            reg_ptr: 0,
            state:   TransactionState::Idle,
            reads:   0,
            writes:  0,
            errors:  0,
            log:     Vec::new(),
        }
    }

    /// Called by bus simulator when master initiates a write transaction
    pub fn receive_write(&mut self, addr: u8, data: &[u8])
        -> Result<(), I2CError>
    {
        if addr != self.address {
            return Err(I2CError::AddressMismatch);
        }
        if data.is_empty() {
            return Ok(());
        }

        // First byte selects the register
        self.reg_ptr = data[0];
        self.state   = TransactionState::RegisterSelected;

        for (i, &byte) in data[1..].iter().enumerate() {
            let reg = self.reg_ptr.wrapping_add(i as u8);
            if self.regs.readonly[reg as usize] {
                self.errors += 1;
                return Err(I2CError::WriteToReadonly(reg));
            }
            self.regs.write(reg, byte);
        }

        if data.len() > 1 {
            self.reg_ptr = self.reg_ptr.wrapping_add((data.len() - 1) as u8);
        }

        self.writes += 1;
        self.log.push(format!(
            "WRITE addr=0x{:02X} reg=0x{:02X} data={:02X?}",
            addr, data[0], &data[1..]
        ));
        Ok(())
    }

    /// Called by bus simulator when master reads from this device
    pub fn receive_read(&mut self, addr: u8, len: usize)
        -> Result<Vec<u8>, I2CError>
    {
        if addr != self.address {
            return Err(I2CError::AddressMismatch);
        }

        let mut buf = Vec::with_capacity(len);
        for _ in 0..len {
            buf.push(self.regs.read(self.reg_ptr));
            self.reg_ptr = self.reg_ptr.wrapping_add(1);
        }

        self.reads += 1;
        self.log.push(format!(
            "READ  addr=0x{:02X} data={:02X?}",
            addr, buf
        ));
        Ok(buf)
    }

    pub fn print_log(&self) {
        for entry in &self.log {
            println!("[I2C SIM] {}", entry);
        }
    }
}

/// A bus that routes transactions to the correct simulated peripheral
pub struct I2CBusSim {
    devices:     Vec<I2CPeripheralSim>,
    inject_nack: bool,
}

impl I2CBusSim {
    pub fn new() -> Self {
        Self { devices: Vec::new(), inject_nack: false }
    }

    pub fn attach(&mut self, dev: I2CPeripheralSim) {
        self.devices.push(dev);
    }

    pub fn inject_nack(&mut self) {
        self.inject_nack = true;
    }

    pub fn write_read(&mut self, addr: u8, write_data: &[u8], read_len: usize)
        -> Result<Vec<u8>, I2CError>
    {
        if self.inject_nack {
            self.inject_nack = false;
            return Err(I2CError::AddressMismatch);
        }
        let dev = self.devices.iter_mut()
            .find(|d| d.address == addr)
            .ok_or(I2CError::AddressMismatch)?;

        dev.receive_write(addr, write_data)?;
        dev.receive_read(addr, read_len)
    }

    pub fn write(&mut self, addr: u8, data: &[u8])
        -> Result<(), I2CError>
    {
        if self.inject_nack {
            self.inject_nack = false;
            return Err(I2CError::AddressMismatch);
        }
        let dev = self.devices.iter_mut()
            .find(|d| d.address == addr)
            .ok_or(I2CError::AddressMismatch)?;
        dev.receive_write(addr, data)
    }
}
```

```rust
// src/async_i2c_sim.rs — Async I2C peripheral with artificial latency

use tokio::sync::Mutex;
use std::sync::Arc;
use std::time::Duration;

#[async_trait::async_trait]
pub trait AsyncI2CSlave: Send + Sync {
    async fn on_write(&self, data: &[u8]) -> Result<(), String>;
    async fn on_read(&self, len: usize)   -> Result<Vec<u8>, String>;
}

pub struct AsyncI2CBus {
    devices: Vec<(u8, Arc<dyn AsyncI2CSlave>)>,
    latency: Duration,  // Simulated bus latency per transaction
}

impl AsyncI2CBus {
    pub fn new(latency_us: u64) -> Self {
        Self {
            devices: Vec::new(),
            latency: Duration::from_micros(latency_us),
        }
    }

    pub fn attach(&mut self, addr: u8, dev: Arc<dyn AsyncI2CSlave>) {
        self.devices.push((addr, dev));
    }

    pub async fn write_read(
        &self, addr: u8, write_data: &[u8], read_len: usize
    ) -> Result<Vec<u8>, String> {
        // Simulate bus latency
        tokio::time::sleep(self.latency).await;

        let dev = self.devices.iter()
            .find(|(a, _)| *a == addr)
            .map(|(_, d)| d.clone())
            .ok_or_else(|| format!("NACK: no device at 0x{:02X}", addr))?;

        dev.on_write(write_data).await?;
        dev.on_read(read_len).await
    }
}

/// Example: simulated TMP117 temperature sensor with async read
pub struct Tmp117Sim {
    regs:    Arc<Mutex<[u8; 256]>>,
    temp_c:  f32,
}

impl Tmp117Sim {
    pub fn new(temp_c: f32) -> Arc<Self> {
        let mut r = [0u8; 256];
        // Encode temperature: TMP117 is 16-bit, LSB = 7.8125 m°C
        let raw = (temp_c / 0.0078125) as i16;
        r[0x00] = ((raw >> 8) & 0xFF) as u8;   // Temp MSB
        r[0x01] = (raw & 0xFF) as u8;           // Temp LSB
        r[0x0F] = 0x01;                          // Device ID MSB
        r[0x10] = 0x17;                          // Device ID LSB
        Arc::new(Self { regs: Arc::new(Mutex::new(r)), temp_c })
    }
}

#[async_trait::async_trait]
impl AsyncI2CSlave for Tmp117Sim {
    async fn on_write(&self, data: &[u8]) -> Result<(), String> {
        // For simplicity, just update register pointer (stored implicitly)
        // A full implementation would track reg_ptr per transaction
        Ok(())
    }

    async fn on_read(&self, len: usize) -> Result<Vec<u8>, String> {

        tokio::time::sleep(Duration::from_millis(16)).await;
        let regs = self.regs.lock().await;
        Ok(regs[0..len.min(256)].to_vec())
    }
}
```

## 7. Advanced Techniques

### 7.1 Clock Stretching Simulation

Clock stretching (the slave holding SCL LOW to pause the master) is an important edge case. In a software simulator, this manifests as a configurable delay before acknowledging a transaction.

```c
/* Clock stretch: delay before completing a read */
typedef struct {
    I2CSimDevice base;
    int stretch_cycles;   /* Number of SCL cycles to "stretch" */
    int stretch_on_reg;   /* Which register triggers stretching */
} ClockStretchSim;

uint8_t stretching_read_hook(uint8_t reg, void *ctx) {
    ClockStretchSim *s = (ClockStretchSim *)ctx;
    if (reg == s->stretch_on_reg && s->stretch_cycles > 0) {
        /* In a real simulation, hold SCL low here for N cycles.
           In software, simulate the delay: */
        struct timespec ts = { .tv_nsec = s->stretch_cycles * 1000L };
        nanosleep(&ts, NULL);
        printf("[SIM] Clock stretched %d cycles on reg 0x%02X\n",
               s->stretch_cycles, reg);
    }
    return s->base.regs[reg];
}
```

### 7.2 Bus Arbitration and Multi-Master

In multi-master simulations, arbitration is resolved by a priority scheme. When two simulated masters attempt simultaneous access, the lower-address master wins (mimicking real open-drain collision detection).

```rust
pub struct MultiMasterBus {
    devices: Vec<(u8, Box<dyn I2CSlave>)>,
    arbitration_winner: Option<u8>,  // Address of winning master
}

impl MultiMasterBus {
    /// Simulate arbitration: lowest address wins
    pub fn arbitrate(&mut self, masters: &[u8]) -> u8 {
        let winner = *masters.iter().min().unwrap_or(&0xFF);
        self.arbitration_winner = Some(winner);
        winner
    }
}
```

## 10. Summary

I2C Peripheral Simulation is the practice of building software models of I2C slave devices so that firmware drivers and application logic can be developed, tested, and validated entirely without physical hardware.

The core pattern — regardless of language — involves three elements: a **register map** representing device state, **read/write handlers** that implement device behavior, and a **bus router** that dispatches transactions to the correct simulated peripheral by address.

In **C/C++**, this is typically achieved through a HAL abstraction layer that is swapped between a real hardware backend and an in-process simulator. The `i2c-stub` Linux kernel module additionally allows kernel-level drivers and userspace tools to interact with a virtual bus using standard `ioctl` calls.

In **Rust**, the `embedded-hal` trait ecosystem is the natural integration point. The `embedded-hal-mock` crate provides ready-made transaction-level mocks, while custom simulator implementations using the `I2c` trait offer full register-map behavior. Rust's strong type system and ownership model make it particularly well-suited to building simulators that enforce correct transaction ordering and catch races at compile time.

Advanced simulation capabilities — clock stretching, arbitration, partial ACKs, CRC errors, and timed responses — allow teams to achieve edge-case coverage that is difficult or impossible to reproduce with real hardware, while seamlessly integrating into CI/CD pipelines without any special hardware requirements.

The result is faster development cycles, higher driver quality, and deterministic regression testing across every commit.
