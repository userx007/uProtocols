## Introduction

User-space I2C access refers to the ability of ordinary applications — running without kernel privileges — to communicate directly with I2C devices through the Linux device filesystem. Rather than writing a kernel driver, a developer opens a file descriptor on `/dev/i2c-X`, uses standard `ioctl()` calls to configure the target device address and transfer mode, and then reads or writes data as with any file.

This approach is particularly valuable for:

- **Rapid prototyping** — test sensors and actuators without writing kernel code
- **User-space drivers** — implement full device drivers entirely in application space
- **Scripting and tooling** — tools like `i2cdetect`, `i2cget`, and `i2cset` all use this mechanism
- **Embedded Linux systems** — directly control hardware peripherals from application code

---

## I2C Protocol Fundamentals

I2C (Inter-Integrated Circuit) is a two-wire serial communication protocol developed by Philips. It uses:

- **SDA** (Serial Data Line) — bidirectional data
- **SCL** (Serial Clock Line) — clock generated by the master

Key protocol characteristics:

| Property | Detail |
|---|---|
| Topology | Multi-master, multi-slave on a single bus |
| Addressing | 7-bit (128 addresses) or 10-bit (1024 addresses) |
| Speed | Standard (100 kHz), Fast (400 kHz), Fast-plus (1 MHz), High-speed (3.4 MHz) |
| Data framing | Start condition → Address + R/W bit → ACK → Data bytes → ACK → Stop condition |
| Pull-ups | SDA and SCL require pull-up resistors (typically 4.7 kΩ at 100 kHz) |

A transaction begins with the master asserting a **START** condition, then transmitting a 7-bit slave address plus a read/write bit. The addressed slave acknowledges, and data flows one byte at a time, each followed by an ACK/NACK from the receiver. The master closes with a **STOP** condition (or a repeated START for combined transactions).

## Linux I2C Subsystem Architecture

Linux models I2C communication in three layers:

```
┌─────────────────────────────────────┐
│         User-Space Application      │
│  open("/dev/i2c-1") + ioctl + r/w   │
└──────────────┬──────────────────────┘
               │ system calls
┌──────────────▼──────────────────────┐
│      i2c-dev character driver       │  ← /dev/i2c-X  (kernel module: i2c-dev)
├─────────────────────────────────────┤
│       I2C Core (i2c-core.ko)        │  ← algorithm dispatch, bus management
├─────────────────────────────────────┤
│    I2C Adapter / Bus Driver         │  ← hardware-specific (e.g. i2c-bcm2835)
└──────────────┬──────────────────────┘
               │
        Physical SDA/SCL pins
```

The `i2c-dev` kernel module creates `/dev/i2c-N` character devices — one per I2C bus adapter discovered in the system. All user-space access flows through these nodes.

## The `/dev/i2c-X` Interface

### Identifying Available Buses

```bash
# List I2C buses
ls /dev/i2c-*
# or
i2cdetect -l

# Scan bus 1 for connected devices
i2cdetect -y 1
```

### Opening the Device

```c
int fd = open("/dev/i2c-1", O_RDWR);
if (fd < 0) {
    perror("Failed to open I2C device");
    exit(EXIT_FAILURE);
}
```

Once opened, the file descriptor represents the bus, not a specific device. The target slave address is set per-transaction via `ioctl`.

## Key ioctl Commands

All defined in `<linux/i2c-dev.h>` and `<linux/i2c.h>`:

| ioctl Request | Purpose |
|---|---|
| `I2C_SLAVE` | Set slave address (requires no privileges) |
| `I2C_SLAVE_FORCE` | Set slave address even if a kernel driver is bound |
| `I2C_TENBIT` | Enable 10-bit addressing mode (0 = 7-bit, 1 = 10-bit) |
| `I2C_PEC` | Enable Packet Error Checking for SMBus |
| `I2C_FUNCS` | Query bus capabilities (returns a bitmask) |
| `I2C_RDWR` | Perform a combined read/write transaction |
| `I2C_SMBUS` | Perform an SMBus transaction |
| `I2C_RETRIES` | Set number of retries on NACK |
| `I2C_TIMEOUT` | Set timeout (in units of 10 ms) |

### Basic Read/Write Operations

The simplest way to communicate: set the slave address with `I2C_SLAVE`, then use `read()` / `write()` directly.

```c
#include <stdio.h>
#include <stdlib.h>
#include <fcntl.h>
#include <unistd.h>
#include <sys/ioctl.h>
#include <linux/i2c-dev.h>
#include <linux/i2c.h>

#define I2C_BUS     "/dev/i2c-1"
#define SLAVE_ADDR  0x48   /* example: TMP102 temperature sensor */

/* Open the I2C bus and set the target slave address.
   Returns the file descriptor, or -1 on error. */
int i2c_open(const char *bus, uint8_t addr) {
    int fd = open(bus, O_RDWR);
    if (fd < 0) {
        perror("i2c_open: open");
        return -1;
    }
    if (ioctl(fd, I2C_SLAVE, addr) < 0) {
        perror("i2c_open: ioctl I2C_SLAVE");
        close(fd);
        return -1;
    }
    return fd;
}

/* Write a register address, then read back `len` bytes. */
int i2c_read_register(int fd, uint8_t reg, uint8_t *buf, size_t len) {
    /* Write the register pointer */
    if (write(fd, &reg, 1) != 1) {
        perror("i2c_read_register: write");
        return -1;
    }
    /* Read the response */
    ssize_t n = read(fd, buf, len);
    if (n != (ssize_t)len) {
        perror("i2c_read_register: read");
        return -1;
    }
    return 0;
}

/* Write a register address followed by data bytes. */
int i2c_write_register(int fd, uint8_t reg, const uint8_t *data, size_t len) {
    uint8_t buf[len + 1];
    buf[0] = reg;
    memcpy(buf + 1, data, len);
    if (write(fd, buf, len + 1) != (ssize_t)(len + 1)) {
        perror("i2c_write_register: write");
        return -1;
    }
    return 0;
}

int main(void) {
    int fd = i2c_open(I2C_BUS, SLAVE_ADDR);
    if (fd < 0) return EXIT_FAILURE;

    uint8_t raw[2];
    if (i2c_read_register(fd, 0x00, raw, 2) < 0) {
        close(fd);
        return EXIT_FAILURE;
    }

    /* TMP102: 12-bit two's complement, 0.0625 °C per LSB */
    int16_t temp_raw = ((int16_t)(raw[0] << 4)) | (raw[1] >> 4);
    double temp_c = temp_raw * 0.0625;
    printf("Temperature: %.4f °C\n", temp_c);

    close(fd);
    return EXIT_SUCCESS;
}
```

> **Limitation:** Simple `write()` + `read()` generates a STOP between the write and the read. Some devices require a **repeated START** (no STOP) between the address write and the data read. Use `I2C_RDWR` for those cases.

### SMBus Operations

SMBus is a subset of I2C with a more rigidly defined transaction set. The `i2c_smbus_*` helper functions (available via `<linux/i2c-dev.h>` on many systems, or implemented manually) wrap the `I2C_SMBUS` ioctl.

```c
#include <linux/i2c.h>
#include <linux/i2c-dev.h>
#include <sys/ioctl.h>

/* Low-level SMBus ioctl wrapper */
static inline int32_t smbus_access(int fd, char rw, uint8_t cmd,
                                   int size, union i2c_smbus_data *data) {
    struct i2c_smbus_ioctl_data args = {
        .read_write = rw,
        .command    = cmd,
        .size       = size,
        .data       = data,
    };
    return ioctl(fd, I2C_SMBUS, &args);
}

/* Read a single byte from a register */
int32_t smbus_read_byte_data(int fd, uint8_t reg) {
    union i2c_smbus_data data;
    if (smbus_access(fd, I2C_SMBUS_READ, reg, I2C_SMBUS_BYTE_DATA, &data) < 0)
        return -1;
    return data.byte & 0xFF;
}

/* Write a single byte to a register */
int smbus_write_byte_data(int fd, uint8_t reg, uint8_t value) {
    union i2c_smbus_data data = { .byte = value };
    return smbus_access(fd, I2C_SMBUS_WRITE, reg, I2C_SMBUS_BYTE_DATA, &data);
}

/* Read a 16-bit word (little-endian) from a register */
int32_t smbus_read_word_data(int fd, uint8_t reg) {
    union i2c_smbus_data data;
    if (smbus_access(fd, I2C_SMBUS_READ, reg, I2C_SMBUS_WORD_DATA, &data) < 0)
        return -1;
    return data.word & 0xFFFF;
}

/* Read a block of up to 32 bytes */
int smbus_read_block(int fd, uint8_t reg, uint8_t *buf, size_t len) {
    union i2c_smbus_data data;
    if (smbus_access(fd, I2C_SMBUS_READ, reg, I2C_SMBUS_I2C_BLOCK_DATA, &data) < 0)
        return -1;
    size_t count = (data.block[0] < len) ? data.block[0] : len;
    memcpy(buf, data.block + 1, count);
    return (int)count;
}

/* Example usage */
int main(void) {
    int fd = i2c_open("/dev/i2c-1", 0x68); /* MPU-6050 IMU */
    if (fd < 0) return EXIT_FAILURE;

    /* Wake up the MPU-6050 (clear sleep bit in PWR_MGMT_1) */
    smbus_write_byte_data(fd, 0x6B, 0x00);

    /* Read WHO_AM_I register (should return 0x68) */
    int32_t who = smbus_read_byte_data(fd, 0x75);
    printf("WHO_AM_I = 0x%02X\n", who);

    /* Read raw accelerometer X-axis (two bytes, big-endian) */
    int32_t hi = smbus_read_byte_data(fd, 0x3B);
    int32_t lo = smbus_read_byte_data(fd, 0x3C);
    int16_t accel_x = (int16_t)((hi << 8) | lo);
    printf("Accel X = %d (raw)\n", accel_x);

    close(fd);
    return EXIT_SUCCESS;
}
```

### Combined Transactions (I2C_RDWR)

The `I2C_RDWR` ioctl sends an array of `struct i2c_msg` in a single bus transaction. Each message can target a different address and direction, and they are chained with **repeated STARTs** rather than STOP conditions between them.

```c
#include <linux/i2c.h>
#include <linux/i2c-dev.h>
#include <sys/ioctl.h>

/*
 * Perform a write-then-read without a STOP in between.
 * This is required by many devices (e.g. EEPROM, sensors with
 * register-pointer model that demand a repeated START).
 */
int i2c_rdwr_read_register(int fd, uint16_t addr,
                            const uint8_t *cmd, size_t cmd_len,
                            uint8_t *buf, size_t buf_len) {
    struct i2c_msg msgs[2];
    struct i2c_rdwr_ioctl_data ioctl_data;

    /* Message 0: write the register/command bytes */
    msgs[0].addr  = addr;
    msgs[0].flags = 0;               /* write */
    msgs[0].len   = (uint16_t)cmd_len;
    msgs[0].buf   = (uint8_t *)cmd;  /* cast away const; kernel won't modify */

    /* Message 1: read the response */
    msgs[1].addr  = addr;
    msgs[1].flags = I2C_M_RD;       /* read */
    msgs[1].len   = (uint16_t)buf_len;
    msgs[1].buf   = buf;

    ioctl_data.msgs  = msgs;
    ioctl_data.nmsgs = 2;

    if (ioctl(fd, I2C_RDWR, &ioctl_data) < 0) {
        perror("I2C_RDWR");
        return -1;
    }
    return 0;
}

/* Example: read 6 bytes from register 0x3B on MPU-6050 */
int main(void) {
    int fd = open("/dev/i2c-1", O_RDWR);
    if (fd < 0) { perror("open"); return 1; }

    uint8_t reg  = 0x3B;
    uint8_t data[6];

    if (i2c_rdwr_read_register(fd, 0x68, &reg, 1, data, 6) < 0) {
        close(fd);
        return 1;
    }

    int16_t ax = (int16_t)((data[0] << 8) | data[1]);
    int16_t ay = (int16_t)((data[2] << 8) | data[3]);
    int16_t az = (int16_t)((data[4] << 8) | data[5]);
    printf("Accel: X=%d  Y=%d  Z=%d\n", ax, ay, az);

    close(fd);
    return 0;
}
```

#### Useful `i2c_msg` Flags

| Flag | Value | Meaning |
|---|---|---|
| `I2C_M_RD` | 0x0001 | Direction is read (from slave) |
| `I2C_M_TEN` | 0x0010 | 10-bit slave address |
| `I2C_M_RECV_LEN` | 0x0400 | First byte of reply is the length |
| `I2C_M_NO_RD_ACK` | 0x0800 | Do not ACK last read byte |
| `I2C_M_IGNORE_NAK` | 0x1000 | Continue even if slave NAKs |
| `I2C_M_REV_DIR_ADDR` | 0x2000 | Toggle read/write bit |
| `I2C_M_NOSTART` | 0x4000 | No START or repeated START |

### Device Probing and Enumeration

```c
/* Query what capabilities the adapter supports */
void query_i2c_funcs(int fd) {
    unsigned long funcs;
    if (ioctl(fd, I2C_FUNCS, &funcs) < 0) {
        perror("I2C_FUNCS");
        return;
    }
    printf("Adapter capabilities:\n");
    if (funcs & I2C_FUNC_I2C)              puts("  I2C_FUNC_I2C");
    if (funcs & I2C_FUNC_SMBUS_READ_BYTE)  puts("  SMBUS_READ_BYTE");
    if (funcs & I2C_FUNC_SMBUS_WRITE_BYTE) puts("  SMBUS_WRITE_BYTE");
    if (funcs & I2C_FUNC_SMBUS_BYTE_DATA)  puts("  SMBUS_BYTE_DATA");
    if (funcs & I2C_FUNC_SMBUS_WORD_DATA)  puts("  SMBUS_WORD_DATA");
    if (funcs & I2C_FUNC_SMBUS_BLOCK_DATA) puts("  SMBUS_BLOCK_DATA");
    if (funcs & I2C_FUNC_10BIT_ADDR)       puts("  10-BIT ADDRESSING");
}

/* Probe all 7-bit addresses (0x03–0x77) — mimics i2cdetect */
void i2c_probe_bus(int fd) {
    printf("Probing I2C bus...\n");
    for (int addr = 0x03; addr <= 0x77; addr++) {
        /* Use a quick write (SMBus quick command) to probe */
        union i2c_smbus_data data;
        struct i2c_smbus_ioctl_data args = {
            .read_write = I2C_SMBUS_WRITE,
            .command    = 0,
            .size       = I2C_SMBUS_QUICK,
            .data       = &data,
        };
        if (ioctl(fd, I2C_SLAVE, addr) < 0) continue;
        if (ioctl(fd, I2C_SMBUS, &args) >= 0)
            printf("  Device found at 0x%02X\n", addr);
    }
}
```

### Error Handling

```c
#include <errno.h>
#include <string.h>

typedef enum {
    I2C_OK = 0,
    I2C_ERR_OPEN,
    I2C_ERR_ADDR,
    I2C_ERR_WRITE,
    I2C_ERR_READ,
    I2C_ERR_NACK,
} I2CError;

const char *i2c_strerror(I2CError err) {
    switch (err) {
    case I2C_OK:        return "Success";
    case I2C_ERR_OPEN:  return "Failed to open I2C device";
    case I2C_ERR_ADDR:  return "Failed to set slave address";
    case I2C_ERR_WRITE: return "Write failed";
    case I2C_ERR_READ:  return "Read failed";
    case I2C_ERR_NACK:  return "Device did not acknowledge (NACK)";
    default:            return "Unknown error";
    }
}

/* Wrapper that retries on transient errors */
int i2c_write_retry(int fd, const uint8_t *buf, size_t len, int retries) {
    for (int attempt = 0; attempt <= retries; attempt++) {
        ssize_t n = write(fd, buf, len);
        if (n == (ssize_t)len) return 0;
        if (errno == ENXIO || errno == EREMOTEIO) {
            /* NACK — device not responding */
            if (attempt < retries) usleep(1000); /* 1 ms back-off */
            continue;
        }
        /* Other error — do not retry */
        break;
    }
    return -1;
}
```

#### Basic Read/Write

```rust
use i2cdev::core::I2CDevice;
use i2cdev::linux::{LinuxI2CDevice, LinuxI2CError};

fn main() -> Result<(), LinuxI2CError> {
    // Open bus 1, target address 0x48 (TMP102)
    let mut dev = LinuxI2CDevice::new("/dev/i2c-1", 0x48)?;

    // Write register pointer (0x00 = temperature register)
    dev.write(&[0x00])?;

    // Read 2 bytes back
    let mut buf = [0u8; 2];
    dev.read(&mut buf)?;

    // Decode TMP102 12-bit temperature
    let raw = (((buf[0] as i16) << 4) | ((buf[1] as i16) >> 4)) as f64;
    let temp_c = raw * 0.0625;
    println!("Temperature: {:.4} °C", temp_c);

    Ok(())
}
```

### Raw ioctl in Rust

For complete control — including `I2C_RDWR` combined transactions — you can call ioctl directly using the `nix` crate or `libc`.

**`Cargo.toml`:**
```toml
[dependencies]
libc     = "0.2"
nix      = { version = "0.27", features = ["ioctl"] }
```

```rust
use std::fs::{File, OpenOptions};
use std::os::unix::io::AsRawFd;
use libc::{c_int, c_ulong, ioctl};

// Mirror the kernel structs
#[repr(C)]
struct I2cMsg {
    addr:  u16,
    flags: u16,
    len:   u16,
    buf:   *mut u8,
}

#[repr(C)]
struct I2cRdwrIoctlData {
    msgs:  *mut I2cMsg,
    nmsgs: u32,
}

const I2C_RDWR:        c_ulong = 0x0707;
const I2C_SLAVE:       c_ulong = 0x0703;
const I2C_M_RD:        u16     = 0x0001;

/// Perform a write-then-read without a STOP between them.
fn i2c_write_read(
    fd: c_int,
    addr: u16,
    write_buf: &[u8],
    read_buf: &mut [u8],
) -> std::io::Result<()> {
    let mut msgs = [
        I2cMsg {
            addr,
            flags: 0,
            len: write_buf.len() as u16,
            buf: write_buf.as_ptr() as *mut u8,
        },
        I2cMsg {
            addr,
            flags: I2C_M_RD,
            len: read_buf.len() as u16,
            buf: read_buf.as_mut_ptr(),
        },
    ];

    let mut data = I2cRdwrIoctlData {
        msgs: msgs.as_mut_ptr(),
        nmsgs: 2,
    };

    let ret = unsafe { ioctl(fd, I2C_RDWR, &mut data) };
    if ret < 0 {
        Err(std::io::Error::last_os_error())
    } else {
        Ok(())
    }
}

fn main() -> std::io::Result<()> {
    let file = OpenOptions::new()
        .read(true)
        .write(true)
        .open("/dev/i2c-1")?;
    let fd = file.as_raw_fd();

    // Set slave address (optional when using I2C_RDWR with per-message addr)
    let ret = unsafe { ioctl(fd, I2C_SLAVE, 0x68u64) };
    if ret < 0 {
        return Err(std::io::Error::last_os_error());
    }

    let cmd = [0x3Bu8]; // Accelerometer XOUT_H register
    let mut data = [0u8; 6];
    i2c_write_read(fd, 0x68, &cmd, &mut data)?;

    let ax = i16::from_be_bytes([data[0], data[1]]);
    let ay = i16::from_be_bytes([data[2], data[3]]);
    let az = i16::from_be_bytes([data[4], data[5]]);
    println!("Accel: X={}  Y={}  Z={}", ax, ay, az);

    Ok(())
}
```

### Async I2C with Tokio

For embedded Linux applications that need concurrent I2C access alongside async networking or timers:

**`Cargo.toml`:**
```toml
[dependencies]
tokio  = { version = "1", features = ["full"] }
i2cdev = "0.6"
```

```rust
use i2cdev::core::I2CDevice;
use i2cdev::linux::LinuxI2CDevice;
use tokio::task;
use tokio::time::{interval, Duration};

async fn read_temperature_loop(bus: &str, addr: u16) {
    let bus = bus.to_string();
    let mut ticker = interval(Duration::from_secs(1));

    loop {
        ticker.tick().await;

        // I2C operations are blocking; run them in a thread pool
        let bus_clone = bus.clone();
        let result = task::spawn_blocking(move || -> Result<f64, String> {
            let mut dev = LinuxI2CDevice::new(&bus_clone, addr)
                .map_err(|e| e.to_string())?;
            dev.write(&[0x00]).map_err(|e| e.to_string())?;
            let mut buf = [0u8; 2];
            dev.read(&mut buf).map_err(|e| e.to_string())?;
            let raw = (((buf[0] as i16) << 4) | ((buf[1] as i16) >> 4)) as f64;
            Ok(raw * 0.0625)
        })
        .await;

        match result {
            Ok(Ok(temp)) => println!("Temperature: {:.4} °C", temp),
            Ok(Err(e))   => eprintln!("I2C error: {}", e),
            Err(e)       => eprintln!("Task error: {}", e),
        }
    }
}

#[tokio::main]
async fn main() {
    read_temperature_loop("/dev/i2c-1", 0x48).await;
}
```

> **Note:** Because `/dev/i2c-X` is a synchronous character device, `tokio::io::AsyncRead` does not apply. Always offload I2C calls to `spawn_blocking` to avoid blocking the async executor.

## Practical Device Examples

### Reading a Temperature Sensor (TMP102)

The TI TMP102 is a 12-bit I2C temperature sensor with a default address of 0x48.

```c
/* tmp102.c — full example with configuration */
#include <stdio.h>
#include <stdint.h>
#include <fcntl.h>
#include <unistd.h>
#include <sys/ioctl.h>
#include <linux/i2c-dev.h>

#define TMP102_ADDR     0x48
#define REG_TEMP        0x00
#define REG_CONFIG      0x01
#define REG_TLOW        0x02
#define REG_THIGH       0x03

/* Config register bits */
#define CFG_SD          (1 << 8)   /* Shutdown mode */
#define CFG_TM          (1 << 9)   /* Thermostat mode (comparator/interrupt) */
#define CFG_POL         (1 << 10)  /* Alert polarity */
#define CFG_CR_1HZ      (0 << 6)   /* Conversion rate: 1 Hz */
#define CFG_CR_4HZ      (1 << 6)   /* Conversion rate: 4 Hz (default) */
#define CFG_EM          (1 << 4)   /* Extended mode (13-bit) */

int fd;

void tmp102_write_reg16(uint8_t reg, uint16_t val) {
    uint8_t buf[3] = { reg, (val >> 8) & 0xFF, val & 0xFF };
    write(fd, buf, 3);
}

uint16_t tmp102_read_reg16(uint8_t reg) {
    write(fd, &reg, 1);
    uint8_t buf[2];
    read(fd, buf, 2);
    return ((uint16_t)buf[0] << 8) | buf[1];
}

double tmp102_read_celsius(void) {
    uint16_t raw = tmp102_read_reg16(REG_TEMP);
    /* 12-bit two's complement, MSB-aligned in 16-bit field */
    int16_t temp12 = (int16_t)(raw >> 4);
    if (temp12 & 0x800) temp12 |= 0xF000; /* sign extend */
    return temp12 * 0.0625;
}

int main(void) {
    fd = open("/dev/i2c-1", O_RDWR);
    ioctl(fd, I2C_SLAVE, TMP102_ADDR);

    /* Configure: 4 Hz conversion, 12-bit, continuous */
    tmp102_write_reg16(REG_CONFIG, CFG_CR_4HZ);

    for (int i = 0; i < 5; i++) {
        printf("Temp: %+.4f °C\n", tmp102_read_celsius());
        usleep(250000); /* 250 ms */
    }

    close(fd);
    return 0;
}
```

### Controlling an OLED Display (SSD1306)

The SSD1306 is a popular 128×64 monochrome OLED controller, commonly used with I2C on address 0x3C.

```c
/* ssd1306.c — minimal initialization and pixel write */
#include <stdio.h>
#include <string.h>
#include <stdint.h>
#include <fcntl.h>
#include <unistd.h>
#include <sys/ioctl.h>
#include <linux/i2c-dev.h>

#define SSD1306_ADDR    0x3C
#define SSD1306_CMD     0x00   /* Control byte: command stream */
#define SSD1306_DATA    0x40   /* Control byte: data stream    */
#define WIDTH           128
#define HEIGHT          64
#define PAGES           (HEIGHT / 8)

static int fd;

/* Send a command byte to the SSD1306 */
static void oled_cmd(uint8_t cmd) {
    uint8_t buf[2] = { SSD1306_CMD, cmd };
    write(fd, buf, 2);
}

/* Send multiple command bytes */
static void oled_cmds(const uint8_t *cmds, size_t len) {
    for (size_t i = 0; i < len; i++) oled_cmd(cmds[i]);
}

/* Initialize the display */
static void oled_init(void) {
    static const uint8_t init_seq[] = {
        0xAE,       /* Display OFF */
        0xD5, 0x80, /* Clock divide ratio */
        0xA8, 0x3F, /* Multiplex ratio (64-1) */
        0xD3, 0x00, /* Display offset: 0 */
        0x40,       /* Start line: 0 */
        0x8D, 0x14, /* Charge pump: enable */
        0x20, 0x00, /* Memory mode: horizontal */
        0xA1,       /* Segment remap */
        0xC8,       /* COM scan direction: remapped */
        0xDA, 0x12, /* COM pins configuration */
        0x81, 0xCF, /* Contrast */
        0xD9, 0xF1, /* Pre-charge period */
        0xDB, 0x40, /* VCOMH deselect level */
        0xA4,       /* Display from RAM */
        0xA6,       /* Normal display (not inverted) */
        0xAF,       /* Display ON */
    };
    oled_cmds(init_seq, sizeof(init_seq));
}

/* Set the memory window (columns and pages) */
static void oled_set_window(uint8_t col_start, uint8_t col_end,
                             uint8_t page_start, uint8_t page_end) {
    uint8_t cmds[] = {
        0x21, col_start,  col_end,   /* Column address */
        0x22, page_start, page_end,  /* Page address   */
    };
    oled_cmds(cmds, sizeof(cmds));
}

/* Clear the display (write all zeros) */
static void oled_clear(void) {
    oled_set_window(0, WIDTH - 1, 0, PAGES - 1);
    /* Write 1024 zero bytes as data (128 cols × 8 pages) */
    uint8_t buf[WIDTH + 1];
    buf[0] = SSD1306_DATA;
    memset(buf + 1, 0x00, WIDTH);
    for (int page = 0; page < PAGES; page++)
        write(fd, buf, WIDTH + 1);
}

/* Write a full frame buffer to the display */
static void oled_display(const uint8_t framebuf[PAGES][WIDTH]) {
    oled_set_window(0, WIDTH - 1, 0, PAGES - 1);
    uint8_t buf[WIDTH + 1];
    buf[0] = SSD1306_DATA;
    for (int page = 0; page < PAGES; page++) {
        memcpy(buf + 1, framebuf[page], WIDTH);
        write(fd, buf, WIDTH + 1);
    }
}

int main(void) {
    fd = open("/dev/i2c-1", O_RDWR);
    ioctl(fd, I2C_SLAVE, SSD1306_ADDR);

    oled_init();
    oled_clear();

    /* Draw a simple checkerboard pattern */
    uint8_t fb[PAGES][WIDTH];
    for (int p = 0; p < PAGES; p++)
        for (int c = 0; c < WIDTH; c++)
            fb[p][c] = ((p + c) % 2) ? 0xAA : 0x55;

    oled_display(fb);
    sleep(3);
    oled_clear();

    close(fd);
    return 0;
}
```

## Permissions and Security

By default, `/dev/i2c-X` is owned by `root` with group `i2c`. Access options:

```bash
# Option 1: Add user to the i2c group (recommended)
sudo usermod -aG i2c $USER
# Log out and back in for the change to take effect

# Option 2: Grant world-read/write (not recommended for production)
sudo chmod 666 /dev/i2c-1

# Option 3: Use udev rule for permanent group assignment
# /etc/udev/rules.d/99-i2c.rules:
SUBSYSTEM=="i2c-dev", GROUP="i2c", MODE="0660"
```

**Security considerations:**

- Unprivileged processes with access to `/dev/i2c-X` can access **any device on that bus**, including EEPROMs that may store cryptographic material, SPD data, or firmware.
- On production systems, grant I2C access only to specific users or service accounts.
- Use `I2C_SLAVE` (not `I2C_SLAVE_FORCE`) unless you are certain no kernel driver manages the target device — forcing access to a driver-owned device causes undefined behaviour.
- Validate all data received from I2C devices; I2C provides no authentication.

## Performance Considerations

| Factor | Details |
|---|---|
| **Bus speed** | The adapter clock rate is set in the kernel or device tree (`clock-frequency` property). Typical: 100 kHz or 400 kHz. |
| **Per-transaction overhead** | Each `write()`/`read()` syscall involves at least one context switch. For high-frequency polling, use `I2C_RDWR` to batch operations. |
| **Polling vs interrupt** | User-space I2C cannot receive hardware interrupts. For interrupt-driven devices, use a kernel driver that exposes events (e.g., via a character device or GPIO interrupt). |
| **CPU utilization** | Busy-polling I2C at > 1 kHz may consume significant CPU. Use `select()`/`poll()` where applicable, or a background thread with rate limiting. |
| **Bus sharing** | Multiple processes can open the same `/dev/i2c-X` simultaneously; the kernel serializes transactions. This is safe but may cause latency spikes. |

### Measuring Transaction Time (C)

```c
#include <time.h>

struct timespec t0, t1;
clock_gettime(CLOCK_MONOTONIC, &t0);

/* ... I2C transaction ... */

clock_gettime(CLOCK_MONOTONIC, &t1);
long ns = (t1.tv_sec - t0.tv_sec) * 1000000000L
        + (t1.tv_nsec - t0.tv_nsec);
printf("Transaction took %ld µs\n", ns / 1000);
```

### Kernel Module

# Ensure the i2c-dev module is loaded

# Scan bus 1 (safe read-only probe)

# Read a byte from address 0x48, register 0x00

# Write 0x60 to address 0x48, register 0x01

# Dump all registers from address 0x48

### Common Errors and Fixes

| Error | Likely Cause | Resolution |
|---|---|---|
| `ENOENT: No such file` | `i2c-dev` not loaded, or wrong bus number | `modprobe i2c-dev`; check `ls /dev/i2c-*` |
| `EACCES: Permission denied` | User not in `i2c` group | `sudo usermod -aG i2c $USER` |
| `ENXIO: No such device` | Slave not responding (NACK or missing) | Check wiring, pull-ups, slave address, VCC |
| `EBUSY: Device or resource busy` | Kernel driver already bound to that address | Use `I2C_SLAVE_FORCE` or unbind kernel driver |
| `ETIMEDOUT` | Bus stuck (SDA or SCL held low) | Power-cycle; issue 9 SCL clocks manually to unstick |
| Short read/write | Buffer too small or connection flaky | Check cable length, pull-up strength, bus speed |

## Summary

User-space I2C access through `/dev/i2c-X` and `ioctl` is a practical, flexible mechanism for communicating with I2C peripherals directly from applications, without requiring kernel driver development. The key concepts are:

**Architecture:** The `i2c-dev` kernel module exposes one character device per I2C bus adapter. All operations use standard POSIX file descriptors, making the interface portable and debuggable.

**Core ioctl interface:** `I2C_SLAVE` sets the target device address. `I2C_RDWR` sends batched `i2c_msg` structures with repeated STARTs — essential for devices that cannot tolerate a STOP between a write and a read. `I2C_SMBUS` provides a structured SMBus transaction API. `I2C_FUNCS` queries what the adapter supports.

**C/C++ programming:** Direct `write()`/`read()` calls handle simple transfers. The `i2c_smbus_*` wrappers cover SMBus transactions. `I2C_RDWR` with an array of `struct i2c_msg` handles combined transactions. Robust production code adds retry logic, back-off, and comprehensive error handling using `errno`.

**Rust programming:** The `i2cdev` crate provides safe, idiomatic wrappers for basic and SMBus operations. Raw `ioctl` via `libc` or `nix` gives full access to `I2C_RDWR` combined transactions. Async applications should offload blocking I2C calls to `tokio::task::spawn_blocking`.

**Practical guidance:** Always check bus capabilities with `I2C_FUNCS` before use. Prefer `I2C_SLAVE` over `I2C_SLAVE_FORCE`. Grant access via the `i2c` system group rather than world-writable permissions. Use `i2cdetect` and kernel trace events for debugging. For high-frequency polling, batch transactions with `I2C_RDWR` to minimize syscall overhead.

User-space I2C access strikes an excellent balance between convenience and capability, making it the standard approach for sensor drivers, hardware testing frameworks, and embedded Linux application development.
