# I2C Protocol Overview

## Introduction

The **Inter-Integrated Circuit (I2C)** protocol is a synchronous, multi-master, multi-slave serial communication bus invented by Philips Semiconductor (now NXP Semiconductors) in 1982. It was designed to provide a simple and efficient way to connect low-speed peripherals to microcontrollers and processors using just two wires.

## Bus Architecture

### Physical Layer

I2C uses a **two-wire interface**:

- **SDA (Serial Data Line)**: Bidirectional data line for transmitting and receiving data
- **SCL (Serial Clock Line)**: Clock signal generated by the master device

Both lines are **open-drain** (or open-collector) with pull-up resistors, allowing multiple devices to share the bus without bus contention. When a device isn't actively driving the line low, the pull-up resistor pulls it high.

### Topology

I2C supports a **multi-master, multi-slave** architecture where:
- Multiple master devices can initiate communication
- Multiple slave devices can respond to master requests
- Each slave has a unique 7-bit or 10-bit address
- Masters arbitrate bus access using clock synchronization and arbitration protocols

### Addressing

- **7-bit addressing**: Supports up to 128 devices (with 16 reserved addresses, practically 112 devices)
- **10-bit addressing**: Supports up to 1024 devices (less commonly used)

## Fundamental Concepts

### Speed Modes

I2C defines several speed modes:
- **Standard Mode**: 100 kbit/s
- **Fast Mode**: 400 kbit/s
- **Fast Mode Plus**: 1 Mbit/s
- **High-Speed Mode**: 3.4 Mbit/s
- **Ultra Fast Mode**: 5 Mbit/s

### Data Transfer

Data is transferred in **8-bit bytes** followed by an acknowledgment (ACK) or non-acknowledgment (NACK) bit. The master generates the clock, and data is valid when SCL is high and changes when SCL is low.

### Start and Stop Conditions

- **START (S)**: SDA transitions from HIGH to LOW while SCL is HIGH
- **STOP (P)**: SDA transitions from LOW to HIGH while SCL is HIGH
- **Repeated START (Sr)**: A START condition without a preceding STOP, used to change direction or address without releasing the bus

### Basic Transaction Flow

1. Master sends START condition
2. Master sends slave address (7 bits) + R/W bit (1 bit)
3. Addressed slave responds with ACK
4. Data bytes are transferred with ACK/NACK after each byte
5. Master sends STOP condition to end transaction

## Code Examples

### C/C++ Example (Bit-Banging Implementation)

Here's a simple bit-banging I2C master implementation for embedded systems:

```c
#include <stdint.h>
#include <stdbool.h>

// Hardware abstraction - implement these for your platform
void sda_high(void);
void sda_low(void);
void scl_high(void);
void scl_low(void);
bool sda_read(void);
void delay_us(uint32_t us);

#define I2C_DELAY_US 5  // For ~100kHz operation

// I2C START condition: SDA goes LOW while SCL is HIGH
void i2c_start(void) {
    sda_high();
    scl_high();
    delay_us(I2C_DELAY_US);
    sda_low();
    delay_us(I2C_DELAY_US);
    scl_low();
    delay_us(I2C_DELAY_US);
}

// I2C STOP condition: SDA goes HIGH while SCL is HIGH
void i2c_stop(void) {
    sda_low();
    scl_high();
    delay_us(I2C_DELAY_US);
    sda_high();
    delay_us(I2C_DELAY_US);
}

// Write a single bit to the I2C bus
void i2c_write_bit(bool bit) {
    if (bit) {
        sda_high();
    } else {
        sda_low();
    }
    delay_us(I2C_DELAY_US);
    scl_high();
    delay_us(I2C_DELAY_US);
    scl_low();
    delay_us(I2C_DELAY_US);
}

// Read a single bit from the I2C bus
bool i2c_read_bit(void) {
    bool bit;
    sda_high();  // Release SDA to allow slave to control it
    delay_us(I2C_DELAY_US);
    scl_high();
    delay_us(I2C_DELAY_US);
    bit = sda_read();
    scl_low();
    delay_us(I2C_DELAY_US);
    return bit;
}

// Write a byte to the I2C bus and return ACK status
bool i2c_write_byte(uint8_t byte) {
    for (int i = 7; i >= 0; i--) {
        i2c_write_bit((byte >> i) & 0x01);
    }
    // Read ACK/NACK bit (0 = ACK, 1 = NACK)
    return !i2c_read_bit();
}

// Read a byte from the I2C bus and send ACK/NACK
uint8_t i2c_read_byte(bool send_ack) {
    uint8_t byte = 0;
    for (int i = 7; i >= 0; i--) {
        byte |= (i2c_read_bit() << i);
    }
    // Send ACK (0) or NACK (1)
    i2c_write_bit(!send_ack);
    return byte;
}

// Write data to a slave device
bool i2c_write(uint8_t slave_addr, const uint8_t *data, uint16_t len) {
    i2c_start();
    
    // Send slave address with write bit (0)
    if (!i2c_write_byte(slave_addr << 1)) {
        i2c_stop();
        return false;  // No ACK received
    }
    
    // Send data bytes
    for (uint16_t i = 0; i < len; i++) {
        if (!i2c_write_byte(data[i])) {
            i2c_stop();
            return false;  // No ACK received
        }
    }
    
    i2c_stop();
    return true;
}

// Read data from a slave device
bool i2c_read(uint8_t slave_addr, uint8_t *data, uint16_t len) {
    i2c_start();
    
    // Send slave address with read bit (1)
    if (!i2c_write_byte((slave_addr << 1) | 0x01)) {
        i2c_stop();
        return false;  // No ACK received
    }
    
    // Read data bytes
    for (uint16_t i = 0; i < len; i++) {
        // Send ACK for all bytes except the last one
        data[i] = i2c_read_byte(i < len - 1);
    }
    
    i2c_stop();
    return true;
}

// Write to a register then read from it (common pattern)
bool i2c_write_read(uint8_t slave_addr, uint8_t reg_addr, 
                    uint8_t *data, uint16_t len) {
    i2c_start();
    
    // Write phase: send slave address + register address
    if (!i2c_write_byte(slave_addr << 1)) {
        i2c_stop();
        return false;
    }
    
    if (!i2c_write_byte(reg_addr)) {
        i2c_stop();
        return false;
    }
    
    // Repeated START for read phase
    i2c_start();
    
    if (!i2c_write_byte((slave_addr << 1) | 0x01)) {
        i2c_stop();
        return false;
    }
    
    // Read data bytes
    for (uint16_t i = 0; i < len; i++) {
        data[i] = i2c_read_byte(i < len - 1);
    }
    
    i2c_stop();
    return true;
}
```

### C++ Example (Hardware Abstraction Layer)

Here's a more object-oriented approach using C++:

```cpp
#include <cstdint>
#include <vector>
#include <optional>

// Abstract interface for I2C hardware
class I2CHardwareInterface {
public:
    virtual ~I2CHardwareInterface() = default;
    
    virtual bool initialize(uint32_t frequency) = 0;
    virtual bool start() = 0;
    virtual bool stop() = 0;
    virtual bool writeByte(uint8_t byte) = 0;
    virtual std::optional<uint8_t> readByte(bool sendAck) = 0;
};

// I2C Master class
class I2CMaster {
private:
    I2CHardwareInterface& hw_;
    bool initialized_;

public:
    explicit I2CMaster(I2CHardwareInterface& hw) 
        : hw_(hw), initialized_(false) {}
    
    bool init(uint32_t frequency = 100000) {
        initialized_ = hw_.initialize(frequency);
        return initialized_;
    }
    
    // Write data to a slave device
    bool write(uint8_t slaveAddr, const std::vector<uint8_t>& data) {
        if (!initialized_) return false;
        
        if (!hw_.start()) return false;
        
        // Send slave address with write bit (0)
        if (!hw_.writeByte(slaveAddr << 1)) {
            hw_.stop();
            return false;
        }
        
        // Send data bytes
        for (uint8_t byte : data) {
            if (!hw_.writeByte(byte)) {
                hw_.stop();
                return false;
            }
        }
        
        return hw_.stop();
    }
    
    // Read data from a slave device
    std::optional<std::vector<uint8_t>> read(uint8_t slaveAddr, size_t length) {
        if (!initialized_ || length == 0) return std::nullopt;
        
        if (!hw_.start()) return std::nullopt;
        
        // Send slave address with read bit (1)
        if (!hw_.writeByte((slaveAddr << 1) | 0x01)) {
            hw_.stop();
            return std::nullopt;
        }
        
        std::vector<uint8_t> data;
        data.reserve(length);
        
        // Read data bytes
        for (size_t i = 0; i < length; i++) {
            auto byte = hw_.readByte(i < length - 1);
            if (!byte.has_value()) {
                hw_.stop();
                return std::nullopt;
            }
            data.push_back(byte.value());
        }
        
        if (!hw_.stop()) return std::nullopt;
        
        return data;
    }
    
    // Write to register then read from it
    std::optional<std::vector<uint8_t>> writeRead(
        uint8_t slaveAddr, 
        uint8_t regAddr, 
        size_t readLength) {
        
        if (!initialized_ || readLength == 0) return std::nullopt;
        
        if (!hw_.start()) return std::nullopt;
        
        // Write phase
        if (!hw_.writeByte(slaveAddr << 1)) {
            hw_.stop();
            return std::nullopt;
        }
        
        if (!hw_.writeByte(regAddr)) {
            hw_.stop();
            return std::nullopt;
        }
        
        // Repeated START for read phase
        if (!hw_.start()) {
            hw_.stop();
            return std::nullopt;
        }
        
        if (!hw_.writeByte((slaveAddr << 1) | 0x01)) {
            hw_.stop();
            return std::nullopt;
        }
        
        std::vector<uint8_t> data;
        data.reserve(readLength);
        
        for (size_t i = 0; i < readLength; i++) {
            auto byte = hw_.readByte(i < readLength - 1);
            if (!byte.has_value()) {
                hw_.stop();
                return std::nullopt;
            }
            data.push_back(byte.value());
        }
        
        if (!hw_.stop()) return std::nullopt;
        
        return data;
    }
    
    // Scan the bus for devices
    std::vector<uint8_t> scanBus() {
        std::vector<uint8_t> foundDevices;
        
        for (uint16_t addr = 0x08; addr < 0x78; addr++) {
            if (hw_.start()) {
                if (hw_.writeByte(addr << 1)) {
                    foundDevices.push_back(addr);
                }
                hw_.stop();
            }
        }
        
        return foundDevices;
    }
};

// Example I2C device class (e.g., EEPROM)
class I2CDevice {
protected:
    I2CMaster& master_;
    uint8_t address_;

public:
    I2CDevice(I2CMaster& master, uint8_t address)
        : master_(master), address_(address) {}
    
    virtual ~I2CDevice() = default;
    
    uint8_t getAddress() const { return address_; }
};

// Example: 24C32 EEPROM device
class EEPROM_24C32 : public I2CDevice {
public:
    static constexpr size_t PAGE_SIZE = 32;
    static constexpr size_t TOTAL_SIZE = 4096;
    
    EEPROM_24C32(I2CMaster& master, uint8_t address)
        : I2CDevice(master, address) {}
    
    // Write a single byte to EEPROM
    bool writeByte(uint16_t memAddr, uint8_t data) {
        std::vector<uint8_t> writeData = {
            static_cast<uint8_t>(memAddr >> 8),    // High byte
            static_cast<uint8_t>(memAddr & 0xFF),  // Low byte
            data
        };
        return master_.write(address_, writeData);
    }
    
    // Read bytes from EEPROM
    std::optional<std::vector<uint8_t>> readBytes(uint16_t memAddr, size_t length) {
        std::vector<uint8_t> addrData = {
            static_cast<uint8_t>(memAddr >> 8),
            static_cast<uint8_t>(memAddr & 0xFF)
        };
        
        // Write address first
        if (!master_.write(address_, addrData)) {
            return std::nullopt;
        }
        
        // Then read data
        return master_.read(address_, length);
    }
};
```


### Rust Example (Embedded HAL)

Here's a Rust implementation using the embedded-hal traits:

```rust
// Using embedded-hal traits for hardware abstraction
use embedded_hal::blocking::i2c::{Read, Write, WriteRead};

/// I2C error types
#[derive(Debug, Clone, Copy, PartialEq)]
pub enum I2CError {
    NoAcknowledge,
    BusError,
    ArbitrationLost,
    Timeout,
    InvalidParameter,
}

/// I2C Master interface
pub struct I2CMaster<I2C> {
    i2c: I2C,
    initialized: bool,
}

impl<I2C> I2CMaster<I2C> 
where
    I2C: Write + Read + WriteRead,
{
    /// Create a new I2C master instance
    pub fn new(i2c: I2C) -> Self {
        Self {
            i2c,
            initialized: true,
        }
    }
    
    /// Write data to a slave device
    pub fn write(&mut self, addr: u8, data: &[u8]) -> Result<(), I2CError> {
        self.i2c.write(addr, data)
            .map_err(|_| I2CError::NoAcknowledge)
    }
    
    /// Read data from a slave device
    pub fn read(&mut self, addr: u8, buffer: &mut [u8]) -> Result<(), I2CError> {
        self.i2c.read(addr, buffer)
            .map_err(|_| I2CError::NoAcknowledge)
    }
    
    /// Write to register then read from it (combined transaction)
    pub fn write_read(
        &mut self, 
        addr: u8, 
        write_data: &[u8], 
        read_buffer: &mut [u8]
    ) -> Result<(), I2CError> {
        self.i2c.write_read(addr, write_data, read_buffer)
            .map_err(|_| I2CError::NoAcknowledge)
    }
    
    /// Scan the I2C bus for devices
    pub fn scan(&mut self) -> Vec<u8> {
        let mut found_devices = Vec::new();
        
        // Standard 7-bit addressing range (skip reserved addresses)
        for addr in 0x08..0x78 {
            let mut dummy = [0u8; 1];
            if self.read(addr, &mut dummy).is_ok() {
                found_devices.push(addr);
            }
        }
        
        found_devices
    }
    
    /// Check if a device is present at the given address
    pub fn is_device_present(&mut self, addr: u8) -> bool {
        let mut dummy = [0u8; 1];
        self.read(addr, &mut dummy).is_ok()
    }
}

/// Bit-banging I2C implementation (software I2C)
pub mod bitbang {
    use embedded_hal::digital::v2::{InputPin, OutputPin};
    use super::I2CError;
    
    pub struct BitBangI2C<SDA, SCL, DELAY> 
    where
        SDA: OutputPin + InputPin,
        SCL: OutputPin,
        DELAY: Fn(u32),
    {
        sda: SDA,
        scl: SCL,
        delay: DELAY,
        delay_us: u32,
    }
    
    impl<SDA, SCL, DELAY> BitBangI2C<SDA, SCL, DELAY>
    where
        SDA: OutputPin + InputPin,
        SCL: OutputPin,
        DELAY: Fn(u32),
    {
        pub fn new(sda: SDA, scl: SCL, delay: DELAY, frequency: u32) -> Self {
            let delay_us = 1_000_000 / (frequency * 2);
            Self { sda, scl, delay, delay_us }
        }
        
        fn delay(&self) {
            (self.delay)(self.delay_us);
        }
        
        /// Generate START condition
        fn start(&mut self) -> Result<(), I2CError> {
            self.sda.set_high().map_err(|_| I2CError::BusError)?;
            self.scl.set_high().map_err(|_| I2CError::BusError)?;
            self.delay();
            self.sda.set_low().map_err(|_| I2CError::BusError)?;
            self.delay();
            self.scl.set_low().map_err(|_| I2CError::BusError)?;
            self.delay();
            Ok(())
        }
        
        /// Generate STOP condition
        fn stop(&mut self) -> Result<(), I2CError> {
            self.sda.set_low().map_err(|_| I2CError::BusError)?;
            self.scl.set_high().map_err(|_| I2CError::BusError)?;
            self.delay();
            self.sda.set_high().map_err(|_| I2CError::BusError)?;
            self.delay();
            Ok(())
        }
        
        /// Write a single bit
        fn write_bit(&mut self, bit: bool) -> Result<(), I2CError> {
            if bit {
                self.sda.set_high().map_err(|_| I2CError::BusError)?;
            } else {
                self.sda.set_low().map_err(|_| I2CError::BusError)?;
            }
            self.delay();
            self.scl.set_high().map_err(|_| I2CError::BusError)?;
            self.delay();
            self.scl.set_low().map_err(|_| I2CError::BusError)?;
            self.delay();
            Ok(())
        }
        
        /// Read a single bit
        fn read_bit(&mut self) -> Result<bool, I2CError> {
            self.sda.set_high().map_err(|_| I2CError::BusError)?;
            self.delay();
            self.scl.set_high().map_err(|_| I2CError::BusError)?;
            self.delay();
            let bit = self.sda.is_high().map_err(|_| I2CError::BusError)?;
            self.scl.set_low().map_err(|_| I2CError::BusError)?;
            self.delay();
            Ok(bit)
        }
        
        /// Write a byte and return ACK status
        fn write_byte(&mut self, byte: u8) -> Result<bool, I2CError> {
            for i in (0..8).rev() {
                self.write_bit((byte >> i) & 0x01 != 0)?;
            }
            // Read ACK bit (false = ACK)
            Ok(!self.read_bit()?)
        }
        
        /// Read a byte and send ACK/NACK
        fn read_byte(&mut self, send_ack: bool) -> Result<u8, I2CError> {
            let mut byte = 0u8;
            for i in (0..8).rev() {
                if self.read_bit()? {
                    byte |= 1 << i;
                }
            }
            self.write_bit(!send_ack)?;
            Ok(byte)
        }
        
        /// Write data to slave
        pub fn write(&mut self, addr: u8, data: &[u8]) -> Result<(), I2CError> {
            self.start()?;
            
            if !self.write_byte(addr << 1)? {
                self.stop()?;
                return Err(I2CError::NoAcknowledge);
            }
            
            for &byte in data {
                if !self.write_byte(byte)? {
                    self.stop()?;
                    return Err(I2CError::NoAcknowledge);
                }
            }
            
            self.stop()
        }
        
        /// Read data from slave
        pub fn read(&mut self, addr: u8, buffer: &mut [u8]) -> Result<(), I2CError> {
            self.start()?;
            
            if !self.write_byte((addr << 1) | 0x01)? {
                self.stop()?;
                return Err(I2CError::NoAcknowledge);
            }
            
            let len = buffer.len();
            for (i, byte) in buffer.iter_mut().enumerate() {
                *byte = self.read_byte(i < len - 1)?;
            }
            
            self.stop()
        }
    }
}

pub trait I2CSensor {

/// Example implementation: Temperature sensor
pub struct TemperatureSensor<I2C> 
where
    I2C: Write + WriteRead,
{
    i2c: I2C,
    address: u8,
}

impl<I2C> TemperatureSensor<I2C>
where
    I2C: Write + WriteRead,
{
    const TEMP_REG: u8 = 0x00;
    const CONFIG_REG: u8 = 0x01;
    
    pub fn new(i2c: I2C, address: u8) -> Self {
        Self { i2c, address }
    }
    
    pub fn read_temperature(&mut self) -> Result<f32, I2CError> {
        let mut buffer = [0u8; 2];
        self.i2c
            .write_read(self.address, &[Self::TEMP_REG], &mut buffer)
            .map_err(|_| I2CError::NoAcknowledge)?;
        
        let raw = i16::from_be_bytes(buffer);
        Ok(raw as f32 * 0.0625) // Typical temperature sensor conversion
    }
    
    pub fn configure(&mut self, config: u8) -> Result<(), I2CError> {
        self.i2c
            .write(self.address, &[Self::CONFIG_REG, config])
            .map_err(|_| I2CError::NoAcknowledge)
    }
}
```

## Historical Context

The I2C protocol was developed by Philips in the early 1980s to reduce the wiring complexity in television sets and other consumer electronics. The goal was to connect various ICs on a printed circuit board using minimal pins and wires.

**Key milestones:**
- **1982**: I2C protocol introduced by Philips
- **1992**: First standardization with 100 kbit/s and 400 kbit/s modes
- **1998**: 10-bit addressing mode introduced
- **2000s**: High-speed modes (3.4 Mbit/s) and Ultra Fast-mode (5 Mbit/s) added
- **2006**: Royalty-free licensing for implementations

## Advantages and Limitations

**Advantages:**
- Simple two-wire interface minimizes pin count
- Multi-master and multi-slave capability
- Built-in addressing allows many devices on one bus
- Acknowledgment mechanism ensures reliable communication
- Well-documented and widely supported

**Limitations:**
- Relatively slow compared to SPI or parallel buses
- Limited cable length due to capacitance issues (typically < 1 meter)
- Pull-up resistors consume power and limit speed
- Open-drain architecture can be susceptible to noise
- Clock stretching can complicate timing

## Common Applications

I2C is widely used for connecting:
- Sensors (temperature, pressure, accelerometers)
- EEPROMs and flash memory
- Real-time clocks (RTCs)
- ADCs and DACs
- Display controllers
- GPIO expanders
- Power management ICs

The protocol's simplicity and efficiency make it ideal for low-speed peripheral communication in embedded systems, IoT devices, and consumer electronics.
