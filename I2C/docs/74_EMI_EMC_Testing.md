## 1. Introduction <a name="introduction"></a>

Electromagnetic Interference (EMI) and Electromagnetic Compatibility (EMC) are critical considerations
in any digital communication system, and the I2C bus is no exception. As I2C is widely deployed in
embedded systems, automotive electronics, industrial equipment, medical devices, and consumer
electronics, ensuring that a design neither emits excessive electromagnetic radiation nor succumbs to
external interference is mandatory for both regulatory approval and reliable operation.

**EMI** refers to disturbances generated by an electronic device that degrade the performance of
another device. **EMC** is the broader discipline of ensuring that devices operate compatibly in their
shared electromagnetic environment — neither causing interference nor being susceptible to it.

I2C's open-drain topology, relatively slow speeds (100 kHz to 5 MHz), and short-range nature might
suggest it is inherently low-risk. However, factors such as long cable runs, high-speed harmonics,
inadequate grounding, and improper termination routinely cause I2C systems to fail EMC qualification
tests or suffer data corruption in the field.

## 2. Fundamentals of EMI/EMC in I2C Systems <a name="fundamentals"></a>

### 2.1 Signal Integrity and EMI Relationship

EMI problems in I2C almost always originate from compromised signal integrity. The trapezoidal
waveforms of I2C transitions contain energy at odd harmonics of the fundamental clock frequency. At
400 kHz Fast-mode, significant energy exists at 1.2 MHz, 2.0 MHz, 4.0 MHz, and beyond — all
frequencies that fall within regulated emission bands.

Key relationships:

- **Rise/fall time**: Faster edges → more high-frequency content → more EMI. The I2C spec constrains
  rise times (e.g., max 300 ns in Standard-mode) partly for this reason.
- **Pull-up resistor value**: Lower resistance → faster rise times → more EMI.
- **Bus capacitance**: Higher capacitance → slower rise times → less EMI but reduced speed.
- **Cable length**: Acts as an antenna; longer cables increase both emissions and susceptibility.

### 2.2 EMC Modes

Two propagation modes are relevant to I2C:

| Mode | Description | Typical Cause |
|------|-------------|---------------|
| **Conducted** | Noise travels along conductors (power/signal lines) | Poor decoupling, ground loops |
| **Radiated** | Noise propagates through space as electromagnetic waves | Long traces/cables, inadequate shielding |

### 2.3 Coupling Mechanisms

Interference couples into I2C buses through four primary mechanisms:

1. **Capacitive (electric field) coupling** — High-frequency signals on adjacent traces induce voltage on the I2C lines via parasitic capacitance.
2. **Inductive (magnetic field) coupling** — Current loops near the I2C bus induce current via mutual inductance.
3. **Conducted coupling** — Noise on the power supply rail affects pull-up voltage, corrupting logic levels.
4. **Common-mode coupling** — Ground potential differences between devices cause differential noise.

## 3. EMI Sources in I2C Systems <a name="emi-sources"></a>

### 3.1 Internal Sources

```
┌─────────────────────────────────────────────────────────┐
│                   I2C System EMI Sources                │
│                                                         │
│  Master MCU ──── SCL transitions (dV/dt harmonics)      │
│              └── SDA transitions                        │
│                                                         │
│  Pull-up Network ── Resistor-capacitor discharge        │
│                  └── Supply ripple coupling             │
│                                                         │
│  PCB Layout ──── Trace impedance discontinuities        │
│             └── Via parasitic inductance                │
│                                                         │
│  Connectors ──── Stub resonances on off-board cables    │
└─────────────────────────────────────────────────────────┘
```

### 3.2 External Sources

- **Switching power supplies** (50 kHz–2 MHz common-mode noise)
- **RF transmitters** (GSM burst interference at 217 Hz modulation is a classic I2C disruptor)
- **Motor drives and relays** (broadband impulsive noise)
- **Adjacent high-speed buses** (SPI, USB, Ethernet crosstalk)
- **ESD events** (fast transients that overwhelm bus protection)

### 3.3 Frequency Spectrum of I2C Emissions

For a 400 kHz Fast-mode I2C clock with 100 ns rise time:

| Harmonic | Frequency | Relative Amplitude |
|----------|-----------|-------------------|
| 1st | 400 kHz | 0 dB (reference) |
| 3rd | 1.2 MHz | −9.5 dB |
| 5th | 2.0 MHz | −14 dB |
| 7th | 2.8 MHz | −17.5 dB |
| 10th | 4.0 MHz | −20 dB |

The spectral envelope rolls off at −20 dB/decade above the knee frequency (1/π·t_rise).

## 4. Regulatory Standards and Compliance <a name="standards"></a>

### 4.1 Key Standards

| Standard | Region | Applicability |
|----------|--------|---------------|
| **CISPR 32 / EN 55032** | EU/International | Multimedia equipment emissions |
| **CISPR 25** | International | Automotive in-vehicle equipment |
| **FCC Part 15** | USA | Unintentional radiators |
| **IEC 61000-4-2** | International | ESD immunity |
| **IEC 61000-4-4** | International | Electrical fast transient (EFT) immunity |
| **IEC 61000-4-6** | International | Conducted RF immunity |
| **ISO 11452** | Automotive | Radiated immunity |
| **AEC-Q100** | Automotive | Component reliability |

### 4.2 I2C-Specific Compliance Considerations

The NXP UM10204 I2C specification itself does not mandate EMC compliance — that responsibility lies
with the system designer. However, it constrains electrical parameters (rise times, logic levels,
bus capacitance) in ways that directly influence the EMC performance envelope.

Fast-mode Plus (FM+, 1 MHz) and Ultra-fast mode (UFm, 5 MHz) require particular attention because
their higher fundamental frequencies place clock harmonics directly in AM broadcast and VHF bands
where regulatory limits are most stringent.

## 5. Hardware Mitigation Techniques <a name="hardware-mitigation"></a>

### 5.1 Pull-up Resistor Optimization

The classic trade-off: lower resistance improves noise margin and speed; higher resistance reduces
emissions and power consumption.

**Recommended approach**: Use the highest pull-up resistance consistent with the required rise time
and bus capacitance.

```
t_rise ≈ 0.8473 × R_pullup × C_bus

For 400 kHz (max t_rise = 300 ns):
C_bus = 100 pF → R_pullup ≤ 3.53 kΩ  (use 3.3 kΩ)
C_bus = 400 pF → R_pullup ≤ 884 Ω    (use 820 Ω)
```

### 5.2 Series Termination Resistors

Placing small resistors (22–100 Ω) in series on SCL and SDA at the driver output:

- Limits the peak current during transitions
- Reduces edge rate without affecting static levels
- Dampens reflections on longer traces
- Acts as part of an RC low-pass filter

### 5.3 Filtering Topologies

#### RC Filter (Basic)
```
         Rs (22Ω)        
Line ───/\/\/───┬──── To bus
                │
               Cf
               (10-100pF)
                │
               GND
```

#### Common-Mode Choke
For cable applications, a common-mode choke suppresses common-mode noise while passing differential
signals largely unaffected. Typical values: 100 µH–10 mH common-mode inductance, <1 µH differential.

```
SDA ──[===]── SDA'
SCL ──[===]── SCL'    (Coupled inductor symbol, windings in same direction)
```

### 5.4 PCB Layout Best Practices

1. **Route SDA and SCL as a differential pair** — matched length, parallel routing reduces loop area.
2. **Minimize loop area** — the area enclosed by the signal path and its return path is the dominant
   antenna. Keep I2C signals close to their ground reference.
3. **Ground plane continuity** — never route I2C over splits in the ground plane.
4. **Decoupling capacitors** — place 100 nF and 10 µF ceramic capacitors at each device VDD pin,
   as close as possible to the pin.
5. **Avoid long stubs** — branch lines on the bus act as antennas; use star topology or keep stubs < 10 mm.
6. **Guard traces** — grounded traces on each side of I2C signals reduce crosstalk on dense boards.

### 5.5 Shielding

For I2C on cables:
- Use twisted-pair cable (SDA+GND and SCL+GND pairs) to reduce loop area
- Shielded cable with drain wire connected to chassis ground at one end
- For severe environments, use I2C extenders (e.g., PCA9615 differential bus buffer) to convert to
  differential signaling before cable runs > 30 cm

## 6. Software and Protocol-Level Mitigation <a name="software-mitigation"></a>

### 6.1 Bus Speed Selection

Always use the lowest I2C speed that meets application timing requirements. Lower speed means:
- Lower fundamental frequency and harmonics
- Gentler edge rates possible with larger RC filter constants
- More noise margin relative to glitch duration

### 6.2 Transaction Rate Control

Continuous I2C polling generates a periodic interference pattern at the polling interval. Strategies
to reduce this:

- **Interrupt-driven I2C** — bus active only on real events
- **Burst reads** — read all sensor data in one transaction instead of multiple short ones
- **Clock stretching** — allows slaves to control when the bus is active

### 6.3 Error Detection and Recovery

Robust software must detect and recover from EMI-induced errors:

- **NACK detection** — retry with exponential backoff
- **Bus timeout detection** — detect stuck SDA/SCL and perform bus reset
- **CRC/checksum** at application layer (I2C has no built-in CRC in standard mode)
- **Alert Response Address (ARA)** — SMBus 2.0 mechanism to identify interrupt sources
- **Packet Error Checking (PEC)** — SMBus CRC-8 over entire transaction

### 6.4 Spread Spectrum Clocking

Some MCUs support spread spectrum modulation of I2C clock, which distributes spectral energy across
a range of frequencies, reducing peak emissions at any single frequency — directly improving CISPR
test results.

## 7. Testing Methodologies <a name="testing"></a>

### 7.1 Pre-Compliance Testing (Lab)

Before formal EMC testing, perform these checks:

1. **Oscilloscope eye diagram** — captures cumulative jitter and noise margin violations
2. **Spectrum analyzer scan** — identify dominant emission frequencies from your I2C bus
3. **BCI (Bulk Current Injection)** — inject conducted RF immunity test currents onto the cable harness
4. **ESD gun simulation** — apply IEC 61000-4-2 levels to connectors and nearby metal

### 7.2 Automated Bus Monitoring

Implement software monitoring to log I2C errors during EMC testing — correlate error counts with
the RF frequencies being injected to identify susceptibility windows.

### 7.3 Test Points and Instrumentation Access

Design PCBs with:
- Test points on SCL, SDA, VDD, and GND within 25 mm of each I2C device
- JTAG/SWD debug interface active during EMC testing (enables firmware logging)
- Hardware error counters readable via debug interface

### 8.1 I2C Bus Error Monitoring and Statistics

```c
/**
 * i2c_emc_monitor.h
 * EMI/EMC monitoring subsystem for I2C buses.
 * Tracks error statistics to correlate with EMC test conditions.
 */

#ifndef I2C_EMC_MONITOR_H
#define I2C_EMC_MONITOR_H

#include <stdint.h>
#include <stdbool.h>
#include <time.h>

/* Error type classification */
typedef enum {
    I2C_ERR_NONE         = 0,
    I2C_ERR_NACK         = (1 << 0),  /* No acknowledge from slave */
    I2C_ERR_ARB_LOST     = (1 << 1),  /* Arbitration lost (multi-master) */
    I2C_ERR_BUS_TIMEOUT  = (1 << 2),  /* SCL/SDA held low too long */
    I2C_ERR_BUS_BUSY     = (1 << 3),  /* Bus busy at start of transaction */
    I2C_ERR_OVERRUN      = (1 << 4),  /* Data overrun / underrun */
    I2C_ERR_PEC          = (1 << 5),  /* SMBus PEC (CRC) mismatch */
    I2C_ERR_GLITCH       = (1 << 6),  /* Spurious pulse detected by spike filter */
} I2cErrorType;

/* Statistics window — ring buffer of recent error events */
#define EMC_STATS_WINDOW    256
#define EMC_ERROR_LOG_SIZE  1024

typedef struct {
    uint32_t      timestamp_ms;
    uint8_t       slave_addr;
    I2cErrorType  error_type;
    uint8_t       retry_count;
} I2cErrorEvent;

typedef struct {
    /* Cumulative counters */
    uint64_t total_transactions;
    uint64_t total_errors;
    uint64_t errors_by_type[8];      /* indexed by bit position in I2cErrorType */

    /* Rate tracking */
    uint32_t errors_per_second;      /* rolling 1-second window */
    uint32_t peak_error_rate;        /* peak observed error rate */
    uint32_t error_burst_count;      /* consecutive errors without success */
    uint32_t max_consecutive_errors;

    /* Bus reset statistics */
    uint32_t bus_reset_count;
    uint32_t last_reset_timestamp_ms;

    /* Event log (ring buffer) */
    I2cErrorEvent event_log[EMC_ERROR_LOG_SIZE];
    uint32_t      event_log_head;
    uint32_t      event_log_count;
} I2cEmcStats;

/* Global statistics instance */
extern I2cEmcStats g_i2c_emc_stats;

/* API */
void     i2c_emc_stats_reset(void);
void     i2c_emc_record_transaction(bool success, uint8_t addr, I2cErrorType err,
                                    uint8_t retries);
void     i2c_emc_update_rates(uint32_t now_ms);
uint32_t i2c_emc_get_error_rate(void);
bool     i2c_emc_threshold_exceeded(uint32_t max_errors_per_second);
void     i2c_emc_dump_report(void);

#endif /* I2C_EMC_MONITOR_H */
```

```c
/**
 * i2c_emc_monitor.c
 * Implementation of EMI/EMC monitoring for I2C.
 */

#include "i2c_emc_monitor.h"
#include <stdio.h>
#include <string.h>

I2cEmcStats g_i2c_emc_stats;

void i2c_emc_stats_reset(void) {
    memset(&g_i2c_emc_stats, 0, sizeof(g_i2c_emc_stats));
}

void i2c_emc_record_transaction(bool success, uint8_t addr,
                                 I2cErrorType err, uint8_t retries) {
    I2cEmcStats *s = &g_i2c_emc_stats;
    s->total_transactions++;

    if (!success) {
        s->total_errors++;
        s->error_burst_count++;
        if (s->error_burst_count > s->max_consecutive_errors) {
            s->max_consecutive_errors = s->error_burst_count;
        }
        /* Count by type (each bit position) */
        for (int bit = 0; bit < 8; bit++) {
            if (err & (1u << bit)) {
                s->errors_by_type[bit]++;
            }
        }
        /* Log the event */
        I2cErrorEvent *ev = &s->event_log[s->event_log_head];
        ev->slave_addr  = addr;
        ev->error_type  = err;
        ev->retry_count = retries;
        /* timestamp would be filled by platform HAL */
        s->event_log_head = (s->event_log_head + 1) % EMC_ERROR_LOG_SIZE;
        if (s->event_log_count < EMC_ERROR_LOG_SIZE)
            s->event_log_count++;
    } else {
        s->error_burst_count = 0; /* reset burst counter on success */
    }
}

bool i2c_emc_threshold_exceeded(uint32_t max_errors_per_second) {
    return g_i2c_emc_stats.errors_per_second > max_errors_per_second;
}

void i2c_emc_dump_report(void) {
    I2cEmcStats *s = &g_i2c_emc_stats;
    printf("=== I2C EMC Statistics Report ===\n");
    printf("Total transactions : %llu\n", (unsigned long long)s->total_transactions);
    printf("Total errors       : %llu\n", (unsigned long long)s->total_errors);
    printf("Error rate         : %.4f%%\n",
           s->total_transactions > 0
           ? (100.0 * s->total_errors / s->total_transactions) : 0.0);
    printf("Max consecutive err: %u\n", s->max_consecutive_errors);
    printf("Bus resets         : %u\n", s->bus_reset_count);
    printf("\nErrors by type:\n");
    const char *names[] = {
        "NACK", "ARB_LOST", "BUS_TIMEOUT", "BUS_BUSY",
        "OVERRUN", "PEC_MISMATCH", "GLITCH", "UNKNOWN"
    };
    for (int i = 0; i < 8; i++) {
        if (s->errors_by_type[i]) {
            printf("  %-16s: %llu\n", names[i],
                   (unsigned long long)s->errors_by_type[i]);
        }
    }
}
```

### 8.2 Robust I2C with Retry, Backoff, and Bus Reset

```c
/**
 * i2c_robust.c
 * Production-grade I2C transaction wrapper with EMI resilience.
 * Implements retry with exponential backoff and hardware bus reset.
 */

#include <stdint.h>
#include <stdbool.h>
#include "hal_i2c.h"     /* Platform HAL: hal_i2c_write, hal_i2c_read, etc. */
#include "hal_gpio.h"
#include "hal_delay.h"
#include "i2c_emc_monitor.h"

/* ---- Configuration ---- */
#define I2C_MAX_RETRIES          5
#define I2C_RETRY_BASE_DELAY_US  100    /* Initial retry delay */
#define I2C_RETRY_MAX_DELAY_US   50000  /* Cap at 50 ms */
#define I2C_BUS_RESET_CLOCKS     9      /* 9 SCL pulses to recover stuck slave */
#define I2C_BUS_RESET_PERIOD_US  5      /* Half-period for manual clock pulses */
#define I2C_STUCK_TIMEOUT_MS     25     /* Declare bus stuck after 25 ms */

/* ---- Bus Reset (bit-bang recovery) ---- */
/**
 * Performs the standard I2C bus recovery sequence:
 * 1. Drive SCL high, check SDA — if high, bus is free.
 * 2. Generate up to 9 SCL pulses until SDA goes high.
 * 3. Generate a STOP condition.
 *
 * Requires temporary GPIO control of SCL/SDA pins.
 */
static bool i2c_bus_reset(uint8_t bus_id) {
    hal_i2c_disable(bus_id);               /* Release peripheral control */
    hal_gpio_set_output(PIN_SCL, 1);
    hal_gpio_set_output(PIN_SDA, 1);
    hal_delay_us(I2C_BUS_RESET_PERIOD_US);

    /* Check if SDA is already high (bus not stuck) */
    if (hal_gpio_read(PIN_SDA)) {
        goto recovery_done;
    }

    /* Generate up to 9 SCL pulses */
    for (int i = 0; i < I2C_BUS_RESET_CLOCKS; i++) {
        hal_gpio_set_output(PIN_SCL, 0);
        hal_delay_us(I2C_BUS_RESET_PERIOD_US);
        hal_gpio_set_output(PIN_SCL, 1);
        hal_delay_us(I2C_BUS_RESET_PERIOD_US);
        if (hal_gpio_read(PIN_SDA)) {
            break;  /* SDA released — slave has finished its byte */
        }
    }

    /* Generate STOP condition: SDA low→high while SCL is high */
    hal_gpio_set_output(PIN_SDA, 0);
    hal_delay_us(I2C_BUS_RESET_PERIOD_US);
    hal_gpio_set_output(PIN_SCL, 1);
    hal_delay_us(I2C_BUS_RESET_PERIOD_US);
    hal_gpio_set_output(PIN_SDA, 1);
    hal_delay_us(I2C_BUS_RESET_PERIOD_US);

recovery_done:
    hal_i2c_enable(bus_id);               /* Return peripheral control */
    g_i2c_emc_stats.bus_reset_count++;

    return hal_gpio_read(PIN_SDA);         /* true = bus recovered */
}

/* ---- Exponential backoff delay ---- */
static void retry_delay(uint8_t attempt) {
    uint32_t delay_us = I2C_RETRY_BASE_DELAY_US;
    for (uint8_t i = 0; i < attempt; i++) {
        delay_us *= 2;
    }
    if (delay_us > I2C_RETRY_MAX_DELAY_US) {
        delay_us = I2C_RETRY_MAX_DELAY_US;
    }
    hal_delay_us(delay_us);
}

/* ---- Public API ---- */

/**
 * Robust I2C write with retry and bus recovery.
 *
 * @param bus_id    I2C peripheral index
 * @param addr      7-bit slave address
 * @param data      Pointer to data buffer
 * @param len       Number of bytes to write
 * @return          true on success, false on persistent failure
 */
bool i2c_write_robust(uint8_t bus_id, uint8_t addr,
                      const uint8_t *data, size_t len) {
    for (uint8_t attempt = 0; attempt <= I2C_MAX_RETRIES; attempt++) {
        HalI2cStatus status = hal_i2c_write(bus_id, addr, data, len);

        if (status == HAL_I2C_OK) {
            i2c_emc_record_transaction(true, addr, I2C_ERR_NONE, attempt);
            return true;
        }

        /* Classify error */
        I2cErrorType err = I2C_ERR_NONE;
        if (status == HAL_I2C_NACK)        err = I2C_ERR_NACK;
        else if (status == HAL_I2C_TIMEOUT) err = I2C_ERR_BUS_TIMEOUT;
        else if (status == HAL_I2C_ARB)     err = I2C_ERR_ARB_LOST;

        /* After 3 consecutive NACKs or any timeout, attempt bus reset */
        if (attempt >= 2 || status == HAL_I2C_TIMEOUT) {
            i2c_bus_reset(bus_id);
        }

        if (attempt < I2C_MAX_RETRIES) {
            retry_delay(attempt);
        } else {
            i2c_emc_record_transaction(false, addr, err, attempt);
        }
    }
    return false;
}

/**
 * Robust I2C read with retry and bus recovery.
 */
bool i2c_read_robust(uint8_t bus_id, uint8_t addr,
                     uint8_t *buf, size_t len) {
    for (uint8_t attempt = 0; attempt <= I2C_MAX_RETRIES; attempt++) {
        HalI2cStatus status = hal_i2c_read(bus_id, addr, buf, len);

        if (status == HAL_I2C_OK) {
            i2c_emc_record_transaction(true, addr, I2C_ERR_NONE, attempt);
            return true;
        }

        I2cErrorType err = (status == HAL_I2C_NACK) ? I2C_ERR_NACK
                         : (status == HAL_I2C_TIMEOUT) ? I2C_ERR_BUS_TIMEOUT
                         : I2C_ERR_ARB_LOST;

        if (attempt >= 2 || status == HAL_I2C_TIMEOUT) {
            i2c_bus_reset(bus_id);
        }
        if (attempt < I2C_MAX_RETRIES) {
            retry_delay(attempt);
        } else {
            i2c_emc_record_transaction(false, addr, err, attempt);
        }
    }
    return false;
}
```

### 8.3 SMBus Packet Error Checking (PEC / CRC-8)

```c
/**
 * smbus_pec.c
 * SMBus Packet Error Checking — CRC-8 over I2C address + data payload.
 * Polynomial: x^8 + x^2 + x + 1 (0x07), initial value 0x00.
 * Provides application-layer error detection for EMI-induced bit flips.
 */

#include <stdint.h>
#include <stddef.h>
#include <stdbool.h>

#define SMBUS_PEC_POLY  0x07u

/**
 * Compute CRC-8 (SMBus PEC) over a data buffer.
 * Can be called incrementally: pass previous CRC as initial value.
 */
uint8_t smbus_crc8(uint8_t crc, const uint8_t *data, size_t len) {
    for (size_t i = 0; i < len; i++) {
        crc ^= data[i];
        for (int bit = 0; bit < 8; bit++) {
            if (crc & 0x80u) {
                crc = (uint8_t)((crc << 1) ^ SMBUS_PEC_POLY);
            } else {
                crc <<= 1;
            }
        }
    }
    return crc;
}

/**
 * Build a SMBus write-byte frame with PEC appended.
 *
 * Frame format:
 *   [ADDR<<1|W] [COMMAND] [DATA] [PEC]
 *
 * PEC covers the address byte (with R/W bit), command, and data.
 */
typedef struct {
    uint8_t command;
    uint8_t data;
    uint8_t pec;
} SmbusWriteFrame;

void smbus_build_write_frame(uint8_t slave_addr, uint8_t command,
                              uint8_t data, SmbusWriteFrame *frame) {
    frame->command = command;
    frame->data    = data;

    /* CRC covers: [address byte] [command] [data] */
    uint8_t addr_byte = (uint8_t)((slave_addr << 1) | 0x00u);  /* write bit */
    uint8_t crc = smbus_crc8(0x00, &addr_byte, 1);
    crc = smbus_crc8(crc, &command, 1);
    crc = smbus_crc8(crc, &data, 1);
    frame->pec = crc;
}

/**
 * Validate a received SMBus read-word frame.
 * Returns true if PEC matches (no EMI corruption).
 */
bool smbus_validate_read_word(uint8_t slave_addr, uint8_t command,
                               uint8_t data_lo, uint8_t data_hi,
                               uint8_t received_pec) {
    /* CRC covers: [addr|W] [command] [addr|R] [data_lo] [data_hi] */
    uint8_t addr_w = (uint8_t)((slave_addr << 1) | 0x00u);
    uint8_t addr_r = (uint8_t)((slave_addr << 1) | 0x01u);

    uint8_t crc = smbus_crc8(0x00, &addr_w, 1);
    crc = smbus_crc8(crc, &command, 1);
    crc = smbus_crc8(crc, &addr_r, 1);
    crc = smbus_crc8(crc, &data_lo, 1);
    crc = smbus_crc8(crc, &data_hi, 1);

    return (crc == received_pec);
}
```

### 8.4 Adaptive Clock Speed Based on Error Rate

```cpp
/**
 * i2c_adaptive_speed.cpp
 * Dynamically reduces I2C clock speed when EMI-induced errors exceed a threshold.
 * Reduces emissions and improves noise margin in high-interference environments.
 */

#include <cstdint>
#include <array>

enum class I2cSpeed : uint32_t {
    STANDARD   = 100000,   /* 100 kHz  - lowest EMI */
    FAST       = 400000,   /* 400 kHz */
    FAST_PLUS  = 1000000,  /* 1 MHz */
    HIGH_SPEED = 3400000,  /* 3.4 MHz - highest EMI */
};

class AdaptiveI2cController {
public:
    static constexpr uint32_t kErrorThresholdStep   = 3;   /* errors per window to step down */
    static constexpr uint32_t kSuccessThreshold      = 50;  /* successes to allow step up */
    static constexpr uint32_t kWindowSizeTransactions = 100;

    AdaptiveI2cController() :
        current_speed_(I2cSpeed::FAST),
        window_errors_(0),
        window_successes_(0),
        window_count_(0)
    {}

    /**
     * Record transaction result and adapt speed if necessary.
     */
    void recordResult(bool success) {
        window_count_++;
        if (success) {
            window_successes_++;
        } else {
            window_errors_++;
        }

        if (window_count_ >= kWindowSizeTransactions) {
            adaptSpeed();
            resetWindow();
        }
    }

    I2cSpeed currentSpeed() const { return current_speed_; }

private:
    static constexpr std::array<I2cSpeed, 4> kSpeedLevels = {
        I2cSpeed::STANDARD,
        I2cSpeed::FAST,
        I2cSpeed::FAST_PLUS,
        I2cSpeed::HIGH_SPEED
    };

    void adaptSpeed() {
        int idx = speedIndex();

        if (window_errors_ >= kErrorThresholdStep && idx > 0) {
            /* Too many errors — step down speed to reduce emissions/susceptibility */
            current_speed_ = kSpeedLevels[idx - 1];
            logSpeedChange("DOWN", current_speed_);
        } else if (window_errors_ == 0 && window_successes_ >= kSuccessThreshold &&
                   idx < (int)kSpeedLevels.size() - 1) {
            /* Clean window — try stepping speed up */
            current_speed_ = kSpeedLevels[idx + 1];
            logSpeedChange("UP", current_speed_);
        }
    }

    int speedIndex() const {
        for (int i = 0; i < (int)kSpeedLevels.size(); i++) {
            if (kSpeedLevels[i] == current_speed_) return i;
        }
        return 1;
    }

    void resetWindow() {
        window_count_    = 0;
        window_errors_   = 0;
        window_successes_ = 0;
    }

    void logSpeedChange(const char *dir, I2cSpeed speed) {
        /* Platform-specific log — replace with your logging macro */
        (void)dir; (void)speed;
    }

    I2cSpeed current_speed_;
    uint32_t window_errors_;
    uint32_t window_successes_;
    uint32_t window_count_;
};
```

### 9.1 I2C Error Monitoring with Atomic Counters

```rust
// i2c_emc_monitor.rs
// Thread-safe I2C EMC statistics using Rust atomics.
// Suitable for bare-metal (no_std) and hosted environments.

#![allow(dead_code)]

use core::sync::atomic::{AtomicU32, AtomicU64, Ordering};

/// Error type bitmask — matches I2C HAL error codes.
#[repr(u8)]
#[derive(Debug, Clone, Copy, PartialEq)]
pub enum I2cErrorType {
    None        = 0,
    Nack        = 1 << 0,
    ArbLost     = 1 << 1,
    BusTimeout  = 1 << 2,
    BusBusy     = 1 << 3,
    Overrun     = 1 << 4,
    PecMismatch = 1 << 5,
    Glitch      = 1 << 6,
}

/// Lockless statistics for EMC test correlation.
pub struct I2cEmcStats {
    pub total_transactions:    AtomicU64,
    pub total_errors:          AtomicU64,
    pub nack_count:            AtomicU32,
    pub timeout_count:         AtomicU32,
    pub arb_lost_count:        AtomicU32,
    pub pec_error_count:       AtomicU32,
    pub bus_reset_count:       AtomicU32,
    pub max_consecutive_errors: AtomicU32,
    consecutive_errors:        AtomicU32,
}

impl I2cEmcStats {
    pub const fn new() -> Self {
        Self {
            total_transactions:    AtomicU64::new(0),
            total_errors:          AtomicU64::new(0),
            nack_count:            AtomicU32::new(0),
            timeout_count:         AtomicU32::new(0),
            arb_lost_count:        AtomicU32::new(0),
            pec_error_count:       AtomicU32::new(0),
            bus_reset_count:       AtomicU32::new(0),
            max_consecutive_errors: AtomicU32::new(0),
            consecutive_errors:    AtomicU32::new(0),
        }
    }

    pub fn record(&self, success: bool, error: I2cErrorType) {
        self.total_transactions.fetch_add(1, Ordering::Relaxed);

        if success {
            self.consecutive_errors.store(0, Ordering::Relaxed);
            return;
        }

        self.total_errors.fetch_add(1, Ordering::Relaxed);

        let consecutive = self.consecutive_errors.fetch_add(1, Ordering::Relaxed) + 1;
        // Update max via compare-exchange loop
        let mut max = self.max_consecutive_errors.load(Ordering::Relaxed);
        while consecutive > max {
            match self.max_consecutive_errors.compare_exchange_weak(
                max, consecutive, Ordering::Relaxed, Ordering::Relaxed,
            ) {
                Ok(_) => break,
                Err(current) => max = current,
            }
        }

        match error {
            I2cErrorType::Nack        => { self.nack_count.fetch_add(1, Ordering::Relaxed); }
            I2cErrorType::BusTimeout  => { self.timeout_count.fetch_add(1, Ordering::Relaxed); }
            I2cErrorType::ArbLost     => { self.arb_lost_count.fetch_add(1, Ordering::Relaxed); }
            I2cErrorType::PecMismatch => { self.pec_error_count.fetch_add(1, Ordering::Relaxed); }
            _ => {}
        }
    }

    pub fn error_rate_percent(&self) -> f64 {
        let total = self.total_transactions.load(Ordering::Relaxed);
        let errors = self.total_errors.load(Ordering::Relaxed);
        if total == 0 { return 0.0; }
        (errors as f64 / total as f64) * 100.0
    }

    pub fn reset(&self) {
        self.total_transactions.store(0, Ordering::Relaxed);
        self.total_errors.store(0, Ordering::Relaxed);
        self.nack_count.store(0, Ordering::Relaxed);
        self.timeout_count.store(0, Ordering::Relaxed);
        self.arb_lost_count.store(0, Ordering::Relaxed);
        self.pec_error_count.store(0, Ordering::Relaxed);
        self.bus_reset_count.store(0, Ordering::Relaxed);
        self.consecutive_errors.store(0, Ordering::Relaxed);
        self.max_consecutive_errors.store(0, Ordering::Relaxed);
    }
}

// Global static instance — safe for interrupt context use.
pub static I2C_EMC_STATS: I2cEmcStats = I2cEmcStats::new();
```

### 9.2 Robust I2C Transactions with Retry, Backoff, and Bus Recovery

```rust
// i2c_robust.rs
// Resilient I2C driver wrapper using embedded-hal traits.
// Implements retry, exponential backoff, and bus recovery for EMI environments.

use embedded_hal::i2c::{I2c, ErrorKind};
use core::fmt;

#[derive(Debug, Clone, Copy, PartialEq)]
pub enum RobustI2cError {
    /// All retries exhausted — persistent failure
    PersistentFailure {
        attempts: u8,
        last_kind: ErrorKind,
    },
    /// Bus recovery (bit-bang reset) failed
    BusRecoveryFailed,
}

impl fmt::Display for RobustI2cError {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        match self {
            Self::PersistentFailure { attempts, .. } =>
                write!(f, "I2C failed after {} attempts", attempts),
            Self::BusRecoveryFailed =>
                write!(f, "I2C bus recovery failed — SDA stuck low"),
        }
    }
}

pub struct RobustI2cConfig {
    /// Maximum number of retry attempts per transaction
    pub max_retries: u8,
    /// Base delay in microseconds for exponential backoff
    pub base_delay_us: u32,
    /// Maximum backoff delay cap in microseconds  
    pub max_delay_us: u32,
    /// After this many consecutive errors, perform bus reset
    pub reset_after_errors: u8,
}

impl Default for RobustI2cConfig {
    fn default() -> Self {
        Self {
            max_retries: 5,
            base_delay_us: 100,
            max_delay_us: 50_000,
            reset_after_errors: 3,
        }
    }
}

pub struct RobustI2c<I2C, DELAY> {
    bus: I2C,
    delay: DELAY,
    config: RobustI2cConfig,
}

impl<I2C, DELAY> RobustI2c<I2C, DELAY>
where
    I2C: I2c,
    DELAY: embedded_hal::delay::DelayNs,
{
    pub fn new(bus: I2C, delay: DELAY, config: RobustI2cConfig) -> Self {
        Self { bus, delay, config }
    }

    /// Write bytes to a slave address with retry and optional bus recovery.
    pub fn write(&mut self, addr: u8, data: &[u8]) -> Result<(), RobustI2cError> {
        let mut last_err_kind = ErrorKind::Other;

        for attempt in 0..=self.config.max_retries {
            match self.bus.write(addr, data) {
                Ok(()) => {
                    crate::I2C_EMC_STATS.record(true, crate::I2cErrorType::None);
                    return Ok(());
                }
                Err(e) => {
                    last_err_kind = e.kind();
                    let emc_err = Self::classify_error(e.kind());
                    crate::I2C_EMC_STATS.record(false, emc_err);

                    if attempt >= self.config.reset_after_errors
                        || e.kind() == ErrorKind::Bus
                    {
                        // Bus reset would be performed here via GPIO bit-banging.
                        // In embedded-hal we signal this through a platform-specific
                        // extension; shown conceptually here.
                        crate::I2C_EMC_STATS
                            .bus_reset_count
                            .fetch_add(1, core::sync::atomic::Ordering::Relaxed);
                    }

                    if attempt < self.config.max_retries {
                        self.backoff_delay(attempt);
                    }
                }
            }
        }

        Err(RobustI2cError::PersistentFailure {
            attempts: self.config.max_retries + 1,
            last_kind: last_err_kind,
        })
    }

    /// Read bytes from a slave address with retry.
    pub fn read(&mut self, addr: u8, buf: &mut [u8]) -> Result<(), RobustI2cError> {
        let mut last_err_kind = ErrorKind::Other;

        for attempt in 0..=self.config.max_retries {
            match self.bus.read(addr, buf) {
                Ok(()) => {
                    crate::I2C_EMC_STATS.record(true, crate::I2cErrorType::None);
                    return Ok(());
                }
                Err(e) => {
                    last_err_kind = e.kind();
                    crate::I2C_EMC_STATS.record(false, Self::classify_error(e.kind()));
                    if attempt < self.config.max_retries {
                        self.backoff_delay(attempt);
                    }
                }
            }
        }

        Err(RobustI2cError::PersistentFailure {
            attempts: self.config.max_retries + 1,
            last_kind: last_err_kind,
        })
    }

    /// Write then read (register-addressed read) with retry.
    pub fn write_read(
        &mut self,
        addr: u8,
        write_buf: &[u8],
        read_buf: &mut [u8],
    ) -> Result<(), RobustI2cError> {
        let mut last_err_kind = ErrorKind::Other;

        for attempt in 0..=self.config.max_retries {
            match self.bus.write_read(addr, write_buf, read_buf) {
                Ok(()) => {
                    crate::I2C_EMC_STATS.record(true, crate::I2cErrorType::None);
                    return Ok(());
                }
                Err(e) => {
                    last_err_kind = e.kind();
                    crate::I2C_EMC_STATS.record(false, Self::classify_error(e.kind()));
                    if attempt < self.config.max_retries {
                        self.backoff_delay(attempt);
                    }
                }
            }
        }

        Err(RobustI2cError::PersistentFailure {
            attempts: self.config.max_retries + 1,
            last_kind: last_err_kind,
        })
    }

    fn backoff_delay(&mut self, attempt: u8) {
        let delay_us = (self.config.base_delay_us << attempt as u32)
            .min(self.config.max_delay_us);
        self.delay.delay_us(delay_us);
    }

    fn classify_error(kind: ErrorKind) -> crate::I2cErrorType {
        match kind {
            ErrorKind::NoAcknowledge(_) => crate::I2cErrorType::Nack,
            ErrorKind::ArbitrationLoss  => crate::I2cErrorType::ArbLost,
            ErrorKind::Bus              => crate::I2cErrorType::BusTimeout,
            _                           => crate::I2cErrorType::None,
        }
    }
}
```

### 9.3 SMBus PEC (CRC-8) in Rust

```rust
// smbus_pec.rs
// SMBus Packet Error Checking (CRC-8) for I2C data integrity.
// No-std compatible — no heap allocation required.

const PEC_POLY: u8 = 0x07;

/// Compute SMBus CRC-8 over a data slice.
/// Can be chained: pass previous CRC as `initial`.
pub fn smbus_crc8(initial: u8, data: &[u8]) -> u8 {
    let mut crc = initial;
    for &byte in data {
        crc ^= byte;
        for _ in 0..8 {
            if crc & 0x80 != 0 {
                crc = crc.wrapping_shl(1) ^ PEC_POLY;
            } else {
                crc = crc.wrapping_shl(1);
            }
        }
    }
    crc
}

/// Build a SMBus write-byte transaction frame with PEC.
///
/// Returns `(command, data, pec)` — send all three bytes after the address.
pub fn smbus_write_byte_with_pec(slave_addr: u7, command: u8, data: u8) -> (u8, u8, u8) {
    let addr_byte = (slave_addr.value() << 1) | 0x00; // write
    let crc = smbus_crc8(0x00, &[addr_byte, command, data]);
    (command, data, crc)
}

/// Validate a received SMBus read-word response.
///
/// # Arguments
/// * `slave_addr` — 7-bit address
/// * `command`    — register/command byte
/// * `data`       — `[low_byte, high_byte]` received
/// * `received_pec` — PEC byte from device
///
/// Returns `Ok(u16)` with the word value if PEC matches, `Err(())` on mismatch.
pub fn smbus_read_word_validate(
    slave_addr: u8,
    command: u8,
    data: [u8; 2],
    received_pec: u8,
) -> Result<u16, ()> {
    let addr_w = (slave_addr << 1) | 0x00;
    let addr_r = (slave_addr << 1) | 0x01;

    let computed_pec = smbus_crc8(0x00, &[addr_w, command, addr_r, data[0], data[1]]);

    if computed_pec == received_pec {
        Ok(u16::from_le_bytes(data))
    } else {
        Err(())
    }
}

pub struct u7(u8);
impl u7 {

        assert!(val < 128, "I2C address must be 7-bit (0-127)");
        Self(val)
    }
    pub const fn value(&self) -> u8 { self.0 }
}
```

### 9.4 Adaptive I2C Speed Control in Rust

```rust
// i2c_adaptive_speed.rs
// Adaptive I2C clock rate selection based on observed error rate.
// Reduces bus speed (and emissions) when EMI degrades link quality.

#[derive(Debug, Clone, Copy, PartialEq, Eq, PartialOrd, Ord)]
pub enum I2cSpeed {
    Standard  = 100_000,   // 100 kHz — minimum emissions
    Fast      = 400_000,   // 400 kHz
    FastPlus  = 1_000_000, // 1 MHz
    HighSpeed = 3_400_000, // 3.4 MHz — maximum emissions
}

impl I2cSpeed {
    const LEVELS: [I2cSpeed; 4] = [
        I2cSpeed::Standard,
        I2cSpeed::Fast,
        I2cSpeed::FastPlus,
        I2cSpeed::HighSpeed,
    ];

    fn index(self) -> usize {
        Self::LEVELS.iter().position(|&s| s == self).unwrap_or(1)
    }

    fn step_down(self) -> Self {
        let idx = self.index();
        if idx > 0 { Self::LEVELS[idx - 1] } else { self }
    }

    fn step_up(self) -> Self {
        let idx = self.index();
        if idx + 1 < Self::LEVELS.len() { Self::LEVELS[idx + 1] } else { self }
    }
}

pub struct AdaptiveSpeedController {
    current_speed:    I2cSpeed,
    window_size:      u32,
    error_threshold:  u32,  // errors per window to step down
    success_threshold: u32, // clean transactions to step up
    window_errors:    u32,
    window_successes: u32,
    window_count:     u32,
}

impl AdaptiveSpeedController {
    pub fn new(initial: I2cSpeed) -> Self {
        Self {
            current_speed:    initial,
            window_size:      100,
            error_threshold:  3,
            success_threshold: 50,
            window_errors:    0,
            window_successes: 0,
            window_count:     0,
        }
    }

    /// Record a transaction outcome and adapt speed if the window is full.
    /// Returns `Some(new_speed)` if the speed changed, `None` otherwise.
    pub fn record(&mut self, success: bool) -> Option<I2cSpeed> {
        self.window_count += 1;
        if success {
            self.window_successes += 1;
        } else {
            self.window_errors += 1;
        }

        if self.window_count >= self.window_size {
            let new_speed = self.adapt();
            self.reset_window();
            if new_speed != self.current_speed {
                self.current_speed = new_speed;
                return Some(new_speed);
            }
        }
        None
    }

    pub fn current_speed(&self) -> I2cSpeed { self.current_speed }

    fn adapt(&self) -> I2cSpeed {
        if self.window_errors >= self.error_threshold {
            // Too many errors — reduce speed to lower emissions & improve margin
            self.current_speed.step_down()
        } else if self.window_errors == 0
            && self.window_successes >= self.success_threshold
        {
            // Clean window — try higher speed
            self.current_speed.step_up()
        } else {
            self.current_speed
        }
    }

    fn reset_window(&mut self) {
        self.window_count    = 0;
        self.window_errors   = 0;
        self.window_successes = 0;
    }
}
```

## 10. Summary <a name="summary"></a>

EMI/EMC compliance for I2C systems demands attention at every layer of the design:

### Key Takeaways

**Physics and Sources**
- I2C clock transitions generate harmonics that extend well into regulated frequency bands. Even
  at 100 kHz Standard-mode, harmonics at 300 kHz, 500 kHz, and beyond are radiated or conducted
  into adjacent circuitry.
- Long bus traces, off-board cables, and inadequate decoupling transform I2C signals into effective
  antennas for both emission and susceptibility.

**Hardware Countermeasures**
- Optimize pull-up resistors to the highest value that still satisfies rise-time requirements —
  this simultaneously reduces emissions and power.
- Place 22–100 Ω series resistors at driver outputs to limit edge rate and damp reflections.
- Use common-mode chokes on any I2C signals that leave the PCB.
- Maintain a continuous, uninterrupted ground plane under all I2C routing.
- For cable runs exceeding ~30 cm, convert to differential signaling using I2C buffer ICs.

**Software Countermeasures**
- Implement robust retry logic with exponential backoff to survive transient EMI bursts.
- Maintain the bus recovery (9-clock bit-bang reset) sequence for stuck-slave recovery.
- Add application-layer integrity checking via SMBus PEC (CRC-8) where data corruption is critical.
- Track per-error-type statistics during EMC testing to correlate failures with specific
  interference mechanisms.
- Adapt I2C clock speed dynamically — reduce speed when error rates climb, trading throughput for
  improved noise margin and lower emissions.

**Testing Strategy**
- Perform pre-compliance spectral scanning and eye-diagram analysis before formal testing.
- Instrument firmware to log I2C error counts during RF injection tests — this isolates which
  frequencies and levels cause susceptibility failures.
- Design PCBs with accessible test points and debug interfaces active during EMC qualification.

**Regulatory Context**
- Compliance with CISPR 32, FCC Part 15, IEC 61000-4-x, or automotive standards (CISPR 25,
  ISO 11452) is product-specific. The design techniques above address the physical mechanisms
  tested by all these frameworks, but final compliance requires laboratory validation.

By combining careful hardware design, robust software error handling, and systematic testing,
I2C systems can achieve reliable operation in demanding electromagnetic environments while meeting
applicable regulatory requirements.
