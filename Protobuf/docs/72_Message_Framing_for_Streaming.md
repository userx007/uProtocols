# Message Framing for Streaming in Protocol Buffers

## Overview

Message framing is a technique used to delimit individual Protocol Buffer messages when transmitting multiple messages over a continuous stream (like TCP sockets, files, or pipes). Since protobuf messages themselves don't include length information, you need a framing protocol to know where one message ends and the next begins.

The most common approach is **length-prefixed framing**: prepend each serialized message with its length, allowing the receiver to read exactly the right number of bytes for each message.

## Why Framing is Necessary

Protocol Buffers use a wire format that doesn't self-delimit when multiple messages are concatenated. Without framing:
- You can't distinguish between consecutive messages
- Partial reads could split messages incorrectly
- You don't know how much data to read for each message

## Length-Prefixed Framing Protocol

The standard pattern:
1. **Write**: Serialize the message, determine its length, write the length (typically as a varint), then write the message bytes
2. **Read**: Read the length, then read exactly that many bytes, deserialize into a message

## C++ Implementation

```cpp
#include <google/protobuf/io/coded_stream.h>
#include <google/protobuf/io/zero_copy_stream_impl.h>
#include <fstream>
#include "your_message.pb.h"

using google::protobuf::io::CodedInputStream;
using google::protobuf::io::CodedOutputStream;
using google::protobuf::io::OstreamOutputStream;
using google::protobuf::io::IstreamInputStream;

// Write a length-delimited message to a stream
bool writeDelimitedTo(const google::protobuf::MessageLite& message, 
                       std::ostream* output) {
    // Serialize the message
    std::string serialized;
    if (!message.SerializeToString(&serialized)) {
        return false;
    }
    
    // Create coded output stream
    OstreamOutputStream raw_output(output);
    CodedOutputStream coded_output(&raw_output);
    
    // Write the size as a varint
    coded_output.WriteVarint32(serialized.size());
    // Write the message bytes
    coded_output.WriteString(serialized);
    
    return !coded_output.HadError();
}

// Read a length-delimited message from a stream
bool readDelimitedFrom(std::istream* input, 
                        google::protobuf::MessageLite* message) {
    IstreamInputStream raw_input(input);
    CodedInputStream coded_input(&raw_input);
    
    // Read the message size
    uint32_t size;
    if (!coded_input.ReadVarint32(&size)) {
        return false;
    }
    
    // Set a limit to prevent reading too much
    CodedInputStream::Limit limit = coded_input.PushLimit(size);
    
    // Parse the message
    if (!message.ParseFromCodedStream(&coded_input)) {
        return false;
    }
    
    coded_input.PopLimit(limit);
    return coded_input.ConsumedEntireMessage();
}

// Example usage
int main() {
    // Writing multiple messages
    std::ofstream output("messages.bin", std::ios::binary);
    
    YourMessage msg1, msg2, msg3;
    msg1.set_id(1);
    msg1.set_data("First message");
    
    msg2.set_id(2);
    msg2.set_data("Second message");
    
    msg3.set_id(3);
    msg3.set_data("Third message");
    
    writeDelimitedTo(msg1, &output);
    writeDelimitedTo(msg2, &output);
    writeDelimitedTo(msg3, &output);
    output.close();
    
    // Reading multiple messages
    std::ifstream input("messages.bin", std::ios::binary);
    
    YourMessage received;
    while (readDelimitedFrom(&input, &received)) {
        std::cout << "Received message ID: " << received.id() 
                  << ", Data: " << received.data() << std::endl;
        received.Clear();
    }
    input.close();
    
    return 0;
}
```

### Alternative: Using WriteDelimitedTo/ParseDelimitedFrom (Java-style)

```cpp
// Some C++ protobuf implementations provide these helper methods
bool writeMessage(const google::protobuf::Message& message, 
                   CodedOutputStream* output) {
    // Write size
    output->WriteVarint32(message.ByteSizeLong());
    // Write message
    return message.SerializeToCodedStream(output);
}

bool readMessage(CodedInputStream* input, 
                  google::protobuf::Message* message) {
    uint32_t size;
    if (!input->ReadVarint32(&size)) return false;
    
    auto limit = input->PushLimit(size);
    bool success = message->ParseFromCodedStream(input);
    input->PopLimit(limit);
    
    return success;
}
```

## Rust Implementation

```rust
use prost::Message;
use std::io::{self, Read, Write};

// Assuming you have a protobuf message type generated by prost
// Example: your_proto_mod::YourMessage

/// Write a length-delimited message to a writer
pub fn write_delimited<M, W>(message: &M, writer: &mut W) -> io::Result<()>
where
    M: Message,
    W: Write,
{
    // Encode the message
    let mut buf = Vec::new();
    message.encode(&mut buf).map_err(|e| {
        io::Error::new(io::ErrorKind::InvalidData, e)
    })?;
    
    // Write length as varint
    write_varint(buf.len() as u64, writer)?;
    
    // Write message bytes
    writer.write_all(&buf)?;
    Ok(())
}

/// Read a length-delimited message from a reader
pub fn read_delimited<M, R>(reader: &mut R) -> io::Result<M>
where
    M: Message + Default,
    R: Read,
{
    // Read the length varint
    let len = read_varint(reader)? as usize;
    
    // Read exactly that many bytes
    let mut buf = vec![0u8; len];
    reader.read_exact(&mut buf)?;
    
    // Decode the message
    M::decode(&buf[..]).map_err(|e| {
        io::Error::new(io::ErrorKind::InvalidData, e)
    })
}

/// Write a varint (variable-length integer)
fn write_varint<W: Write>(mut value: u64, writer: &mut W) -> io::Result<()> {
    loop {
        let mut byte = (value & 0x7F) as u8;
        value >>= 7;
        if value != 0 {
            byte |= 0x80;
        }
        writer.write_all(&[byte])?;
        if value == 0 {
            break;
        }
    }
    Ok(())
}

/// Read a varint from a reader
fn read_varint<R: Read>(reader: &mut R) -> io::Result<u64> {
    let mut result: u64 = 0;
    let mut shift = 0;
    let mut buf = [0u8; 1];
    
    loop {
        reader.read_exact(&mut buf)?;
        let byte = buf[0];
        
        result |= ((byte & 0x7F) as u64) << shift;
        
        if byte & 0x80 == 0 {
            break;
        }
        
        shift += 7;
        if shift >= 64 {
            return Err(io::Error::new(
                io::ErrorKind::InvalidData,
                "Varint too long"
            ));
        }
    }
    
    Ok(result)
}

// Example usage
fn main() -> io::Result<()> {
    use std::fs::File;
    
    // Define your message type (generated by prost from .proto file)
    // mod your_proto { include!(concat!(env!("OUT_DIR"), "/your.proto.rs")); }
    // use your_proto::YourMessage;
    
    // Writing multiple messages
    let mut file = File::create("messages.bin")?;
    
    let msg1 = YourMessage {
        id: 1,
        data: "First message".to_string(),
    };
    let msg2 = YourMessage {
        id: 2,
        data: "Second message".to_string(),
    };
    let msg3 = YourMessage {
        id: 3,
        data: "Third message".to_string(),
    };
    
    write_delimited(&msg1, &mut file)?;
    write_delimited(&msg2, &mut file)?;
    write_delimited(&msg3, &mut file)?;
    
    drop(file);
    
    // Reading multiple messages
    let mut file = File::open("messages.bin")?;
    
    loop {
        match read_delimited::<YourMessage, _>(&mut file) {
            Ok(msg) => {
                println!("Received message ID: {}, Data: {}", msg.id, msg.data);
            }
            Err(e) if e.kind() == io::ErrorKind::UnexpectedEof => {
                // End of stream
                break;
            }
            Err(e) => return Err(e),
        }
    }
    
    Ok(())
}
```

### Using tokio for Async Streaming

```rust
use tokio::io::{AsyncRead, AsyncReadExt, AsyncWrite, AsyncWriteExt};

pub async fn write_delimited_async<M, W>(
    message: &M, 
    writer: &mut W
) -> io::Result<()>
where
    M: Message,
    W: AsyncWrite + Unpin,
{
    let mut buf = Vec::new();
    message.encode(&mut buf).map_err(|e| {
        io::Error::new(io::ErrorKind::InvalidData, e)
    })?;
    
    let mut len_buf = Vec::new();
    write_varint_to_vec(buf.len() as u64, &mut len_buf);
    
    writer.write_all(&len_buf).await?;
    writer.write_all(&buf).await?;
    Ok(())
}

pub async fn read_delimited_async<M, R>(
    reader: &mut R
) -> io::Result<M>
where
    M: Message + Default,
    R: AsyncRead + Unpin,
{
    let len = read_varint_async(reader).await? as usize;
    let mut buf = vec![0u8; len];
    reader.read_exact(&mut buf).await?;
    
    M::decode(&buf[..]).map_err(|e| {
        io::Error::new(io::ErrorKind::InvalidData, e)
    })
}
```

## Summary

**Message framing for streaming** solves the fundamental problem of delimiting individual Protocol Buffer messages in continuous streams. The length-prefixed framing approach is the industry standard:

- **Write operation**: Calculate message size → encode size as varint → write size → write serialized message
- **Read operation**: Read size varint → read exactly that many bytes → deserialize message
- **Key advantages**: Efficient variable-length encoding, self-synchronizing, minimal overhead
- **Common use cases**: Network protocols (gRPC uses this), file formats with multiple messages, message queues, streaming APIs

Both C++ and Rust implementations follow the same pattern, using varints for compact length encoding. The C++ version leverages protobuf's `CodedInputStream`/`CodedOutputStream` utilities, while Rust typically requires manual varint implementation with libraries like `prost`. This framing technique is essential for any application that needs to send multiple protobuf messages over TCP, files, or other stream-oriented transports.
