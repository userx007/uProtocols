# CAN Bus Authentication: Detailed Guide

## Overview

Controller Area Network (CAN) bus is a robust vehicle bus standard designed to allow microcontrollers and devices to communicate with each other without a host computer. However, traditional CAN lacks built-in security features, making it vulnerable to spoofing and replay attacks. CAN Bus Authentication addresses these vulnerabilities by implementing cryptographic mechanisms to verify message authenticity and integrity.

## The Security Problem

Traditional CAN bus has several security vulnerabilities:

1. **No Authentication**: Any node can send messages claiming to be from any other node
2. **No Encryption**: All messages are transmitted in plaintext
3. **Broadcast Nature**: All nodes receive all messages
4. **No Replay Protection**: Attackers can capture and retransmit valid messages
5. **Predictable Message IDs**: CAN IDs are standardized and well-known

## Authentication Mechanisms

### 1. Message Authentication Codes (MAC)

MACs provide cryptographic authentication by appending a tag to each message that can only be generated by parties with the shared secret key.

**Common MAC Algorithms:**
- **HMAC-SHA256**: Hash-based MAC (too large for CAN's 8-byte payload)
- **CMAC-AES**: Cipher-based MAC using AES
- **Truncated MACs**: Shortened versions to fit CAN constraints
- **GMAC**: Galois Message Authentication Code

### 2. Truncated MAC Approach

Due to CAN's 8-byte data payload limitation, we typically use truncated MACs (2-4 bytes) alongside the actual data.

**Typical Frame Structure:**
```
[CAN ID][Data (4-6 bytes)][MAC (2-4 bytes)]
```

### 3. Counter-Based Replay Protection

Include a monotonically increasing counter in authenticated data to prevent replay attacks.

## C/C++ Implementation

```c
#include <stdio.h>
#include <stdint.h>
#include <string.h>
#include <openssl/hmac.h>
#include <openssl/evp.h>

// CAN frame structure
typedef struct {
    uint32_t can_id;          // CAN identifier
    uint8_t  data_len;        // Length of data (0-8)
    uint8_t  data[8];         // Data payload
} can_frame_t;

// Authenticated CAN frame
typedef struct {
    uint32_t can_id;
    uint8_t  counter[4];      // 32-bit counter for replay protection
    uint8_t  payload[2];      // Actual data (reduced due to MAC)
    uint8_t  mac[2];          // Truncated MAC (2 bytes)
} authenticated_can_frame_t;

// Shared secret key (256-bit)
static const uint8_t shared_key[32] = {
    0x00, 0x01, 0x02, 0x03, 0x04, 0x05, 0x06, 0x07,
    0x08, 0x09, 0x0a, 0x0b, 0x0c, 0x0d, 0x0e, 0x0f,
    0x10, 0x11, 0x12, 0x13, 0x14, 0x15, 0x16, 0x17,
    0x18, 0x19, 0x1a, 0x1b, 0x1c, 0x1d, 0x1e, 0x1f
};

// Compute truncated HMAC-SHA256
void compute_mac(const uint8_t* data, size_t data_len, 
                 uint8_t* mac_out, size_t mac_len) {
    unsigned char full_mac[EVP_MAX_MD_SIZE];
    unsigned int full_mac_len;
    
    HMAC(EVP_sha256(), shared_key, sizeof(shared_key),
         data, data_len, full_mac, &full_mac_len);
    
    // Truncate to requested length
    memcpy(mac_out, full_mac, mac_len);
}

// Create authenticated CAN frame
void create_authenticated_frame(uint32_t can_id, uint32_t counter,
                                const uint8_t* payload, size_t payload_len,
                                authenticated_can_frame_t* frame) {
    // Build frame
    frame->can_id = can_id;
    
    // Counter in big-endian
    frame->counter[0] = (counter >> 24) & 0xFF;
    frame->counter[1] = (counter >> 16) & 0xFF;
    frame->counter[2] = (counter >> 8) & 0xFF;
    frame->counter[3] = counter & 0xFF;
    
    // Copy payload (max 2 bytes in this example)
    memcpy(frame->payload, payload, payload_len);
    
    // Prepare data for MAC computation
    uint8_t mac_input[4 + 4 + 2]; // CAN_ID + Counter + Payload
    memcpy(mac_input, &can_id, 4);
    memcpy(mac_input + 4, frame->counter, 4);
    memcpy(mac_input + 8, frame->payload, payload_len);
    
    // Compute MAC
    compute_mac(mac_input, sizeof(mac_input), frame->mac, 2);
}

// Verify authenticated CAN frame
int verify_authenticated_frame(const authenticated_can_frame_t* frame,
                               uint32_t* last_counter) {
    // Extract counter
    uint32_t recv_counter = ((uint32_t)frame->counter[0] << 24) |
                           ((uint32_t)frame->counter[1] << 16) |
                           ((uint32_t)frame->counter[2] << 8) |
                           ((uint32_t)frame->counter[3]);
    
    // Check for replay attack
    if (recv_counter <= *last_counter) {
        printf("Replay attack detected! Counter: %u <= %u\n", 
               recv_counter, *last_counter);
        return 0; // Reject
    }
    
    // Recompute MAC
    uint8_t mac_input[4 + 4 + 2];
    memcpy(mac_input, &frame->can_id, 4);
    memcpy(mac_input + 4, frame->counter, 4);
    memcpy(mac_input + 8, frame->payload, 2);
    
    uint8_t computed_mac[2];
    compute_mac(mac_input, sizeof(mac_input), computed_mac, 2);
    
    // Verify MAC
    if (memcmp(computed_mac, frame->mac, 2) != 0) {
        printf("MAC verification failed!\n");
        return 0; // Reject
    }
    
    // Update counter
    *last_counter = recv_counter;
    return 1; // Accept
}

// Example usage
int main() {
    printf("=== CAN Bus Authentication Example ===\n\n");
    
    // Sender side
    uint32_t counter = 1000;
    uint8_t payload[2] = {0x42, 0x13}; // Some sensor data
    authenticated_can_frame_t tx_frame;
    
    create_authenticated_frame(0x123, counter, payload, 2, &tx_frame);
    
    printf("Transmitted Frame:\n");
    printf("  CAN ID: 0x%03X\n", tx_frame.can_id);
    printf("  Counter: %u\n", counter);
    printf("  Payload: 0x%02X 0x%02X\n", 
           tx_frame.payload[0], tx_frame.payload[1]);
    printf("  MAC: 0x%02X%02X\n\n", tx_frame.mac[0], tx_frame.mac[1]);
    
    // Receiver side
    uint32_t last_counter = 999;
    
    printf("Verifying legitimate frame...\n");
    if (verify_authenticated_frame(&tx_frame, &last_counter)) {
        printf("✓ Frame authenticated successfully!\n");
        printf("  Data: 0x%02X 0x%02X\n\n", 
               tx_frame.payload[0], tx_frame.payload[1]);
    }
    
    // Simulate replay attack
    printf("Simulating replay attack (same counter)...\n");
    if (!verify_authenticated_frame(&tx_frame, &last_counter)) {
        printf("✓ Replay attack detected and blocked!\n\n");
    }
    
    // Simulate spoofing attack
    printf("Simulating spoofing attack (modified data)...\n");
    authenticated_can_frame_t spoofed_frame = tx_frame;
    spoofed_frame.payload[0] = 0xFF; // Attacker modifies data
    spoofed_frame.counter[3] = (last_counter + 1) & 0xFF;
    
    if (!verify_authenticated_frame(&spoofed_frame, &last_counter)) {
        printf("✓ Spoofing attack detected and blocked!\n\n");
    }
    
    return 0;
}
```

## Rust Implementation

```rust
use hmac::{Hmac, Mac};
use sha2::Sha256;

type HmacSha256 = Hmac<Sha256>;

// CAN frame structure
#[derive(Debug, Clone)]
pub struct CanFrame {
    pub can_id: u32,
    pub data: Vec<u8>,
}

// Authenticated CAN frame
#[derive(Debug, Clone)]
pub struct AuthenticatedCanFrame {
    pub can_id: u32,
    pub counter: u32,
    pub payload: Vec<u8>,
    pub mac: Vec<u8>,
}

// CAN Authentication Manager
pub struct CanAuthManager {
    shared_key: Vec<u8>,
    last_counter: u32,
    mac_length: usize,
}

impl CanAuthManager {
    pub fn new(key: Vec<u8>, mac_length: usize) -> Self {
        CanAuthManager {
            shared_key: key,
            last_counter: 0,
            mac_length,
        }
    }
    
    // Compute truncated MAC
    fn compute_mac(&self, data: &[u8]) -> Result<Vec<u8>, &'static str> {
        let mut mac = HmacSha256::new_from_slice(&self.shared_key)
            .map_err(|_| "Invalid key length")?;
        
        mac.update(data);
        let result = mac.finalize();
        let full_mac = result.into_bytes();
        
        // Truncate to specified length
        Ok(full_mac[..self.mac_length].to_vec())
    }
    
    // Create authenticated frame
    pub fn create_authenticated_frame(
        &self,
        can_id: u32,
        counter: u32,
        payload: &[u8],
    ) -> Result<AuthenticatedCanFrame, &'static str> {
        // Build MAC input: CAN_ID || Counter || Payload
        let mut mac_input = Vec::new();
        mac_input.extend_from_slice(&can_id.to_be_bytes());
        mac_input.extend_from_slice(&counter.to_be_bytes());
        mac_input.extend_from_slice(payload);
        
        let mac = self.compute_mac(&mac_input)?;
        
        Ok(AuthenticatedCanFrame {
            can_id,
            counter,
            payload: payload.to_vec(),
            mac,
        })
    }
    
    // Verify authenticated frame
    pub fn verify_authenticated_frame(
        &mut self,
        frame: &AuthenticatedCanFrame,
    ) -> Result<bool, &'static str> {
        // Check for replay attack
        if frame.counter <= self.last_counter {
            println!(
                "⚠ Replay attack detected! Counter: {} <= {}",
                frame.counter, self.last_counter
            );
            return Ok(false);
        }
        
        // Recompute MAC
        let mut mac_input = Vec::new();
        mac_input.extend_from_slice(&frame.can_id.to_be_bytes());
        mac_input.extend_from_slice(&frame.counter.to_be_bytes());
        mac_input.extend_from_slice(&frame.payload);
        
        let computed_mac = self.compute_mac(&mac_input)?;
        
        // Constant-time comparison
        if computed_mac != frame.mac {
            println!("⚠ MAC verification failed!");
            return Ok(false);
        }
        
        // Update counter
        self.last_counter = frame.counter;
        Ok(true)
    }
}

// Example usage
fn main() {
    println!("=== CAN Bus Authentication Example (Rust) ===\n");
    
    // Initialize authentication manager
    let shared_key = vec![
        0x00, 0x01, 0x02, 0x03, 0x04, 0x05, 0x06, 0x07,
        0x08, 0x09, 0x0a, 0x0b, 0x0c, 0x0d, 0x0e, 0x0f,
        0x10, 0x11, 0x12, 0x13, 0x14, 0x15, 0x16, 0x17,
        0x18, 0x19, 0x1a, 0x1b, 0x1c, 0x1d, 0x1e, 0x1f,
    ];
    
    let mut sender = CanAuthManager::new(shared_key.clone(), 2);
    let mut receiver = CanAuthManager::new(shared_key, 2);
    
    // Create authenticated frame
    let payload = vec![0x42, 0x13];
    let tx_frame = sender
        .create_authenticated_frame(0x123, 1000, &payload)
        .expect("Failed to create frame");
    
    println!("Transmitted Frame:");
    println!("  CAN ID: 0x{:03X}", tx_frame.can_id);
    println!("  Counter: {}", tx_frame.counter);
    println!("  Payload: {:02X?}", tx_frame.payload);
    println!("  MAC: {:02X?}\n", tx_frame.mac);
    
    // Verify legitimate frame
    println!("Verifying legitimate frame...");
    match receiver.verify_authenticated_frame(&tx_frame) {
        Ok(true) => {
            println!("✓ Frame authenticated successfully!");
            println!("  Data: {:02X?}\n", tx_frame.payload);
        }
        Ok(false) => println!("✗ Frame rejected\n"),
        Err(e) => println!("✗ Error: {}\n", e),
    }
    
    // Simulate replay attack
    println!("Simulating replay attack (same counter)...");
    match receiver.verify_authenticated_frame(&tx_frame) {
        Ok(false) => println!("✓ Replay attack detected and blocked!\n"),
        Ok(true) => println!("✗ Replay attack not detected!\n"),
        Err(e) => println!("✗ Error: {}\n", e),
    }
    
    // Simulate spoofing attack
    println!("Simulating spoofing attack (modified data)...");
    let mut spoofed_frame = tx_frame.clone();
    spoofed_frame.payload[0] = 0xFF;
    spoofed_frame.counter = receiver.last_counter + 1;
    
    match receiver.verify_authenticated_frame(&spoofed_frame) {
        Ok(false) => println!("✓ Spoofing attack detected and blocked!\n"),
        Ok(true) => println!("✗ Spoofing attack not detected!\n"),
        Err(e) => println!("✗ Error: {}\n", e),
    }
}
```

## Advanced Considerations

### 1. Key Management
- **Key Distribution**: Secure methods to distribute keys to ECUs
- **Key Rotation**: Periodic key updates to limit exposure
- **Key Storage**: Hardware security modules (HSMs) or secure elements

### 2. Performance Optimization
- **Hardware Acceleration**: Use crypto accelerators in MCUs
- **Selective Authentication**: Authenticate only critical messages
- **Precomputation**: Cache intermediate values

### 3. Synchronization
- **Counter Synchronization**: Handle counter overflow and reset
- **Time-based Counters**: Use timestamps instead of sequence numbers
- **Tolerance Windows**: Allow small counter variations

### 4. Standards and Protocols
- **SecOC (Secure Onboard Communication)**: AUTOSAR standard for CAN authentication
- **CANauth**: Proposed authentication protocol
- **VeCure**: Vehicular communication security framework

## Summary

**CAN Bus Authentication** is essential for securing modern vehicles against cyber attacks. Key takeaways:

✓ **MACs provide cryptographic authentication** to verify message authenticity and prevent spoofing

✓ **Counter-based replay protection** prevents attackers from retransmitting valid captured messages

✓ **Truncated MACs** balance security with CAN's limited 8-byte payload constraint

✓ **Trade-offs exist** between security strength, payload capacity, and computational overhead

✓ **Implementation requires** careful key management, counter synchronization, and performance optimization

✓ **Both C/C++ and Rust** provide robust libraries for implementing HMAC-based authentication

✓ **Real-world deployment** must consider AUTOSAR SecOC standards and automotive-grade requirements

The examples demonstrate practical implementations showing how to create, transmit, verify, and detect attacks on authenticated CAN messages, forming the foundation for secure automotive communication systems.
